모든 Terraform 파일은 [`terraform/`](https://github.com/f5xc-salesdemos/traffic-generator/tree/main/terraform) 디렉토리에 있습니다. 리포지토리를 클론하고 직접 배포하세요:

```bash
git clone https://github.com/f5xc-salesdemos/traffic-generator.git
cd traffic-generator/terraform
cp terraform.tfvars.example terraform.tfvars
# terraform.tfvars를 Azure 구독 ID와 대상 FQDN으로 편집하세요
```

## Terraform 구성

### Terraform 파일 구조

terraform 디렉토리에는 데모 리소스 표준을 따르는 9개의 파일이 포함되어 있습니다:

- `versions.tf` -- Terraform 및 프로바이더 버전 제약 조건 (azurerm ~> 4.0, azuread ~> 3.0)
- `providers.tf` -- Azure RM 및 Azure AD 프로바이더 구성
- `data.tf` -- 배포자 자동 확인을 위한 Azure AD 데이터 소스
- `locals.tf` -- 배포자 확인, Azure 클라우드 채택 프레임워크 리소스 네이밍, 표준 태그
- `main.tf` -- 리소스 그룹 (이름: `rg-traffic-generator-{environment}-{deployer}`)
- `variables.tf` -- 모든 입력 변수 (필수 2개, 선택 9개) 및 Azure AD를 통한 배포자 자동 확인
- `network.tf` -- VNet (10.201.0.0/16), 서브넷, 공용 IP, NSG (포트 22 SSH), NIC
- `vm.tf` -- templatefile()을 통한 cloud-init이 포함된 Ubuntu 24.04 VM
- `outputs.tf` -- 17개 출력: 모든 데모 리소스가 공유하는 15개 표준 출력 (deployer, public_ip, private_ip, ssh_command, resource_group_name, vm_name, nsg_name, vnet_name, subnet_id, component, environment, resource_group_id, vm_id, nsg_id, location) 및 2개의 컴포넌트별 출력 (target_fqdn, status_check)

### 프로바이더 및 변수

`main.tf`는 Azure 프로바이더를 구성합니다:

```hcl file=../../terraform/main.tf
```

`variables.tf`는 모든 구성 가능한 매개변수를 정의합니다. `vm_size` 기본값은 `Standard_F16s_v2` (16 vCPU 컴퓨팅 최적화)입니다 -- 확장 가이드는 [VM 사이징](#vm-사이징)을 참조하세요:

```hcl file=../../terraform/variables.tf
```

### 네트워크 인프라

`network.tf`는 VNet, 서브넷, 공용 IP, NSG (포트 22 SSH), NIC를 생성합니다:

```hcl file=../../terraform/network.tf
```

### Cloud-Init을 포함한 가상 머신

`vm.tf`는 Ubuntu 24.04 VM을 생성하고 대상 및 도구 변수와 함께 cloud-init 템플릿을 전달합니다:

```hcl file=../../terraform/vm.tf
```

### Cloud-Init 프로비저닝

`cloud-init.yaml`은 4회의 오리진 내구성 테스트를 포함하여 48시간 이상의 지속적 부하 테스트를 통해 검증된 프로덕션 유효 구성입니다. 재시도 로직이 포함된 헬퍼 라이브러리, `/var/log/cloud-init-progress.log`에 진행 로깅, 9단계 프로비저닝, 조건부 상태 보고 (`status.json`에서 ready/degraded)를 사용합니다. 프로비저닝 항목:

- **커널 튜닝** -- `somaxconn=131072`, `tcp_max_tw_buckets=4000000`, `tcp_fin_timeout=5`, 64 MiB 소켓 버퍼, `file-max=4194304`
- **NIC 최적화** -- 모든 코어에 걸친 RPS/RFS, ethtool을 통한 링 버퍼 최대화, THP=always
- **APT 패키지** -- nikto, nmap, masscan, sqlmap, hydra, medusa, ncrack, tshark, hping3, tcpdump, netcat, ngrep, iperf3, mtr, sslscan, socat, dirb, whatweb, wrk, hey, vegeta, ethtool
- **GitHub Releases** -- nuclei, dalfox, ffuf, gobuster, feroxbuster, subfinder, httpx, amass
- **Python 패키지** -- mitmproxy, sslyze, dnsrecon, fierce, theHarvester, playwright, scapy, impacket, arjun
- **Node.js 20.x** -- 봇 시뮬레이션을 위한 스텔스 플러그인이 포함된 Puppeteer
- **브라우저 자동화** -- 헤드리스 브라우저 테스트를 위한 Playwright Chromium
- **Git 클론** -- testssl.sh, recon-ng, spiderfoot, SecLists, waf-bypass, PayloadsAllTheThings
- **스위트 배포** -- 이 리포지토리에서 `/opt/traffic-generator/suites/`로 클론된 19개 트래픽 스위트
- **구성 파일** -- `TARGET_FQDN`, `TARGET_ORIGIN_IP`, `CRAPI_PORT`가 포함된 `config.env`

cloud-init은 Terraform 템플릿 변수 `${target_fqdn}`, `${target_origin_ip}`, `${tool_tier}`를 사용합니다. `$NPROC`과 같은 셸 변수는 Terraform templatefile에서 `$$NPROC`으로 이스케이프됩니다.

```yaml file=../../terraform/cloud-init.yaml
```

### 출력

`outputs.tf`는 공용/사설 IP, SSH 명령어, 리소스 식별자, 컴포넌트별 값 (target_fqdn, status_check)을 포함한 17개 출력을 노출합니다:

```hcl file=../../terraform/outputs.tf
```

### 예제 변수 파일

`terraform.tfvars.example`을 `terraform.tfvars`로 복사하고 값을 입력하세요. `.gitignore`는 자격 증명 커밋을 방지하기 위해 `terraform.tfvars`를 제외합니다:

```hcl file=../../terraform/terraform.tfvars.example
```

`subscription_id`를 Azure 구독 ID (`az account show --query id -o tsv`로 확인)로 교체하고 `target_fqdn`을 F5 XC 로드 밸런서 도메인으로 설정하세요.

## 배포

```bash
terraform init

terraform plan

terraform apply
```

배포가 성공적으로 완료되면 Terraform이 공용 IP, SSH 명령어, 대상 FQDN을 출력합니다.

## VM 사이징

이 CPU 집약적 트래픽 생성 워크로드에는 F-시리즈 컴퓨팅 최적화 VM이 권장됩니다. cloud-init 커널 튜닝 및 도구 동시성은 vCPU 수에 따라 자동으로 확장됩니다.

| VM SKU | vCPU | RAM | 네트워크 | 사용 사례 |
|---|---|---|---|---|
| Standard_F4s_v2 | 4 | 8 GiB | 10 Gbps | 경량 랩/데모 |
| Standard_F8s_v2 | 8 | 16 GiB | 12.5 Gbps | 표준 데모 |
| Standard_F16s_v2 | 16 | 32 GiB | 12.5 Gbps | 부하 테스트 (기본값) |
| Standard_F32s_v2 | 32 | 64 GiB | 16 Gbps | 대규모 벤치마킹 |
| Standard_D16s_v3 | 16 | 64 GiB | 12.5 Gbps | RAM 집약적 워크로드 |

:::tip
F16s_v2는 CDN F16s_v2 및 F32s_v2 대상에 대한 A/B/C/D 벤치마크 테스트를 통해 최적의 가격 대비 성능 옵션으로 검증되었습니다. 전체 결과는 [benchmarks/](https://github.com/f5xc-salesdemos/traffic-generator/tree/main/benchmarks)를 참조하세요.
:::

## 배포 후 검증

Cloud-init 프로비저닝은 **표준 티어의 경우 15-20분**, **풀 티어의 경우 약 25분**이 소요됩니다. VM은 SSH를 통해 즉시 접속 가능하지만, cloud-init이 완료될 때까지 도구 설치가 계속 진행됩니다.

### Cloud-Init 완료 대기

VM에 SSH로 접속하고 프로비저닝이 완료될 때까지 대기하세요:

```bash
ssh azureuser@$(terraform output -raw public_ip)

cloud-init status --wait
```

완료 시 예상 출력:

```
status: done
```

### 상태 파일 확인

cloud-init 스크립트는 완료 시 상태 파일을 작성합니다:

```bash
cat /opt/traffic-generator/status.json
```

예상 출력:

```json
{
  "status": "ready",
  "tier": "standard",
  "target_fqdn": "demo.example.com",
  "completed": "2026-04-25T14:30:00Z",
  "tools_installed": true
}
```

### 핵심 도구 검증

각 핵심 도구가 PATH에서 사용 가능한지 확인하는 검증 루프를 실행하세요:

```bash
for tool in nikto sqlmap nuclei dalfox ffuf gobuster feroxbuster nmap masscan \
  hydra medusa ncrack sslscan tshark hping3 mitmproxy sslyze subfinder httpx \
  amass arjun mtr whatweb dirb socat netcat ngrep iperf3 whois dig wrk hey; do
  if command -v "$tool" > /dev/null 2>&1; then
    echo "OK: $tool"
  else
    echo "MISSING: $tool"
  fi
done
```

### 스위트 러너 검증

러너와 스위트가 설치되었는지 확인하세요:

```bash
ls /opt/traffic-generator/suites/runner.sh
ls /opt/traffic-generator/suites/*/

/opt/traffic-generator/suites/runner.sh web-app-attacks --dry-run
```

| 출력 | 설명 | 예시 |
|---|---|---|

| `nsg_name` | 네트워크 보안 그룹 이름 | `nsg-traffic-generator-dev-jsmith` |


## 업스트림 컴포넌트에서의 연결

트래픽 생성기는 오리진 서버를 앞에 두는 F5 XC HTTP 로드 밸런서를 대상으로 합니다. 먼저 오리진 서버를 배포하고, F5 XC를 구성한 다음, `target_fqdn`을 로드 밸런서의 FQDN으로 설정하세요:

```bash
# origin-server를 배포하고 F5 XC HTTP 로드 밸런서를 생성한 후:
cat > terraform.tfvars <<EOF
subscription_id  = "your-subscription-id"
target_fqdn      = "your-xc-load-balancer.example.com"
target_origin_ip = "$(cd ../../origin-server/terraform && terraform output -raw public_ip)"
EOF
```

| 필수 변수 | 소스 | 설명 |
|---|---|---|
| `target_fqdn` | F5 XC 콘솔 | 오리진 서버를 보호하는 HTTP 로드 밸런서의 FQDN |
| `target_origin_ip` | 오리진 서버 `public_ip` 출력 | 우회 테스트를 위한 직접 오리진 IP (선택 사항) |

전체 도구 레퍼런스는 [도구 카탈로그](../04-tool-catalog/)로, 트래픽 생성을 시작하려면 [스위트](../05-suites/)로 진행하세요.
