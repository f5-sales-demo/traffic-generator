랩 세션이 완료되면 모든 Azure 리소스를 삭제하여 과금을 중지하세요.

## Terraform Destroy

`terraform/` 디렉터리에서 실행합니다:

```bash
cd terraform/

terraform plan -destroy

terraform destroy
```

Terraform이 확인을 요청합니다. `yes`를 입력하여 진행하세요.

## 정리 확인

모든 리소스가 제거되었는지 확인합니다:

```bash
RG_NAME=$(terraform output -raw resource_group_name)

az group show --name "${RG_NAME}" 2>&1 | grep -q "ResourceGroupNotFound" \
  && echo "Resource group deleted" \
  || echo "Resource group still exists"

az resource list --resource-group "${RG_NAME}" -o table 2>/dev/null
```

## 로컬 상태 정리

Terraform 상태 파일 및 캐시 파일을 제거합니다:

```bash
rm -rf .terraform .terraform.lock.hcl terraform.tfstate terraform.tfstate.backup
```

## 결과 보존 (선택 사항)

VM을 삭제하기 전에 트래픽 스위트 결과를 보존하려면 로컬로 복사하세요:

```bash
TGEN_IP=$(terraform output -raw public_ip)

# Copy all results to local machine
scp -r azureuser@${TGEN_IP}:/opt/traffic-generator/results/ ./traffic-results/

# Or copy just the latest run
LATEST=$(ssh azureuser@${TGEN_IP} 'ls -t /opt/traffic-generator/results/ | head -1')
scp -r azureuser@${TGEN_IP}:/opt/traffic-generator/results/${LATEST} ./traffic-results/
```

## F5 XC 정리

트래픽 생성기가 대상으로 사용했던 F5 XC 구성도 제거하려면 다음을 수행하세요:

1. HTTP 로드 밸런서 삭제
2. 오리진 풀 삭제
3. 테스트를 위해 생성한 WAF 정책, Bot Defense 구성, API Security 정책 또는 CSD 구성 삭제

이러한 F5 XC 리소스는 Azure 비용이 발생하지 않지만, 데모 환경이 더 이상 필요하지 않을 때 정리해야 합니다.

:::tip
반복적인 데모 사용의 경우, 전체 수명 주기를 스크립트로 자동화하는 것을 고려하세요: 데모 전에 `cd terraform && terraform apply`를 실행하고, 스위트를 실행한 후, `terraform destroy`로 정리합니다. 스탠다드 티어는 15-20분 내에 프로비저닝되므로, 당일 설정 및 정리가 실용적입니다.
:::
