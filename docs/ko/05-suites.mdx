각 트래픽 스위트는 `/opt/traffic-generator/suites/` 아래의 디렉토리이며, `runner.sh`에 의해 순서대로 실행되는 번호가 매겨진 셸 스크립트를 포함합니다. 모든 스크립트는 대상 FQDN을 첫 번째 인수로 받으며, stdout과 결과 디렉토리 모두에 출력을 기록합니다.

## api-attacks

**스크립트:** 4개 | **예상 소요 시간:** 10-17분

F5 XC를 통한 VAmPI 대상 OWASP API 보안 Top 10 테스트입니다. 테스트 사용자를 등록하고 인증 토큰을 획득한 후, BOLA, 무차별 대입 인증, 과도한 데이터 노출, 손상된 기능 수준 인가, SSRF 패턴, REST API 엔드포인트에 대한 SQL 인젝션, 숨겨진 파라미터 발견, 엔드포인트 퍼징을 테스트합니다. VAmPI 애플리케이션을 대상으로 합니다.

**F5 XC 기능:** API 보안 / API 디스커버리

| 순서 | 스크립트 | 도구 | 설명 |
|---|---|---|---|
| 01 | `01-vampi-owasp-top10.sh` | curl, jq | 종합적인 OWASP API Top 10 테스트 스위트입니다. 테스트 사용자를 등록하고 인증 토큰을 획득한 후, BOLA(다른 사용자의 데이터 접근), 무차별 대입 인증, 과도한 데이터 노출, 손상된 기능 수준 인가, SSRF 패턴, 보안 설정 오류 탐지를 테스트합니다. |
| 02 | `02-sqlmap-api.sh` | sqlmap | VAmPI REST API 엔드포인트에 대한 SQL 인젝션: 사용자 조회, 로그인(POST), 등록(POST)에 JSON 페이로드를 사용합니다. |
| 03 | `03-arjun-param-discovery.sh` | arjun | Juice Shop, DVWA, VAmPI 엔드포인트 전반에 걸친 숨겨진 파라미터 발견입니다. 인젝션 페이로드를 수용할 수 있는 문서화되지 않은 쿼리 파라미터를 발견합니다. |
| 04 | `04-ffuf-api-fuzz.sh` | ffuf | 2단계 퍼징: 모든 애플리케이션 프리픽스에 대해 일반적인 API 경로(api, swagger, graphql, admin, .env, .git) 워드리스트를 사용한 엔드포인트 발견, 그리고 주요 엔드포인트에 대한 HTTP 메서드 퍼징(GET, POST, PUT, DELETE, PATCH, OPTIONS, TRACE, PROPFIND)을 수행합니다. |

**예상 F5 XC 동작:** API 디스커버리가 VAmPI API 엔드포인트를 매핑하고 비정상적인 요청 패턴을 식별해야 합니다. API 보안 정책은 무단 접근 시도, JSON 본문의 인젝션 페이로드, 열거 패턴을 플래그 처리해야 합니다. F5 XC의 API 인벤토리에 요청 수와 함께 발견된 엔드포인트가 표시됩니다.

## bot-simulation

**스크립트:** 4개 | **예상 소요 시간:** 5-10분

F5 XC를 통한 자동화된 브라우저 활동 및 봇 유사 크롤링 패턴입니다. 헤드리스 Chrome, 스텔스 플러그인이 포함된 Puppeteer, 빠른 크롤링, 자동화된 폼 상호작용을 포함합니다. 모든 애플리케이션을 대상으로 합니다.

**F5 XC 기능:** Bot Defense

| 순서 | 스크립트 | 도구 | 설명 |
|---|---|---|---|
| 01 | `01-headless-chrome.sh` | playwright | 헤드리스 Chromium 브라우징: 대상을 로드하고 애플리케이션 경로 간 탐색, 페이지 제목 및 콘텐츠 추출, 스크린샷 캡처를 수행합니다. 스텔스 플러그인 없이 브라우저 기반 트래픽을 생성합니다(자동화로 감지 가능). |
| 02 | `02-puppeteer-stealth.sh` | puppeteer, puppeteer-extra-plugin-stealth | 스텔스 플러그인이 포함된 puppeteer-extra를 사용한 스텔스 브라우징입니다. navigator 속성, WebGL 및 기타 브라우저 핑거프린트를 패치하여 봇 감지를 우회하려고 시도합니다. |
| 03 | `03-rapid-crawl.sh` | curl, wget | curl과 wget을 사용한 고빈도 HTTP 요청을 빠르게 연속으로 수행합니다. 요청 간 지연 없이 공격적인 User-Agent 로테이션으로 스크래퍼 동작을 시뮬레이션합니다. |
| 04 | `04-form-interaction.sh` | playwright | Playwright의 폼 상호작용 API를 사용하여 DVWA 로그인, Juice Shop 등록, VAmPI API 엔드포인트에 대한 자동화된 폼 작성 및 제출을 수행합니다. |

**예상 F5 XC 동작:** Bot Defense는 브라우저 핑거프린트 분석, 요청 타이밍, JavaScript 챌린지 결과를 기반으로 트래픽을 자동화된 것으로 분류해야 합니다. 스텔스 없는 헤드리스 Chrome은 즉시 감지되어야 합니다. 스텔스 Puppeteer는 고급 우회 기술이 감지를 통과하는지 테스트합니다. Bot Defense 대시보드에 봇 분류 카테고리와 완화 조치가 표시됩니다.

## demoapp-attacks

**스크립트:** 2개 | **예상 소요 시간:** 2-3분

F5 DemoApp의 내장 WAF 테스트 엔드포인트(`/WAF/SQL`, `/WAF/XSS`, `/WAF/DIR`)에 대한 타겟 공격 페이로드입니다. WAF 차단 동작을 시연하도록 설계된 엔드포인트에 직접 SQL 인젝션, XSS, 경로 순회 페이로드를 전송합니다. 표준 오리진 서버 애플리케이션 대신 DemoApp 콘텐츠 서버를 대상으로 할 때 이 스위트를 사용하십시오.

**F5 XC 기능:** WAF (웹 애플리케이션 방화벽)

| 순서 | 스크립트 | 도구 | 설명 |
|---|---|---|---|
| 01 | `01-sqli-waf-endpoint.sh` | curl, python3 | `/WAF/SQL?age=`에 대한 15개 SQL 인젝션 페이로드로, UNION SELECT, DROP TABLE, WAITFOR DELAY, xp_cmdshell, 주석 기반 우회, 불리언 로직 페이로드를 포함합니다. |
| 02 | `02-xss-waf-endpoint.sh` | curl, python3 | `/WAF/XSS?update=`에 대한 18개 XSS 페이로드로, 스크립트 태그, 이벤트 핸들러(onerror, onload, ontoggle, onfocus), base64 eval, 스타일 표현식, 폴리글랏 페이로드, 쿠키 탈취를 포함합니다. |

**예상 F5 XC 동작:** WAF는 SQL 인젝션 및 XSS 페이로드가 포함된 요청을 차단하고 차단 페이지를 반환해야 합니다. 보안 이벤트 대시보드에 각 차단된 요청에 대한 위반 카테고리와 시그니처 매칭이 표시됩니다.

## javascript-exploits

**스크립트:** 3개 | **예상 소요 시간:** 3-5분

Client-Side Defense 감지를 트리거하는 클라이언트 측 스크립트 인젝션 및 DOM 조작 패턴입니다. Magecart 스타일 스키머를 모방하는 인라인 스크립트 인젝션, 헤드리스 Chromium을 통한 DOM 조작, 서드파티 스크립트 시뮬레이션을 포함합니다. CSD 데모 애플리케이션을 대상으로 합니다.

**F5 XC 기능:** Client-Side Defense (CSD)

| 순서 | 스크립트 | 도구 | 설명 |
|---|---|---|---|
| 01 | `01-inline-script-injection.sh` | curl | Magecart 스타일 신용카드 스키머 및 데이터 탈취 스크립트를 모방하도록 설계된 인라인 JavaScript 페이로드가 포함된 요청을 전송합니다. |
| 02 | `02-dom-manipulation.sh` | playwright | 헤드리스 Chromium을 사용하여 CSD 데모 페이지를 로드하고 DOM 수정을 주입합니다: 스크립트 태그 추가, 폼 액션 수정, 폼 데이터를 탈취하는 보이지 않는 iframe 삽입을 수행합니다. |
| 03 | `03-third-party-script-sim.sh` | curl, playwright | 외부 JavaScript 소스를 참조하는 요청을 주입하고 페이지 컨텍스트에 무단 스크립트를 로드하려고 시도하여 손상된 서드파티 스크립트를 시뮬레이션합니다. |

**예상 F5 XC 동작:** Client-Side Defense는 보호된 페이지에 대한 무단 스크립트 수정을 감지해야 합니다. CSD 대시보드에 새로운 스크립트 소스, 수정된 DOM 요소, 데이터 탈취 시도에 대한 알림이 표시됩니다. 이는 CSD 2단계 기능을 검증합니다.

<Aside type="note">
javascript-exploits 스위트는 오리진 서버의 CSD 데모 애플리케이션과 함께 작동하도록 설계되었습니다. 이 스위트를 실행하기 전에 오리진 서버에서 `/csd-demo/` 경로가 활성화되어 있는지 확인하십시오.
</Aside>

## reconnaissance

**스크립트:** 6개 | **예상 소요 시간:** 8-15분

대상 인프라에 대한 네트워크 스캐닝, 서비스 열거, 디렉토리 무차별 대입입니다. nmap 서비스 감지, masscan 포트 스윕, 디렉토리 무차별 대입, 서브도메인 열거, 웹 기술 핑거프린팅, DNS 정찰을 포함합니다. 모든 애플리케이션을 대상으로 합니다.

**F5 XC 기능:** WAF / Bot Defense

| 순서 | 스크립트 | 도구 | 설명 |
|---|---|---|---|
| 01 | `01-nmap-scan.sh` | nmap | 대상의 HTTP/HTTPS 포트에 대한 서비스 버전 감지 및 기본 스크립트 스캔입니다. |
| 02 | `02-masscan-sweep.sh` | masscan | 속도 제한이 적용된 일반 웹 포트(80, 443, 8080, 8443)의 고속 포트 스캔입니다. |
| 03 | `03-gobuster-dirs.sh` | gobuster | 모든 애플리케이션 프리픽스에 대해 SecLists 일반 워드리스트를 사용한 디렉토리 무차별 대입입니다. |
| 04 | `04-subfinder-enum.sh` | subfinder, httpx | 대상 도메인의 수동 서브도메인 열거 후 발견된 호스트의 HTTP 프로빙을 수행합니다. |
| 05 | `05-whatweb-fingerprint.sh` | whatweb | 대상 및 모든 애플리케이션 경로의 웹 기술 핑거프린팅입니다. |
| 06 | `06-dns-recon.sh` | dnsrecon, fierce, dig | DNS 레코드 열거, 존 전송 시도, 대상 도메인의 역방향 조회입니다. |

**예상 F5 XC 동작:** WAF는 스캐닝 활동을 감지하고 로깅해야 합니다. Bot Defense는 빠른 순차 요청을 자동화된 것으로 분류할 수 있습니다. 보안 이벤트 대시보드에 정찰 시그니처와 스캐닝 도구 User-Agent 문자열이 표시됩니다.

**F5 XC 기능:** API 보안

## ssl-scanning

**스크립트:** 3개 | **예상 소요 시간:** 3-5분

F5 XC 로드 밸런서의 HTTPS 엔드포인트에 대한 TLS/SSL 구성 분석입니다. 암호화 스위트, 프로토콜 버전, 인증서 세부 정보를 열거하고 알려진 TLS 취약점을 확인합니다. F5 XC 로드 밸런서 엔드포인트를 대상으로 합니다.

**F5 XC 기능:** WAF (정보 제공)

| 순서 | 스크립트 | 도구 | 설명 |
|---|---|---|---|
| 01 | `01-sslscan.sh` | sslscan | 지원되는 암호화 스위트, 프로토콜 버전, 인증서 세부 정보를 열거합니다. |
| 02 | `02-sslyze.sh` | sslyze | 인증서 검증, 암호화 스위트 순서, 취약점 확인(Heartbleed, ROBOT 등)을 포함한 종합적인 TLS 분석입니다. |
| 03 | `03-testssl.sh` | testssl.sh | testssl.sh 프레임워크를 사용한 전체 TLS 평가입니다. 프로토콜 지원, 암호화 선호도, 헤더 분석, 알려진 취약점을 테스트합니다. |

**예상 F5 XC 동작:** SSL 스캐닝은 비정상적인 암호화 스위트 제안을 포함하는 많은 TLS 핸드셰이크를 생성합니다. F5 XC는 일반적으로 이를 차단하지 않지만, 트래픽 패턴은 액세스 로그에서 확인할 수 있습니다. 주요 가치는 F5 XC TLS 구성이 보안 모범 사례를 충족하는지 검증하는 것입니다.

## traffic-generation

**스크립트:** 4개 | **예상 소요 시간:** 5-10분 (구성 가능)

기준선 측정 및 부하 테스트를 위한 대용량 정상 HTTP 트래픽입니다. 공격 스위트 트래픽과 비교하기 위해 F5 XC 분석에서 정상 트래픽 패턴을 설정합니다. 모든 애플리케이션을 대상으로 합니다.

**F5 XC 기능:** 전체 (기준선 비교)

| 순서 | 스크립트 | 도구 | 설명 |
|---|---|---|---|
| 01 | `01-baseline-http.sh` | curl | 표준 브라우저 User-Agent를 사용하여 모든 애플리케이션 경로에 순차적 HTTP GET 요청을 수행합니다. F5 XC 분석에서 정상 트래픽의 기준선을 설정합니다. |
| 02 | `02-concurrent-load.sh` | curl | 다중 동시 연결을 사용한 curl 병렬 HTTP 요청입니다. 부하 처리를 테스트하고 처리량 기준선을 설정합니다. |
| 03 | `03-mixed-methods.sh` | curl | 유효한 페이로드를 사용하여 API 엔드포인트에 대한 혼합 HTTP 메서드(GET, POST, PUT, DELETE)입니다. 공격 트래픽과 비교하기 위한 정상 API 트래픽 패턴을 생성합니다. |
| 04 | `04-user-journey.sh` | playwright | 실제 사용자 여정을 시뮬레이션합니다: 표준 브라우저 설정의 Playwright를 사용하여 페이지 탐색, 제품 검색, 폼 작성, 애플리케이션 간 이동을 수행합니다. |

**예상 F5 XC 동작:** 이 스위트의 모든 요청은 WAF 또는 Bot Defense 개입 없이 통과해야 합니다. 이 스위트를 사용하여 분석 대시보드에 "클린" 트래픽을 생성한 후, 공격 스위트 트래픽과 비교하여 정상 요청 패턴과 악성 요청 패턴의 차이를 시연하십시오.

## waf-encoding-evasion

**스크립트:** 7개 | **예상 소요 시간:** 5-10분

WAF가 검사 전에 페이로드를 적절하게 정규화하고 디코딩하는지 테스트하도록 설계된 다층 인코딩 우회 공격입니다. 단일, 이중, 삼중 URL 인코딩; HTML 엔티티 인코딩(10진수, 16진수, 명명된, 제로 패딩); 유니코드/UTF-8 삽입(제로 너비 공백, BOM, 전각 문자, 소프트 하이픈, 오버롱 시퀀스); 혼합/중첩 다층 인코딩(URL 인코딩된 HTML 엔티티, 엔티티 내부의 이중 URL, 3층 스택); 널 바이트 인젝션; IIS %uXXXX 인코딩; 대소문자 조작; 청크드 전송 인코딩 본문 분할; Base64 파라미터 우회; 헤더/쿠키 인코딩된 페이로드를 다룹니다. 모든 애플리케이션을 대상으로 합니다.

**F5 XC 기능:** WAF (인코딩 정규화 깊이)

| 순서 | 스크립트 | 도구 | 설명 |
|---|---|---|---|
| 01 | `01-url-encoding.sh` | curl | 여러 엔드포인트 경로에 대한 SQLi, XSS, 경로 순회 페이로드의 단일, 이중, 삼중 URL 인코딩입니다. WAF가 URL 파라미터를 재귀적으로 디코딩하는지 테스트합니다. |
| 02 | `02-html-entity-encoding.sh` | curl, python3 | GET 파라미터와 POST 본문 모두에서 공격 페이로드의 10진수, 16진수, 명명된, 제로 패딩 HTML 엔티티 인코딩입니다. `&#123;&#123;7*7&#125;&#125;`을 통한 템플릿 인젝션을 포함합니다. |
| 03 | `03-unicode-utf8-evasion.sh` | curl | SQL 키워드와 XSS 태그에 제로 너비 공백(U+200B), BOM(U+FEFF), 전각 문자, 소프트 하이픈, ZWNJ 삽입입니다. 오버롱 UTF-8 시퀀스(2바이트 및 3바이트)를 테스트합니다. |
| 04 | `04-mixed-nested-encoding.sh` | curl | 다층 페이로드: URL 인코딩된 HTML 엔티티, 엔티티 내부의 이중 URL, 삼중 레이어 스택, 유니코드 이스케이프가 포함된 JSON 본문(`<`), 사용자의 특정 `{{7*7}}` 템플릿 인젝션 패턴입니다. |
| 05 | `05-null-byte-iis-base64.sh` | curl | 확장자 우회를 위한 널 바이트 인젝션(`%00`), IIS `%uXXXX` 유니코드 인코딩, 디코드 플래그가 포함된 URL 파라미터의 Base64 인코딩된 페이로드입니다. |
| 06 | `06-case-chunked-evasion.sh` | curl | 인코딩과 결합된 대소문자 랜덤화(`%3CsCrIpT%3E`), HTTP 청크 간에 SQL/XSS 키워드를 분할하는 청크드 전송 인코딩, Content-Length/Transfer-Encoding 충돌 프로브, 대체 경로 구분자(`%5c`, `%c0%af`)입니다. |
| 07 | `07-header-cookie-evasion.sh` | curl | Cookie, Referer, User-Agent, X-Forwarded-For, X-Original-URL 헤더를 통해 주입된 인코딩된 공격 페이로드입니다. WAF가 쿼리 파라미터와 POST 본문뿐만 아니라 모든 HTTP 헤더 필드를 검사하고 디코딩하는지 테스트합니다. |

**예상 F5 XC 동작:** 적절하게 구성된 WAF는 패턴 매칭 전에 모든 인코딩 레이어를 정규화해야 합니다. 단일 인코딩된 페이로드는 즉시 차단되어야 합니다. 이중 및 삼중 인코딩 테스트는 WAF 디코드 파이프라인의 깊이를 드러냅니다. 유니코드 삽입 테스트는 키워드 매칭 전에 보이지 않는 문자가 제거되는지를 확인합니다. 인코딩된 변형은 통과하지만 평문 등가물이 차단되는 결과는 WAF 정책의 정규화 갭을 나타냅니다.

## web-app-attacks

**스크립트:** 6개 | **예상 소요 시간:** 12-20분

F5 XC를 통한 Juice Shop과 DVWA에 대한 OWASP Top 10 웹 애플리케이션 취약점 테스트입니다. SQL 인젝션, XSS, 커맨드 인젝션, 경로 순회, Nikto 스캐닝, Nuclei 템플릿 기반 스캐닝을 포함합니다. Juice Shop과 DVWA 애플리케이션을 대상으로 합니다.

**F5 XC 기능:** WAF (웹 애플리케이션 방화벽)

| 순서 | 스크립트 | 도구 | 설명 |
|---|---|---|---|
| 01 | `01-sqli.sh` | sqlmap, curl | Juice Shop 검색 API와 DVWA SQLi 엔드포인트에 대한 SQL 인젝션 페이로드입니다. sqlmap 자동 스캔을 실행한 후 10개의 직접 curl 기반 인젝션 페이로드를 전송합니다. |
| 02 | `02-xss.sh` | dalfox, curl | Juice Shop과 DVWA XSS 반사/저장 엔드포인트에 대한 크로스사이트 스크립팅입니다. dalfox 자동 스캔을 실행한 후 스크립트 태그, 이벤트 핸들러, 쿠키 탈취를 포함한 12개의 직접 XSS 페이로드를 전송합니다. |
| 03 | `03-command-injection.sh` | curl | DVWA exec 엔드포인트에 대한 OS 커맨드 인젝션 페이로드입니다. 파이프 체인, 세미콜론, URL 인코딩된 개행, 리버스 셸 패턴을 포함한 12개의 페이로드를 전송합니다. |
| 04 | `04-path-traversal.sh` | curl | 여러 애플리케이션 엔드포인트에 대한 디렉토리 순회입니다. 6개 엔드포인트 경로에 대해 표준 `../` 순회, URL 인코딩 변형, 이중 인코딩 변형, 널 바이트 변형, 유니코드 변형을 테스트합니다. |
| 05 | `05-nikto-scan.sh` | nikto | 120초 시간 제한이 있는 전체 웹 서버 취약점 스캔입니다. WAF 시그니처 매칭을 트리거하는 다양한 HTTP 요청을 생성합니다. |
| 06 | `06-nuclei-scan.sh` | nuclei | 중간, 높음, 치명적 심각도 수준의 템플릿 기반 취약점 스캐닝입니다. 초당 50개 요청으로 속도 제한됩니다. |

**예상 F5 XC 동작:** WAF는 SQL 인젝션, XSS, 커맨드 인젝션, 경로 순회 페이로드가 포함된 요청을 차단하거나 플래그 처리해야 합니다. 보안 이벤트 대시보드에 각 차단된 요청에 대한 위반 카테고리, 시그니처 ID, 요청 세부 정보가 표시됩니다.
