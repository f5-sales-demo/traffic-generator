## 필수 항목

### Azure 구독

다음 리소스를 생성할 수 있는 Contributor 역할 또는 동등한 권한이 있는 활성 Azure 구독:

- 리소스 그룹
- 가상 네트워크 및 서브넷
- 네트워크 보안 그룹
- 공용 IP 주소
- 가상 머신 (기본값 Standard_F16s_v2)

### Azure CLI

구독 ID를 조회하기 위해 Azure CLI를 설치하고 인증합니다:

```bash
curl -sL https://aka.ms/InstallAzureCLIDeb | sudo bash

az login

az account show --query '{name:name, id:id}' -o table
```

`id` 값을 복사하세요 -- 이것이 Terraform에서 필요한 `subscription_id`입니다.

### Terraform

Terraform >= 1.5가 필요합니다:

```bash
terraform version
```

설치되어 있지 않은 경우 HashiCorp APT 리포지토리를 사용하세요:

```bash
wget -O- https://apt.releases.hashicorp.com/gpg | sudo gpg --dearmor -o /usr/share/keyrings/hashicorp-archive-keyring.gpg
echo "deb [signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] https://apt.releases.hashicorp.com $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/hashicorp.list
sudo apt update && sudo apt install terraform
```

### SSH 키 페어

VM 접속을 위한 SSH 키 페어가 필요합니다. Ed25519를 권장합니다:

```bash
ssh-keygen -t ed25519 -f ~/.ssh/traffic-generator-key -N ""
```

### 네트워크 접근

Traffic Generator VM은 프로비저닝 중 다음 소스에서 도구를 다운로드하기 위해 아웃바운드 HTTPS 접근이 필요합니다:

- GitHub Releases (nuclei, dalfox, ffuf, gobuster, feroxbuster, subfinder, httpx, amass)
- PyPI (sqlmap, scapy, impacket, arjun, pwntools, mitmproxy, sslyze, playwright)
- npm 레지스트리 (puppeteer, puppeteer-extra-plugin-stealth)
- Ubuntu APT 리포지토리 (nikto, nmap, masscan, hydra, tshark 및 기타 시스템 패키지)
- Git 리포지토리 (testssl.sh, recon-ng, spiderfoot, SecLists)

Azure 구독에서 아웃바운드 방화벽 규칙이나 NSG 제한이 적용되어 있는 경우, 초기 프로비저닝 기간(15-25분) 동안 이러한 소스로의 HTTPS(포트 443) 접근이 허용되어 있는지 확인하세요.

### 사전 배포 구성 요소

Traffic Generator를 배포하기 전에 다음이 준비되어 있어야 합니다:

1. **Origin Server**가 배포되어 접근 가능한 상태 ([origin-server 문서](https://f5xc-salesdemos.github.io/origin-server/03-deploy/) 참조)
2. Origin Server를 백엔드로 구성한 **F5 XC HTTP Load Balancer**
3. 로드 밸런서에 연결된 **WAF 정책** (WAF 트래픽 검증용)

`target_fqdn` Terraform 변수는 F5 XC 로드 밸런서의 공용 도메인으로 설정해야 합니다.

:::tip
랩을 사용하지 않을 때는 지속적인 비용 발생을 방지하기 위해 `terraform destroy`를 실행하세요. 절차는 [리소스 정리](../08-teardown/)를 참조하세요.
:::

:::note
Standard_F16s_v2 (32 GiB RAM)가 기본값인 이유는 스위트 실행 중 여러 도구가 동시에 실행되기 때문입니다. 헤드리스 Chromium을 사용하는 브라우저 자동화(Playwright, Puppeteer)가 주요 메모리 소비 요인입니다. 더 작은 F-시리즈 SKU는 브라우저를 사용하지 않는 스위트에서는 작동하지만, 봇 시뮬레이션을 실행할 때는 부족할 수 있습니다.
:::
