每个流量测试套件是 `/opt/traffic-generator/suites/` 下的一个目录，包含由 `runner.sh` 按顺序执行的编号 shell 脚本。每个脚本接受目标 FQDN 作为第一个参数，并将输出写入标准输出和结果目录。

## api-attacks

**脚本数量：** 4 | **预计持续时间：** 10-17 分钟

通过 F5 XC 对 VAmPI 进行 OWASP API 安全 Top 10 测试。注册测试用户、获取认证令牌，然后测试 BOLA、暴力破解认证、过度数据暴露、破损的功能级授权、SSRF 模式、针对 REST API 端点的 SQL 注入、隐藏参数发现和端点模糊测试。目标为 VAmPI 应用程序。

**F5 XC 功能：** API 安全 / API 发现

| 顺序 | 脚本 | 工具 | 描述 |
|---|---|---|---|
| 01 | `01-vampi-owasp-top10.sh` | curl, jq | 全面的 OWASP API Top 10 测试套件。注册测试用户、获取认证令牌，然后测试 BOLA（访问其他用户数据）、暴力破解认证、过度数据暴露、破损的功能级授权、SSRF 模式和安全配置错误探测。 |
| 02 | `02-sqlmap-api.sh` | sqlmap | 针对 VAmPI REST API 端点的 SQL 注入：用户查询、登录（POST）和注册（POST），使用 JSON 载荷。 |
| 03 | `03-arjun-param-discovery.sh` | arjun | 跨 Juice Shop、DVWA 和 VAmPI 端点的隐藏参数发现。发现可能接受注入载荷的未记录查询参数。 |
| 04 | `04-ffuf-api-fuzz.sh` | ffuf | 两阶段模糊测试：使用常见 API 路径字典（api、swagger、graphql、admin、.env、.git）跨所有应用程序前缀进行端点发现，然后对关键端点进行 HTTP 方法模糊测试（GET、POST、PUT、DELETE、PATCH、OPTIONS、TRACE、PROPFIND）。 |

**预期 F5 XC 行为：** API 发现应映射 VAmPI API 端点并识别异常请求模式。API 安全策略应标记未授权访问尝试、JSON 正文中的注入载荷和枚举模式。F5 XC 中的 API 清单将显示已发现的端点及请求计数。

## bot-simulation

**脚本数量：** 4 | **预计持续时间：** 5-10 分钟

通过 F5 XC 进行自动化浏览器活动和类机器人爬虫模式测试。包括无头 Chrome、带隐身插件的 Puppeteer、快速爬取和自动化表单交互。目标为所有应用程序。

**F5 XC 功能：** Bot 防御

| 顺序 | 脚本 | 工具 | 描述 |
|---|---|---|---|
| 01 | `01-headless-chrome.sh` | playwright | 无头 Chromium 浏览：加载目标、在应用程序路径间导航、提取页面标题和内容、截取屏幕截图。生成基于浏览器的流量，不使用隐身插件（可被检测为自动化）。 |
| 02 | `02-puppeteer-stealth.sh` | puppeteer, puppeteer-extra-plugin-stealth | 使用 puppeteer-extra 及隐身插件进行隐身浏览。尝试通过修补 navigator 属性、WebGL 和其他浏览器指纹来规避 Bot 检测。 |
| 03 | `03-rapid-crawl.sh` | curl, wget | 使用 curl 和 wget 快速连续发送高频 HTTP 请求。模拟爬虫行为，请求间无延迟并进行激进的 User-Agent 轮换。 |
| 04 | `04-form-interaction.sh` | playwright | 使用 Playwright 的表单交互 API 对 DVWA 登录、Juice Shop 注册和 VAmPI API 端点进行自动化表单填写和提交。 |

**预期 F5 XC 行为：** Bot 防御应基于浏览器指纹分析、请求时序和 JavaScript 挑战结果将流量分类为自动化流量。不带隐身功能的无头 Chrome 应被立即检测到。隐身 Puppeteer 测试高级规避技术是否能绕过检测。Bot 防御仪表板将显示 Bot 分类类别和缓解操作。

## demoapp-attacks

**脚本数量：** 2 | **预计持续时间：** 2-3 分钟

针对 F5 DemoApp 内置 WAF 测试端点（`/WAF/SQL`、`/WAF/XSS`、`/WAF/DIR`）的定向攻击载荷。向专为演示 WAF 拦截行为设计的端点直接发送 SQL 注入、XSS 和路径遍历载荷。当目标为 DemoApp 内容服务器而非标准源站应用程序时使用此套件。

**F5 XC 功能：** WAF（Web 应用程序防火墙）

| 顺序 | 脚本 | 工具 | 描述 |
|---|---|---|---|

| 02 | `02-xss-waf-endpoint.sh` | curl, python3 | 针对 `/WAF/XSS?update=` 的 18 个 XSS 载荷，包括 script 标签、事件处理器（onerror、onload、ontoggle、onfocus）、base64 eval、style expression、多态载荷和 cookie 窃取。 |

**预期 F5 XC 行为：** WAF 应阻止包含 SQL 注入和 XSS 载荷的请求并返回拦截页面。安全事件仪表板将显示每个被阻止请求的违规类别和签名匹配。

## waf-encoding-evasion

**脚本数量：** 7 | **预计持续时间：** 5-10 分钟

多层编码规避攻击，旨在测试 WAF 是否在检查前正确规范化和解码载荷。涵盖单层、双层和三层 URL 编码；HTML 实体编码（十进制、十六进制、命名、零填充）；Unicode/UTF-8 插入（零宽空格、BOM、全角字符、软连字符、超长序列）；混合/嵌套多层编码（URL 编码的 HTML 实体、实体内的双重 URL 编码、三层堆叠）；空字节注入；IIS %uXXXX 编码；大小写变换；分块传输编码正文分割；Base64 参数规避；以及头部/Cookie 编码载荷。目标为任意应用程序。

**F5 XC 功能：** WAF（编码规范化深度）

| 顺序 | 脚本 | 工具 | 描述 |
|---|---|---|---|
| 01 | `01-url-encoding.sh` | curl | 跨多个端点路径对 SQLi、XSS 和路径遍历载荷进行单层、双层和三层 URL 编码。测试 WAF 是否递归解码 URL 参数。 |
| 02 | `02-html-entity-encoding.sh` | curl, python3 | 在 GET 参数和 POST 正文中对攻击载荷进行十进制、十六进制、命名和零填充 HTML 实体编码。包括通过 `&#123;&#123;7*7&#125;&#125;` 的模板注入。 |
| 03 | `03-unicode-utf8-evasion.sh` | curl | 在 SQL 关键字和 XSS 标签中插入零宽空格（U+200B）、BOM（U+FEFF）、全角字符、软连字符和 ZWNJ。测试超长 UTF-8 序列（2 字节和 3 字节）。 |
| 04 | `04-mixed-nested-encoding.sh` | curl | 多层载荷：URL 编码的 HTML 实体、实体内的双重 URL 编码、三层堆叠、带 Unicode 转义的 JSON 正文（`<`）以及用户指定的 `{{7*7}}` 模板注入模式。 |
| 05 | `05-null-byte-iis-base64.sh` | curl | 用于扩展名绕过的空字节注入（`%00`）、IIS `%uXXXX` Unicode 编码和 URL 参数中带解码标志的 Base64 编码载荷。 |
| 06 | `06-case-chunked-evasion.sh` | curl | 大小写随机化结合编码（`%3CsCrIpT%3E`）、分块传输编码将 SQL/XSS 关键字拆分到不同 HTTP 块、Content-Length/Transfer-Encoding 冲突探测以及替代路径分隔符（`%5c`、`%c0%af`）。 |
| 07 | `07-header-cookie-evasion.sh` | curl | 通过 Cookie、Referer、User-Agent、X-Forwarded-For 和 X-Original-URL 头部注入编码攻击载荷。测试 WAF 是否检查和解码所有 HTTP 头部字段，而不仅仅是查询参数和 POST 正文。 |

**预期 F5 XC 行为：** 正确配置的 WAF 应在模式匹配前规范化所有编码层。单层编码的载荷应被立即阻止。双层和三层编码测试揭示 WAF 解码管道的深度。Unicode 插入测试揭示在关键字匹配前是否剥离了不可见字符。编码变体通过而纯文本等效载荷被阻止的结果表明 WAF 策略中存在规范化差距。

## web-app-attacks

**脚本数量：** 6 | **预计持续时间：** 12-20 分钟

通过 F5 XC 对 Juice Shop 和 DVWA 进行 OWASP Top 10 Web 应用漏洞测试。包括 SQL 注入、XSS、命令注入、路径遍历、Nikto 扫描和基于 Nuclei 模板的扫描。目标为 Juice Shop 和 DVWA 应用程序。

**F5 XC 功能：** WAF（Web 应用程序防火墙）

| 顺序 | 脚本 | 工具 | 描述 |
|---|---|---|---|
| 01 | `01-sqli.sh` | sqlmap, curl | 针对 Juice Shop 搜索 API 和 DVWA SQLi 端点的 SQL 注入载荷。运行 sqlmap 自动扫描，然后发送 10 个基于 curl 的直接注入载荷。 |
| 02 | `02-xss.sh` | dalfox, curl | 针对 Juice Shop 和 DVWA XSS 反射型/存储型端点的跨站脚本攻击。运行 dalfox 自动扫描，然后发送 12 个直接 XSS 载荷，包括 script 标签、事件处理器和 cookie 窃取。 |
| 03 | `03-command-injection.sh` | curl | 针对 DVWA exec 端点的操作系统命令注入载荷。发送 12 个载荷，包括管道链、分号、URL 编码换行符和反弹 shell 模式。 |
| 04 | `04-path-traversal.sh` | curl | 针对多个应用程序端点的目录遍历。跨 6 个端点路径测试标准 `../` 遍历、URL 编码变体、双重编码变体、空字节变体和 Unicode 变体。 |
| 05 | `05-nikto-scan.sh` | nikto | 带 120 秒时间限制的完整 Web 服务器漏洞扫描。生成触发 WAF 签名匹配的多样化 HTTP 请求。 |
| 06 | `06-nuclei-scan.sh` | nuclei | 基于模板的漏洞扫描，覆盖中等、高和严重级别。速率限制为每秒 50 个请求。 |

**预期 F5 XC 行为：** WAF 应阻止或标记包含 SQL 注入、XSS、命令注入和路径遍历载荷的请求。安全事件仪表板将显示每个被阻止请求的违规类别、签名 ID 和请求详情。
