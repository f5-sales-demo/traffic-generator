所有 Terraform 文件位于 [`terraform/`](https://github.com/f5xc-salesdemos/traffic-generator/tree/main/terraform) 目录中。克隆仓库并直接部署：

```bash
git clone https://github.com/f5xc-salesdemos/traffic-generator.git
cd traffic-generator/terraform
cp terraform.tfvars.example terraform.tfvars
# 编辑 terraform.tfvars，填入您的 Azure 订阅 ID 和目标 FQDN
```

## Terraform 配置

### Terraform 文件结构

terraform 目录包含 9 个文件，遵循演示资源标准：

- `versions.tf` -- Terraform 和 provider 版本约束（azurerm ~> 4.0, azuread ~> 3.0）
- `providers.tf` -- Azure RM 和 Azure AD provider 配置
- `data.tf` -- 用于部署者自动解析的 Azure AD 数据源
- `locals.tf` -- 部署者解析、Azure 云采用框架资源命名、标准标签
- `main.tf` -- 资源组（命名为 `rg-traffic-generator-{environment}-{deployer}`）
- `variables.tf` -- 所有输入变量（2 个必需，9 个可选），通过 Azure AD 实现部署者自动解析
- `network.tf` -- VNet (10.201.0.0/16)、子网、公共 IP、NSG（端口 22 SSH）、NIC
- `vm.tf` -- Ubuntu 24.04 VM，通过 templatefile() 使用 cloud-init
- `outputs.tf` -- 17 个输出：15 个所有演示资源共享的标准输出（deployer、public_ip、private_ip、ssh_command、resource_group_name、vm_name、nsg_name、vnet_name、subnet_id、component、environment、resource_group_id、vm_id、nsg_id、location）加上 2 个组件特定输出（target_fqdn、status_check）

### Provider 和变量

`main.tf` 配置 Azure provider：

```hcl file=./_data/main.tf
```

`variables.tf` 定义所有可配置参数。`vm_size` 默认值为 `Standard_F16s_v2`（16 vCPU 计算优化）-- 参见 [VM 规格选择](#vm-规格选择) 了解扩展指南：

```hcl file=./_data/variables.tf
```

### 网络基础设施

`network.tf` 创建 VNet、子网、公共 IP、NSG（端口 22 SSH）和 NIC：

```hcl file=./_data/network.tf
```

### 带 Cloud-Init 的虚拟机

`vm.tf` 创建 Ubuntu 24.04 VM，并传递包含目标和工具变量的 cloud-init 模板：

```hcl file=./_data/vm.tf
```

### Cloud-Init 配置

`cloud-init.yaml` 是经过生产验证的配置，已在 48+ 小时的持续负载测试（包括 4 轮源站压力测试）中验证通过。它使用带有重试逻辑的辅助库，将进度日志写入 `/var/log/cloud-init-progress.log`，包含 9 个配置阶段，以及条件状态报告（在 `status.json` 中报告 ready/degraded）。它配置以下内容：

- **内核调优** -- `somaxconn=131072`、`tcp_max_tw_buckets=4000000`、`tcp_fin_timeout=5`、64 MiB 套接字缓冲区、`file-max=4194304`
- **NIC 优化** -- 跨所有核心的 RPS/RFS、通过 ethtool 最大化环形缓冲区、THP=always
- **APT 软件包** -- nikto、nmap、masscan、sqlmap、hydra、medusa、ncrack、tshark、hping3、tcpdump、netcat、ngrep、iperf3、mtr、sslscan、socat、dirb、whatweb、wrk、hey、vegeta、ethtool
- **GitHub Releases** -- nuclei、dalfox、ffuf、gobuster、feroxbuster、subfinder、httpx、amass
- **Python 软件包** -- mitmproxy、sslyze、dnsrecon、fierce、theHarvester、playwright、scapy、impacket、arjun
- **Node.js 20.x** -- 带有 stealth 插件的 Puppeteer，用于机器人模拟
- **浏览器自动化** -- Playwright Chromium，用于无头浏览器测试
- **Git 克隆** -- testssl.sh、recon-ng、spiderfoot、SecLists、waf-bypass、PayloadsAllTheThings
- **套件部署** -- 从本仓库克隆 19 个流量套件到 `/opt/traffic-generator/suites/`
- **配置文件** -- 包含 `TARGET_FQDN`、`TARGET_ORIGIN_IP` 和 `CRAPI_PORT` 的 `config.env`

cloud-init 使用 Terraform 模板变量：`${target_fqdn}`、`${target_origin_ip}` 和 `${tool_tier}`。Shell 变量如 `$NPROC` 在 Terraform templatefile 中转义为 `$$NPROC`。

```yaml file=./_data/cloud-init.yaml
```

### 输出

`outputs.tf` 暴露 17 个输出，包括公共/私有 IP、SSH 命令、资源标识符和组件特定值（target_fqdn、status_check）：

```hcl file=./_data/outputs.tf
```

### 变量文件示例

将 `terraform.tfvars.example` 复制为 `terraform.tfvars` 并填入您的值。`.gitignore` 已排除 `terraform.tfvars` 以防止提交凭据：

```hcl file=./_data/terraform.tfvars.example
```

将 `subscription_id` 替换为您的 Azure 订阅 ID（通过 `az account show --query id -o tsv` 获取），将 `target_fqdn` 替换为您的 F5 XC 负载均衡器域名。

## 部署

```bash
terraform init

terraform plan

terraform apply
```

部署成功后，Terraform 会输出公共 IP、SSH 命令和目标 FQDN。

## VM 规格选择

推荐使用 F 系列计算优化 VM 来运行此 CPU 密集型流量生成工作负载。cloud-init 内核调优和工具并发度会随 vCPU 数量自动扩展。

| VM SKU | vCPU | 内存 | 网络 | 使用场景 |
|---|---|---|---|---|
| Standard_F4s_v2 | 4 | 8 GiB | 10 Gbps | 轻量实验/演示 |
| Standard_F8s_v2 | 8 | 16 GiB | 12.5 Gbps | 标准演示 |
| Standard_F16s_v2 | 16 | 32 GiB | 12.5 Gbps | 负载测试（默认） |
| Standard_F32s_v2 | 32 | 64 GiB | 16 Gbps | 重度基准测试 |
| Standard_D16s_v3 | 16 | 64 GiB | 12.5 Gbps | 内存密集型工作负载 |

:::tip
通过对 CDN F16s_v2 和 F32s_v2 目标进行 A/B/C/D 基准测试，F16s_v2 被验证为最佳性价比选项。查看 [benchmarks/](https://github.com/f5xc-salesdemos/traffic-generator/tree/main/benchmarks) 获取完整结果。
:::

## 部署后验证

Cloud-init 配置过程**标准层级需要 15-20 分钟**，**完整层级约需 25 分钟**。VM 可以立即通过 SSH 访问，但工具在 cloud-init 完成之前仍在安装中。

### 等待 Cloud-Init 完成

通过 SSH 连接到 VM 并等待配置完成：

```bash
ssh azureuser@$(terraform output -raw public_ip)

cloud-init status --wait
```

完成时的预期输出：

```
status: done
```

### 检查状态文件

cloud-init 脚本在完成后会写入状态文件：

```bash
cat /opt/traffic-generator/status.json
```

预期输出：

```json
{
  "status": "ready",
  "tier": "standard",
  "target_fqdn": "demo.example.com",
  "completed": "2026-04-25T14:30:00Z",
  "tools_installed": true
}
```

### 验证关键工具

运行验证循环以确认每个关键工具在 PATH 中可用：

```bash
for tool in nikto sqlmap nuclei dalfox ffuf gobuster feroxbuster nmap masscan \
  hydra medusa ncrack sslscan tshark hping3 mitmproxy sslyze subfinder httpx \
  amass arjun mtr whatweb dirb socat netcat ngrep iperf3 whois dig wrk hey; do
  if command -v "$tool" > /dev/null 2>&1; then
    echo "OK: $tool"
  else
    echo "MISSING: $tool"
  fi
done
```

### 验证套件运行器

确认运行器和套件已安装：

```bash
ls /opt/traffic-generator/suites/runner.sh
ls /opt/traffic-generator/suites/*/

/opt/traffic-generator/suites/runner.sh web-app-attacks --dry-run
```

## 从上游组件接入

流量生成器以 F5 XC HTTP 负载均衡器为目标，该负载均衡器位于源服务器前端。请先部署源服务器，配置 F5 XC，然后将 `target_fqdn` 设置为负载均衡器的 FQDN：

```bash
# 部署 origin-server 并创建 F5 XC HTTP 负载均衡器后：
cat > terraform.tfvars <<EOF
subscription_id  = "your-subscription-id"
target_fqdn      = "your-xc-load-balancer.example.com"
target_origin_ip = "$(cd ../../origin-server/terraform && terraform output -raw public_ip)"
EOF
```

| 必需变量 | 来源 | 描述 |
|---|---|---|
| `target_fqdn` | F5 XC 控制台 | 保护源服务器的 HTTP 负载均衡器的 FQDN |
| `target_origin_ip` | 源服务器 `public_ip` 输出 | 用于绕过测试的直接源站 IP（可选） |

继续查看[工具目录](../04-tool-catalog/)获取完整工具参考，或查看[套件](../05-suites/)开始生成流量。
