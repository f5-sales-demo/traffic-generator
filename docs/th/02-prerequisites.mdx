## สิ่งที่จำเป็น

### Azure Subscription

Azure subscription ที่ใช้งานอยู่พร้อมสิทธิ์ Contributor role หรือสิทธิ์เทียบเท่าเพื่อสร้าง:

- Resource groups
- Virtual networks และ subnets
- Network security groups
- Public IP addresses
- Virtual machines (ค่าเริ่มต้น Standard_F16s_v2)

### Azure CLI

ติดตั้งและยืนยันตัวตนกับ Azure CLI เพื่อดึง subscription ID ของคุณ:

```bash
curl -sL https://aka.ms/InstallAzureCLIDeb | sudo bash

az login

az account show --query '{name:name, id:id}' -o table
```

คัดลอกค่า `id` -- นี่คือ `subscription_id` ที่ Terraform ต้องการ

### Terraform

ต้องใช้ Terraform >= 1.5:

```bash
terraform version
```

หากยังไม่ได้ติดตั้ง ให้ใช้ HashiCorp APT repository:

```bash
wget -O- https://apt.releases.hashicorp.com/gpg | sudo gpg --dearmor -o /usr/share/keyrings/hashicorp-archive-keyring.gpg
echo "deb [signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] https://apt.releases.hashicorp.com $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/hashicorp.list
sudo apt update && sudo apt install terraform
```

### คู่คีย์ SSH

คู่คีย์ SSH สำหรับการเข้าถึง VM แนะนำ Ed25519:

```bash
ssh-keygen -t ed25519 -f ~/.ssh/traffic-generator-key -N ""
```

### การเข้าถึงเครือข่าย

Traffic Generator VM ต้องการการเข้าถึง outbound HTTPS ระหว่างการ provisioning เพื่อดาวน์โหลดเครื่องมือจาก:

- GitHub Releases (nuclei, dalfox, ffuf, gobuster, feroxbuster, subfinder, httpx, amass)
- PyPI (sqlmap, scapy, impacket, arjun, pwntools, mitmproxy, sslyze, playwright)
- npm registry (puppeteer, puppeteer-extra-plugin-stealth)
- Ubuntu APT repositories (nikto, nmap, masscan, hydra, tshark และแพ็คเกจระบบอื่น ๆ)
- Git repositories (testssl.sh, recon-ng, spiderfoot, SecLists)

หาก Azure subscription ของคุณบังคับใช้กฎ outbound firewall หรือข้อจำกัด NSG ให้ตรวจสอบว่า HTTPS (พอร์ต 443) ไปยังแหล่งเหล่านี้ได้รับอนุญาตระหว่างช่วงเวลา provisioning เริ่มต้น (15-25 นาที)

### คอมโพเนนต์ที่ต้องติดตั้งไว้ล่วงหน้า

ก่อนที่จะติดตั้ง Traffic Generator คุณควรมี:

1. **Origin Server** ที่ติดตั้งและเข้าถึงได้แล้ว (ดู [เอกสาร origin-server](https://f5xc-salesdemos.github.io/origin-server/03-deploy/))
2. **F5 XC HTTP Load Balancer** ที่กำหนดค่าโดยมี origin server เป็น backend
3. **WAF policy** ที่แนบกับ load balancer (สำหรับการตรวจสอบ WAF traffic)

ตัวแปร Terraform `target_fqdn` ควรตั้งค่าเป็นโดเมนสาธารณะของ F5 XC load balancer

:::tip
ใช้ `terraform destroy` เมื่อไม่ได้ใช้งานแล็บเพื่อหลีกเลี่ยงค่าใช้จ่ายที่เกิดขึ้นต่อเนื่อง ดู [การรื้อถอน](../08-teardown/) สำหรับขั้นตอน
:::

:::note
Standard_F16s_v2 (32 GiB RAM) เป็นค่าเริ่มต้นเนื่องจากเครื่องมือหลายตัวทำงานพร้อมกันระหว่างการดำเนินการชุดทดสอบ Browser automation (Playwright, Puppeteer) กับ headless Chromium เป็นตัวใช้หน่วยความจำหลัก SKU ขนาดเล็กกว่าในกลุ่ม F-series จะใช้ได้กับชุดทดสอบที่ไม่ใช้เบราว์เซอร์ แต่อาจไม่เพียงพอเมื่อรัน bot-simulation
:::
