ไฟล์ Terraform ทั้งหมดอยู่ในไดเรกทอรี [`terraform/`](https://github.com/f5xc-salesdemos/traffic-generator/tree/main/terraform) โคลนรีโพซิทอรีและปรับใช้งานได้โดยตรง:

```bash
git clone https://github.com/f5xc-salesdemos/traffic-generator.git
cd traffic-generator/terraform
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars with your Azure subscription ID and target FQDN
```

## การกำหนดค่า Terraform

### โครงสร้างไฟล์ Terraform

ไดเรกทอรี terraform ประกอบด้วย 9 ไฟล์ตามมาตรฐานทรัพยากรสาธิต:

- `versions.tf` -- ข้อจำกัดเวอร์ชัน Terraform และ provider (azurerm ~> 4.0, azuread ~> 3.0)
- `providers.tf` -- การกำหนดค่า Azure RM และ Azure AD provider
- `data.tf` -- แหล่งข้อมูล Azure AD สำหรับการระบุตัวตนผู้ปรับใช้งานอัตโนมัติ
- `locals.tf` -- การระบุตัวตนผู้ปรับใช้งาน, การตั้งชื่อทรัพยากรตาม Azure Cloud Adoption Framework, แท็กมาตรฐาน
- `main.tf` -- กลุ่มทรัพยากร (ชื่อ `rg-traffic-generator-{environment}-{deployer}`)
- `variables.tf` -- ตัวแปรอินพุตทั้งหมด (จำเป็น 2 รายการ, เสริม 9 รายการ) พร้อมการระบุตัวตนผู้ปรับใช้งานอัตโนมัติผ่าน Azure AD
- `network.tf` -- VNet (10.201.0.0/16), ซับเน็ต, IP สาธารณะ, NSG (พอร์ต 22 SSH), NIC
- `vm.tf` -- VM Ubuntu 24.04 พร้อม cloud-init ผ่าน templatefile()
- `outputs.tf` -- เอาต์พุต 17 รายการ: เอาต์พุตมาตรฐาน 15 รายการที่ใช้ร่วมกันในทรัพยากรสาธิตทั้งหมด (deployer, public_ip, private_ip, ssh_command, resource_group_name, vm_name, nsg_name, vnet_name, subnet_id, component, environment, resource_group_id, vm_id, nsg_id, location) บวกเอาต์พุตเฉพาะคอมโพเนนต์ 2 รายการ (target_fqdn, status_check)

### Provider และตัวแปร

`main.tf` กำหนดค่า Azure provider:

```hcl file=./_data/main.tf
```

`variables.tf` กำหนดพารามิเตอร์ที่ปรับแต่งได้ทั้งหมด ค่าเริ่มต้นของ `vm_size` คือ `Standard_F16s_v2` (16 vCPU ปรับแต่งสำหรับการประมวลผล) -- ดู [การเลือกขนาด VM](#การเลือกขนาด-vm) สำหรับคำแนะนำในการปรับขนาด:

```hcl file=./_data/variables.tf
```

### โครงสร้างพื้นฐานเครือข่าย

`network.tf` สร้าง VNet, ซับเน็ต, IP สาธารณะ, NSG (พอร์ต 22 SSH), และ NIC:

```hcl file=./_data/network.tf
```

### เครื่องเสมือนพร้อม Cloud-Init

`vm.tf` สร้าง VM Ubuntu 24.04 และส่งเทมเพลต cloud-init พร้อมตัวแปรเป้าหมายและเครื่องมือ:

```hcl file=./_data/vm.tf
```

### การจัดเตรียม Cloud-Init

`cloud-init.yaml` เป็นการกำหนดค่าที่ผ่านการตรวจสอบในสภาพแวดล้อมจริง ซึ่งได้รับการยืนยันภายใต้การทดสอบโหลดต่อเนื่องมากกว่า 48 ชั่วโมง รวมถึงการทดสอบ origin torture 4 รอบ ใช้ไลบรารีตัวช่วยพร้อมตรรกะการลองใหม่, การบันทึกความคืบหน้าไปยัง `/var/log/cloud-init-progress.log`, 9 ขั้นตอนการจัดเตรียม, และการรายงานสถานะแบบมีเงื่อนไข (ready/degraded ใน `status.json`) โดยจัดเตรียม:

- **การปรับจูน Kernel** -- `somaxconn=131072`, `tcp_max_tw_buckets=4000000`, `tcp_fin_timeout=5`, บัฟเฟอร์ซ็อกเก็ต 64 MiB, `file-max=4194304`
- **การปรับแต่ง NIC** -- RPS/RFS ข้ามทุกคอร์, เพิ่มบัฟเฟอร์ ring ให้สูงสุดผ่าน ethtool, THP=always
- **แพ็กเกจ APT** -- nikto, nmap, masscan, sqlmap, hydra, medusa, ncrack, tshark, hping3, tcpdump, netcat, ngrep, iperf3, mtr, sslscan, socat, dirb, whatweb, wrk, hey, vegeta, ethtool
- **GitHub Releases** -- nuclei, dalfox, ffuf, gobuster, feroxbuster, subfinder, httpx, amass
- **แพ็กเกจ Python** -- mitmproxy, sslyze, dnsrecon, fierce, theHarvester, playwright, scapy, impacket, arjun
- **Node.js 20.x** -- Puppeteer พร้อมปลั๊กอิน stealth สำหรับจำลองบอท
- **ระบบอัตโนมัติเบราว์เซอร์** -- Playwright Chromium สำหรับทดสอบเบราว์เซอร์แบบ headless
- **Git clones** -- testssl.sh, recon-ng, spiderfoot, SecLists, waf-bypass, PayloadsAllTheThings
- **การปรับใช้ชุดทราฟฟิก** -- ชุดทราฟฟิก 19 ชุดโคลนจากรีโพซิทอรีนี้ไปยัง `/opt/traffic-generator/suites/`
- **ไฟล์การกำหนดค่า** -- `config.env` พร้อม `TARGET_FQDN`, `TARGET_ORIGIN_IP`, และ `CRAPI_PORT`

cloud-init ใช้ตัวแปรเทมเพลต Terraform: `${target_fqdn}`, `${target_origin_ip}`, และ `${tool_tier}` ตัวแปร Shell เช่น `$NPROC` จะถูก escape เป็น `$$NPROC` ใน Terraform templatefile

```yaml file=./_data/cloud-init.yaml
```

### เอาต์พุต

`outputs.tf` เปิดเผยเอาต์พุต 17 รายการ รวมถึง IP สาธารณะ/ส่วนตัว, คำสั่ง SSH, ตัวระบุทรัพยากร, และค่าเฉพาะคอมโพเนนต์ (target_fqdn, status_check):

```hcl file=./_data/outputs.tf
```

### ตัวอย่างไฟล์ตัวแปร

คัดลอก `terraform.tfvars.example` ไปเป็น `terraform.tfvars` และกรอกค่าของคุณ `.gitignore` จะยกเว้น `terraform.tfvars` เพื่อป้องกันการ commit ข้อมูลรับรอง:

```hcl file=./_data/terraform.tfvars.example
```

แทนที่ `subscription_id` ด้วย Azure subscription ID ของคุณ (จาก `az account show --query id -o tsv`) และ `target_fqdn` ด้วยโดเมนโหลดบาลานเซอร์ F5 XC ของคุณ

## การปรับใช้งาน

```bash
terraform init

terraform plan

terraform apply
```

Terraform จะแสดงผล IP สาธารณะ, คำสั่ง SSH, และ FQDN เป้าหมายหลังจากปรับใช้งานสำเร็จ

## การเลือกขนาด VM

VM ซีรีส์ F ที่ปรับแต่งสำหรับการประมวลผลได้รับการแนะนำสำหรับปริมาณงานสร้างทราฟฟิกที่ใช้ CPU เป็นหลัก การปรับจูน kernel ของ cloud-init และการทำงานพร้อมกันของเครื่องมือจะปรับขนาดอัตโนมัติตามจำนวน vCPU

| VM SKU | vCPU | RAM | เครือข่าย | กรณีการใช้งาน |
|---|---|---|---|---|
| Standard_F4s_v2 | 4 | 8 GiB | 10 Gbps | แล็บ/สาธิตขนาดเล็ก |
| Standard_F8s_v2 | 8 | 16 GiB | 12.5 Gbps | การสาธิตมาตรฐาน |
| Standard_F16s_v2 | 16 | 32 GiB | 12.5 Gbps | การทดสอบโหลด (ค่าเริ่มต้น) |
| Standard_F32s_v2 | 32 | 64 GiB | 16 Gbps | การเบนช์มาร์กหนัก |
| Standard_D16s_v3 | 16 | 64 GiB | 12.5 Gbps | ปริมาณงานที่ใช้ RAM มาก |

:::tip
F16s_v2 ได้รับการตรวจสอบว่าเป็นตัวเลือกที่คุ้มค่าที่สุดจากการทดสอบเบนช์มาร์ก A/B/C/D เทียบกับเป้าหมาย CDN F16s_v2 และ F32s_v2 ดู [benchmarks/](https://github.com/f5xc-salesdemos/traffic-generator/tree/main/benchmarks) สำหรับผลลัพธ์ทั้งหมด
:::

## การตรวจสอบหลังปรับใช้งาน

การจัดเตรียม cloud-init ใช้เวลา **15-20 นาทีสำหรับระดับ standard** และ **ประมาณ 25 นาทีสำหรับระดับ full** VM สามารถเข้าถึงผ่าน SSH ได้ทันที แต่เครื่องมือยังอยู่ระหว่างการติดตั้งจนกว่า cloud-init จะเสร็จสมบูรณ์

### รอ Cloud-Init

SSH เข้าสู่ VM และรอให้การจัดเตรียมเสร็จสิ้น:

```bash
ssh azureuser@$(terraform output -raw public_ip)

cloud-init status --wait
```

ผลลัพธ์ที่คาดหวังเมื่อเสร็จสมบูรณ์:

```
status: done
```

### ตรวจสอบไฟล์สถานะ

สคริปต์ cloud-init จะเขียนไฟล์สถานะเมื่อเสร็จสมบูรณ์:

```bash
cat /opt/traffic-generator/status.json
```

ผลลัพธ์ที่คาดหวัง:

```json
{
  "status": "ready",
  "tier": "standard",
  "target_fqdn": "demo.example.com",
  "completed": "2026-04-25T14:30:00Z",
  "tools_installed": true
}
```

### ตรวจสอบเครื่องมือสำคัญ

เรียกใช้ลูปตรวจสอบเพื่อยืนยันว่าเครื่องมือสำคัญแต่ละตัวพร้อมใช้งานบน PATH:

```bash
for tool in nikto sqlmap nuclei dalfox ffuf gobuster feroxbuster nmap masscan \
  hydra medusa ncrack sslscan tshark hping3 mitmproxy sslyze subfinder httpx \
  amass arjun mtr whatweb dirb socat netcat ngrep iperf3 whois dig wrk hey; do
  if command -v "$tool" > /dev/null 2>&1; then
    echo "OK: $tool"
  else
    echo "MISSING: $tool"
  fi
done
```

### ตรวจสอบตัวรันชุดทราฟฟิก

ยืนยันว่าตัวรันและชุดทราฟฟิกถูกติดตั้งแล้ว:

```bash
ls /opt/traffic-generator/suites/runner.sh
ls /opt/traffic-generator/suites/*/

/opt/traffic-generator/suites/runner.sh web-app-attacks --dry-run
```

## เอาต์พุต Terraform

| เอาต์พุต | คำอธิบาย | ตัวอย่าง |
|---|---|---|
| `deployer` | ตัวระบุผู้ปรับใช้งานที่ถูกระบุ | `jsmith` |
| `public_ip` | ที่อยู่ IP สาธารณะของ VM สร้างทราฟฟิก | `20.12.78.200` |
| `private_ip` | ที่อยู่ IP ส่วนตัวของ VM | `10.201.0.4` |
| `ssh_command` | คำสั่ง SSH สำหรับเชื่อมต่อกับ VM | `ssh azureuser@20.12.78.200` |
| `resource_group_name` | ชื่อกลุ่มทรัพยากร | `rg-traffic-generator-dev-jsmith` |
| `resource_group_id` | Azure resource ID ของกลุ่มทรัพยากร | `/subscriptions/.../resourceGroups/rg-traffic-generator-dev-jsmith` |
| `vm_name` | ชื่อเครื่องเสมือน | `vm-traffic-generator-dev-jsmith` |
| `vm_id` | Azure resource ID ของเครื่องเสมือน | `/subscriptions/.../virtualMachines/vm-traffic-generator-dev-jsmith` |
| `nsg_name` | ชื่อกลุ่มความปลอดภัยเครือข่าย | `nsg-traffic-generator-dev-jsmith` |
| `nsg_id` | Azure resource ID ของกลุ่มความปลอดภัยเครือข่าย | `/subscriptions/.../networkSecurityGroups/nsg-traffic-generator-dev-jsmith` |
| `vnet_name` | ชื่อเครือข่ายเสมือน | `vnet-traffic-generator-dev-jsmith` |
| `subnet_id` | Azure resource ID ของซับเน็ต | `/subscriptions/.../subnets/snet-traffic-generator-dev-jsmith` |
| `component` | ตัวระบุคอมโพเนนต์ | `traffic-generator` |
| `environment` | สภาพแวดล้อมการปรับใช้งาน | `dev` |
| `location` | ภูมิภาค Azure | `eastus2` |
| `target_fqdn` | FQDN เป้าหมายที่กำหนดค่าสำหรับชุดทราฟฟิก | `demo.example.com` |
| `status_check` | คำสั่งสำหรับตรวจสอบสถานะการจัดเตรียม | `ssh azureuser@20.12.78.200 cat /opt/traffic-generator/status.json` |

ดึงเอาต์พุตใดๆ:

```bash
terraform output -raw public_ip
terraform output -raw ssh_command
```

## การเชื่อมต่อจากคอมโพเนนต์ต้นทาง

ตัวสร้างทราฟฟิกกำหนดเป้าหมายไปยัง F5 XC HTTP โหลดบาลานเซอร์ที่อยู่หน้าเซิร์ฟเวอร์ origin ปรับใช้งานเซิร์ฟเวอร์ origin ก่อน กำหนดค่า F5 XC จากนั้นตั้งค่า `target_fqdn` เป็น FQDN ของโหลดบาลานเซอร์:

```bash
# After deploying origin-server and creating an F5 XC HTTP load balancer:
cat > terraform.tfvars <<EOF
subscription_id  = "your-subscription-id"
target_fqdn      = "your-xc-load-balancer.example.com"
target_origin_ip = "$(cd ../../origin-server/terraform && terraform output -raw public_ip)"
EOF
```

| ตัวแปรที่จำเป็น | แหล่งที่มา | คำอธิบาย |
|---|---|---|
| `target_fqdn` | คอนโซล F5 XC | FQDN ของ HTTP โหลดบาลานเซอร์ที่ป้องกันเซิร์ฟเวอร์ origin |
| `target_origin_ip` | เอาต์พุต `public_ip` ของเซิร์ฟเวอร์ origin | IP ของ origin โดยตรงสำหรับทดสอบการบายพาส (ไม่บังคับ) |

ดำเนินการต่อที่ [แคตตาล็อกเครื่องมือ](../04-tool-catalog/) สำหรับข้อมูลอ้างอิงเครื่องมือฉบับสมบูรณ์ หรือ [ชุดทราฟฟิก](../05-suites/) เพื่อเริ่มสร้างทราฟฟิก
