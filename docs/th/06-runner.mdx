## ภาพรวม

`runner.sh` คือตัวจัดการชุดสคริปต์ที่ทำหน้าที่เรียกใช้สคริปต์ทั้งหมดที่มีหมายเลขลำดับในไดเรกทอรีของชุดสคริปต์ตามลำดับ บันทึกผลลัพธ์ และจัดเก็บเมตาดาต้าของผลลัพธ์ ติดตั้งอยู่ที่ `/opt/traffic-generator/suites/runner.sh` บน VM

## การใช้งาน

```bash
runner.sh <suite-name> [--dry-run]
```

**อาร์กิวเมนต์:**

| อาร์กิวเมนต์ | จำเป็น | คำอธิบาย |
|---|---|---|
| `suite-name` | ใช่ | ชื่อไดเรกทอรีของชุดสคริปต์ (เช่น `web-app-attacks`, `api-attacks`) |
| `--dry-run` | ไม่ | แสดงรายการที่จะถูกเรียกใช้โดยไม่ได้เรียกใช้สคริปต์ใด ๆ จริง |

**ตัวอย่าง:**

```bash
# เรียกใช้ชุดสคริปต์ web-app-attacks
/opt/traffic-generator/suites/runner.sh web-app-attacks

# Dry-run เพื่อดูตัวอย่างสคริปต์ของ api-attacks
/opt/traffic-generator/suites/runner.sh api-attacks --dry-run

# เรียกใช้การจำลอง bot
/opt/traffic-generator/suites/runner.sh bot-simulation
```

## การตั้งค่า

### config.env

Runner จะอ่านการตั้งค่าจาก `/opt/traffic-generator/config.env`:

```bash
# จำเป็น: FQDN เป้าหมาย (โดเมน F5 XC load balancer)
TARGET_FQDN=demo.example.com

# ทางเลือก: IP ของ origin โดยตรงสำหรับการทดสอบพื้นฐาน (ข้ามผ่าน F5 XC)
TARGET_ORIGIN_IP=
```

ไฟล์นี้ถูกเขียนโดยอัตโนมัติระหว่างการ provisioning ด้วย Terraform จากตัวแปร `target_fqdn` และ `target_origin_ip` หากต้องการเปลี่ยนเป้าหมายหลังการ deploy ให้แก้ไขไฟล์นี้โดยตรงบน VM

### การแทนที่ตัวแปรสภาพแวดล้อม

ตัวแปรใด ๆ ใน `config.env` สามารถถูกแทนที่ได้โดยการ export ก่อนเรียกใช้ชุดสคริปต์:

```bash
# แทนที่ FQDN เป้าหมายสำหรับการเรียกใช้ครั้งเดียว
TARGET_FQDN=staging.example.com /opt/traffic-generator/suites/runner.sh web-app-attacks

# แทนที่ผ่าน export
export TARGET_FQDN=staging.example.com
/opt/traffic-generator/suites/runner.sh web-app-attacks
```

พาธของไฟล์ config เองก็สามารถถูกแทนที่ได้เช่นกัน:

```bash
CONFIG_FILE=/tmp/my-config.env /opt/traffic-generator/suites/runner.sh web-app-attacks
```

## โหมด Dry-Run

แฟล็ก `--dry-run` จะแสดงแต่ละสคริปต์ที่จะถูกเรียกใช้โดยไม่ได้เรียกใช้จริง:

```bash
/opt/traffic-generator/suites/runner.sh web-app-attacks --dry-run
```

ผลลัพธ์:

```
=== 01-sqli.sh ===
[DRY-RUN] Would execute: /opt/traffic-generator/suites/web-app-attacks/01-sqli.sh demo.example.com
=== 02-xss.sh ===
[DRY-RUN] Would execute: /opt/traffic-generator/suites/web-app-attacks/02-xss.sh demo.example.com
=== 03-command-injection.sh ===
[DRY-RUN] Would execute: /opt/traffic-generator/suites/web-app-attacks/03-command-injection.sh demo.example.com
=== 04-path-traversal.sh ===
[DRY-RUN] Would execute: /opt/traffic-generator/suites/web-app-attacks/04-path-traversal.sh demo.example.com
=== 05-nikto-scan.sh ===
[DRY-RUN] Would execute: /opt/traffic-generator/suites/web-app-attacks/05-nikto-scan.sh demo.example.com
=== 06-nuclei-scan.sh ===
[DRY-RUN] Would execute: /opt/traffic-generator/suites/web-app-attacks/06-nuclei-scan.sh demo.example.com
=== Suite Complete ===
Passed: 0 | Failed: 0 | Skipped: 6

```

ใช้ dry-run เพื่อตรวจสอบโครงสร้างชุดสคริปต์หลังการ deploy หรือก่อนเรียกใช้ชุดสคริปต์กับเป้าหมายใหม่

## ไดเรกทอรีผลลัพธ์

การเรียกใช้ชุดสคริปต์แต่ละครั้งจะสร้างไดเรกทอรีผลลัพธ์ที่มีการประทับเวลา:

```
/opt/traffic-generator/results/<YYYYMMDD-HHMMSS>-<suite-name>/
```

**โครงสร้าง:**

```
/opt/traffic-generator/results/20260425-143000-web-app-attacks/
  meta.json
  01-sqli.sh.log
  02-xss.sh.log
  03-command-injection.sh.log
  04-path-traversal.sh.log
  05-nikto-scan.sh.log
  06-nuclei-scan.sh.log
```

stdout และ stderr ของแต่ละสคริปต์จะถูกบันทึกลงในไฟล์ `.log` ที่ตั้งชื่อตามสคริปต์

### รูปแบบ meta.json

ไฟล์ `meta.json` จะบันทึกเมตาดาต้าของการเรียกใช้ชุดสคริปต์:

```json
{
  "suite": "web-app-attacks",
  "target": "demo.example.com",
  "started": "2026-04-25T14:30:00Z",
  "completed": "2026-04-25T14:45:23Z",
  "status": "completed",
  "passed": 5,
  "failed": 1,
  "skipped": 0
}
```

| ฟิลด์ | คำอธิบาย |
|---|---|
| `suite` | ชื่อชุดสคริปต์ |
| `target` | FQDN เป้าหมายที่ใช้ในการเรียกใช้ |
| `started` | การประทับเวลา UTC เมื่อชุดสคริปต์เริ่มทำงาน |
| `completed` | การประทับเวลา UTC เมื่อชุดสคริปต์ทำงานเสร็จสิ้น |
| `status` | `running` ระหว่างการทำงาน, `completed` เมื่อเสร็จสิ้น |
| `passed` | จำนวนสคริปต์ที่ออกด้วยรหัส 0 |
| `failed` | จำนวนสคริปต์ที่ออกด้วยรหัสที่ไม่ใช่ศูนย์ |
| `skipped` | จำนวนสคริปต์ที่ถูกข้าม (ไม่สามารถเรียกใช้ได้หรือเป็น dry-run) |

<Aside type="tip">
รหัสออกที่ไม่ใช่ศูนย์จากสคริปต์ไม่ได้หมายความว่าชุดสคริปต์ล้มเหลวเสมอไป เครื่องมือด้านความปลอดภัยหลายตัวจะออกด้วยรหัสที่ไม่ใช่ศูนย์เมื่อพบช่องโหว่ ซึ่งเป็นพฤติกรรมที่คาดหวังระหว่างการสร้างทราฟฟิกการโจมตี
</Aside>

## การเรียกใช้สคริปต์แต่ละตัว

แต่ละสคริปต์สามารถเรียกใช้แบบอิสระได้โดยไม่ต้องผ่าน runner:

```bash
# เรียกใช้สคริปต์เดียวโดยตรง
/opt/traffic-generator/suites/web-app-attacks/01-sqli.sh demo.example.com

# เรียกใช้การโจมตี API เฉพาะตัว
/opt/traffic-generator/suites/api-attacks/01-vampi-owasp-top10.sh demo.example.com
```

สคริปต์รับ FQDN เป้าหมายเป็นอาร์กิวเมนต์ตำแหน่งแรก สคริปต์ไม่ได้อ่าน `config.env` โดยตรง -- มีเพียง runner เท่านั้นที่ทำ เมื่อเรียกใช้แบบอิสระ คุณต้องส่ง FQDN อย่างชัดเจน

## การเรียกใช้ชุดสคริปต์ทั้งหมดตามลำดับ

เพื่อเรียกใช้ชุดสคริปต์ทุกชุดต่อเนื่องกัน:

```bash
for suite in web-app-attacks api-attacks bot-simulation reconnaissance ssl-scanning javascript-exploits traffic-generation; do
  echo "========================================="
  echo "Starting suite: ${suite}"
  echo "========================================="
  /opt/traffic-generator/suites/runner.sh "$suite"
  echo ""
done
```

สิ่งนี้จะสร้างทราฟฟิกที่ครอบคลุมทั่วทุกฟีเจอร์ด้านความปลอดภัยของ F5 XC คาดว่าการเรียกใช้ทั้งหมดจะใช้เวลา 45-80 นาที ขึ้นอยู่กับเวลาตอบสนองของเป้าหมาย

## การเรียกใช้ระยะไกลผ่าน SSH

เรียกใช้ชุดสคริปต์จากเครื่องของคุณโดยไม่ต้องคงเซสชัน SSH ไว้:

```bash
TGEN_IP=$(terraform output -raw public_ip)

# เรียกใช้ชุดสคริปต์เดียว
ssh azureuser@${TGEN_IP} '/opt/traffic-generator/suites/runner.sh web-app-attacks'

# เรียกใช้ชุดสคริปต์พร้อมแทนที่เป้าหมาย
ssh azureuser@${TGEN_IP} 'TARGET_FQDN=staging.example.com /opt/traffic-generator/suites/runner.sh api-attacks'

# เรียกใช้ในพื้นหลัง (ปลอดภัยจากการตัดการเชื่อมต่อ)
ssh azureuser@${TGEN_IP} 'nohup /opt/traffic-generator/suites/runner.sh web-app-attacks > /tmp/web-app-attacks.log 2>&1 &'

# ตรวจสอบผลลัพธ์ภายหลัง
ssh azureuser@${TGEN_IP} 'ls -la /opt/traffic-generator/results/ | tail -5'
ssh azureuser@${TGEN_IP} 'cat /opt/traffic-generator/results/$(ls -t /opt/traffic-generator/results/ | head -1)/meta.json'
```

สำหรับชุดสคริปต์ที่ใช้เวลานาน ให้ใช้ `nohup` หรือ `tmux` เพื่อป้องกันการตัดการเชื่อมต่อ SSH จากการยกเลิกกระบวนการ:

```bash
ssh azureuser@${TGEN_IP} 'tmux new-session -d -s traffic "/opt/traffic-generator/suites/runner.sh reconnaissance"'

# เชื่อมต่อกลับภายหลังเพื่อดูความคืบหน้า
ssh -t azureuser@${TGEN_IP} 'tmux attach-session -t traffic'
```

## ลำดับการทำงาน

<Steps>

1. **โหลดการตั้งค่า** -- Runner จะอ่าน `config.env` (หรือค่าที่แทนที่จาก `CONFIG_FILE`) จากนั้นตรวจสอบ `TARGET_FQDN` ในตัวแปรสภาพแวดล้อม

2. **ตรวจสอบชุดสคริปต์** -- ยืนยันว่าไดเรกทอรีชุดสคริปต์ที่ระบุชื่อมีอยู่ภายใต้ไดเรกทอรี suites หากไม่พบจะแสดงรายการชุดสคริปต์ที่มีอยู่

3. **สร้างไดเรกทอรีผลลัพธ์** -- สร้าง `/opt/traffic-generator/results/<timestamp>-<suite>/` และเขียน `meta.json` เริ่มต้นพร้อม `status: running`

4. **เรียกใช้สคริปต์** -- วนซ้ำผ่านไฟล์ที่ตรงกับรูปแบบ `[0-9]*` ในไดเรกทอรีชุดสคริปต์ เรียงตามชื่อ ข้ามไฟล์ที่ไม่สามารถเรียกใช้ได้ ส่ง `TARGET_FQDN` เป็นอาร์กิวเมนต์แรก บันทึกผลลัพธ์ลงใน `<script-name>.log`

5. **บันทึกผลลัพธ์** -- อัปเดต `meta.json` พร้อมการประทับเวลาเสร็จสิ้นและจำนวนผ่าน/ล้มเหลว/ข้าม

</Steps>
