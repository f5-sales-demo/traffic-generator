แต่ละชุดการจราจรเป็นไดเรกทอรีภายใต้ `/opt/traffic-generator/suites/` ที่ประกอบด้วยเชลล์สคริปต์ที่มีหมายเลขลำดับ ซึ่งจะถูกรันตามลำดับโดย `runner.sh` สคริปต์ทุกตัวรับ FQDN เป้าหมายเป็นอาร์กิวเมนต์แรก และเขียนผลลัพธ์ออกทั้ง stdout และไดเรกทอรีผลลัพธ์

## api-attacks

**สคริปต์:** 4 | **ระยะเวลาโดยประมาณ:** 10-17 นาที

การทดสอบ OWASP API Security Top 10 กับ VAmPI ผ่าน F5 XC ลงทะเบียนผู้ใช้ทดสอบ รับโทเค็นการยืนยันตัวตน จากนั้นทดสอบ BOLA, การโจมตีแบบ brute-force, การเปิดเผยข้อมูลมากเกินไป, การอนุญาตระดับฟังก์ชันที่ผิดพลาด, รูปแบบ SSRF, SQL injection กับ REST API endpoint, การค้นหาพารามิเตอร์ที่ซ่อนอยู่ และการ fuzzing endpoint มีเป้าหมายคือแอปพลิเคชัน VAmPI

**ฟีเจอร์ F5 XC:** API Security / API Discovery

| ลำดับ | สคริปต์ | เครื่องมือ | คำอธิบาย |
|---|---|---|---|
| 01 | `01-vampi-owasp-top10.sh` | curl, jq | ชุดทดสอบ OWASP API Top 10 แบบครบวงจร ลงทะเบียนผู้ใช้ทดสอบ รับโทเค็นการยืนยันตัวตน จากนั้นทดสอบ BOLA (เข้าถึงข้อมูลของผู้ใช้อื่น), การโจมตีแบบ brute-force, การเปิดเผยข้อมูลมากเกินไป, การอนุญาตระดับฟังก์ชันที่ผิดพลาด, รูปแบบ SSRF และการตรวจสอบการตั้งค่าความปลอดภัยที่ผิดพลาด |
| 02 | `02-sqlmap-api.sh` | sqlmap | SQL injection กับ VAmPI REST API endpoint: ค้นหาผู้ใช้, ล็อกอิน (POST) และลงทะเบียน (POST) ด้วย JSON payload |
| 03 | `03-arjun-param-discovery.sh` | arjun | การค้นหาพารามิเตอร์ที่ซ่อนอยู่ข้าม endpoint ของ Juice Shop, DVWA และ VAmPI ค้นพบ query parameter ที่ไม่มีในเอกสารซึ่งอาจรับ injection payload ได้ |
| 04 | `04-ffuf-api-fuzz.sh` | ffuf | การ fuzzing สองเฟส: การค้นหา endpoint โดยใช้ wordlist ของเส้นทาง API ทั่วไป (api, swagger, graphql, admin, .env, .git) ข้าม application prefix ทั้งหมด จากนั้น fuzzing เมธอด HTTP (GET, POST, PUT, DELETE, PATCH, OPTIONS, TRACE, PROPFIND) กับ endpoint หลัก |

**พฤติกรรมที่คาดหวังของ F5 XC:** API Discovery ควรแมป VAmPI API endpoint และระบุรูปแบบคำขอที่ผิดปกติ นโยบาย API Security ควรแจ้งเตือนความพยายามเข้าถึงโดยไม่ได้รับอนุญาต, injection payload ใน JSON body และรูปแบบการ enumerate ส่วน API Inventory ใน F5 XC จะแสดง endpoint ที่ค้นพบพร้อมจำนวนคำขอ

## bot-simulation

**สคริปต์:** 4 | **ระยะเวลาโดยประมาณ:** 5-10 นาที

กิจกรรมเบราว์เซอร์อัตโนมัติและรูปแบบการ crawl แบบบอทผ่าน F5 XC รวมถึง headless Chrome, Puppeteer พร้อม stealth plugin, การ crawl แบบรวดเร็ว และการโต้ตอบฟอร์มอัตโนมัติ มีเป้าหมายคือแอปพลิเคชันทั้งหมด

**ฟีเจอร์ F5 XC:** Bot Defense

| ลำดับ | สคริปต์ | เครื่องมือ | คำอธิบาย |
|---|---|---|---|
| 01 | `01-headless-chrome.sh` | playwright | การเรียกดูด้วย Headless Chromium: โหลดเป้าหมาย นำทางระหว่างเส้นทางแอปพลิเคชัน ดึงชื่อหน้าและเนื้อหา ถ่ายภาพหน้าจอ สร้างทราฟฟิกผ่านเบราว์เซอร์โดยไม่มี stealth plugin (ตรวจจับได้ว่าเป็นระบบอัตโนมัติ) |
| 02 | `02-puppeteer-stealth.sh` | puppeteer, puppeteer-extra-plugin-stealth | การเรียกดูแบบ stealth โดยใช้ puppeteer-extra พร้อม stealth plugin พยายามหลบเลี่ยงการตรวจจับบอทโดยการแก้ไขคุณสมบัติ navigator, WebGL และ browser fingerprint อื่น ๆ |
| 03 | `03-rapid-crawl.sh` | curl, wget | คำขอ HTTP ความถี่สูงโดยใช้ curl และ wget อย่างรวดเร็วต่อเนื่อง จำลองพฤติกรรมของ scraper โดยไม่มีการหน่วงเวลาระหว่างคำขอและการหมุนเวียน User-Agent แบบรุกรี้ |
| 04 | `04-form-interaction.sh` | playwright | การกรอกและส่งฟอร์มอัตโนมัติกับ DVWA login, Juice Shop registration และ VAmPI API endpoint โดยใช้ API การโต้ตอบฟอร์มของ Playwright |

**พฤติกรรมที่คาดหวังของ F5 XC:** Bot Defense ควรจำแนกทราฟฟิกว่าเป็นระบบอัตโนมัติตามการวิเคราะห์ browser fingerprint, เวลาของคำขอ และผลลัพธ์ JavaScript challenge Headless Chrome ที่ไม่มี stealth ควรถูกตรวจจับทันที การทดสอบ Stealth Puppeteer ทดสอบว่าเทคนิคการหลบเลี่ยงขั้นสูงสามารถข้ามการตรวจจับได้หรือไม่ แดชบอร์ด Bot Defense จะแสดงหมวดหมู่การจำแนกบอทและการดำเนินการบรรเทา

## demoapp-attacks

**สคริปต์:** 2 | **ระยะเวลาโดยประมาณ:** 2-3 นาที

Payload การโจมตีที่มุ่งเป้าไปยัง endpoint ทดสอบ WAF ที่มีในตัวของ F5 DemoApp (`/WAF/SQL`, `/WAF/XSS`, `/WAF/DIR`) ส่ง SQL injection, XSS และ path traversal payload ตรงไปยัง endpoint ที่ออกแบบมาเพื่อสาธิตพฤติกรรมการบล็อกของ WAF ใช้ชุดนี้เมื่อมุ่งเป้าที่ DemoApp content server แทนแอปพลิเคชัน origin-server มาตรฐาน

**ฟีเจอร์ F5 XC:** WAF (Web Application Firewall)

| ลำดับ | สคริปต์ | เครื่องมือ | คำอธิบาย |
|---|---|---|---|
| 01 | `01-sqli-waf-endpoint.sh` | curl, python3 | SQL injection payload 15 รายการกับ `/WAF/SQL?age=` รวมถึง UNION SELECT, DROP TABLE, WAITFOR DELAY, xp_cmdshell, comment-based bypass และ boolean logic payload |
| 02 | `02-xss-waf-endpoint.sh` | curl, python3 | XSS payload 18 รายการกับ `/WAF/XSS?update=` รวมถึง script tag, event handler (onerror, onload, ontoggle, onfocus), base64 eval, style expression, polyglot payload และ cookie exfiltration |

**พฤติกรรมที่คาดหวังของ F5 XC:** WAF ควรบล็อกคำขอที่มี SQL injection และ XSS payload และส่งคืนหน้าบล็อก แดชบอร์ด Security Events จะแสดงหมวดหมู่การละเมิดและการจับคู่ signature สำหรับแต่ละคำขอที่ถูกบล็อก

## javascript-exploits

**สคริปต์:** 3 | **ระยะเวลาโดยประมาณ:** 3-5 นาที

รูปแบบการ injection สคริปต์ฝั่งไคลเอนต์และการจัดการ DOM ที่ทริกเกอร์การตรวจจับของ Client-Side Defense รวมถึง inline script injection ที่เลียนแบบ Magecart-style skimmer, การจัดการ DOM ผ่าน headless Chromium และการจำลองสคริปต์จากบุคคลที่สาม มีเป้าหมายคือแอปพลิเคชัน CSD Demo

**ฟีเจอร์ F5 XC:** Client-Side Defense (CSD)

| ลำดับ | สคริปต์ | เครื่องมือ | คำอธิบาย |
|---|---|---|---|
| 01 | `01-inline-script-injection.sh` | curl | ส่งคำขอที่มี inline JavaScript payload ที่ออกแบบมาเพื่อเลียนแบบ Magecart-style credit card skimmer และสคริปต์ขโมยข้อมูล |
| 02 | `02-dom-manipulation.sh` | playwright | ใช้ headless Chromium เพื่อโหลดหน้า CSD demo และ inject การแก้ไข DOM: เพิ่ม script tag, แก้ไข form action และแทรก iframe ที่มองไม่เห็นเพื่อขโมยข้อมูลฟอร์ม |
| 03 | `03-third-party-script-sim.sh` | curl, playwright | จำลองสคริปต์จากบุคคลที่สามที่ถูกบุกรุกโดยการ inject คำขอที่อ้างอิงแหล่ง JavaScript ภายนอกและพยายามโหลดสคริปต์ที่ไม่ได้รับอนุญาตเข้าสู่บริบทของหน้า |

**พฤติกรรมที่คาดหวังของ F5 XC:** Client-Side Defense ควรตรวจจับการแก้ไขสคริปต์ที่ไม่ได้รับอนุญาตในหน้าที่ได้รับการป้องกัน แดชบอร์ด CSD จะแสดงการแจ้งเตือนสำหรับแหล่งสคริปต์ใหม่, องค์ประกอบ DOM ที่ถูกแก้ไข และความพยายามขโมยข้อมูล นี่คือการตรวจสอบฟังก์ชันการทำงานของ CSD เฟส 2

การสแกนเครือข่าย, การ enumerate บริการ และการ brute-force ไดเรกทอรีกับโครงสร้างพื้นฐานเป้าหมาย รวมถึง nmap service detection, masscan port sweep, directory brute-forcing, subdomain enumeration, web technology fingerprinting และ DNS reconnaissance มีเป้าหมายคือแอปพลิเคชันทั้งหมด

**ฟีเจอร์ F5 XC:** WAF / Bot Defense

| ลำดับ | สคริปต์ | เครื่องมือ | คำอธิบาย |

| 02 | `02-masscan-sweep.sh` | masscan | การสแกนพอร์ตความเร็วสูงของพอร์ตเว็บทั่วไป (80, 443, 8080, 8443) พร้อมการจำกัดอัตรา |

การทดสอบ OWASP API Security Top 10 2023 กับแอปพลิเคชัน Restaurant API รวมถึง register/auth, BOLA profile, BOLA orders, BOPLA mass assignment, BFLA privilege escalation, rate limiting bypass, SQL injection menu, command injection disk stats, SSRF image URL, JWT weak secret และ sensitive data exposure มีเป้าหมายคือแอปพลิเคชัน Restaurant API

**ฟีเจอร์ F5 XC:** API Security

## ssl-scanning

**สคริปต์:** 3 | **ระยะเวลาโดยประมาณ:** 3-5 นาที

การวิเคราะห์การตั้งค่า TLS/SSL ของ HTTPS endpoint ของ F5 XC load balancer ระบุ cipher suite, เวอร์ชันโปรโตคอล, รายละเอียดใบรับรอง และตรวจสอบช่องโหว่ TLS ที่เป็นที่รู้จัก มีเป้าหมายคือ F5 XC load balancer endpoint

**ฟีเจอร์ F5 XC:** WAF (ข้อมูลเพื่อการอ้างอิง)

| ลำดับ | สคริปต์ | เครื่องมือ | คำอธิบาย |
|---|---|---|---|
| 01 | `01-sslscan.sh` | sslscan | ระบุ cipher suite ที่รองรับ, เวอร์ชันโปรโตคอล และรายละเอียดใบรับรอง |
| 02 | `02-sslyze.sh` | sslyze | การวิเคราะห์ TLS แบบครบวงจร รวมถึงการตรวจสอบใบรับรอง, ลำดับ cipher suite และการตรวจสอบช่องโหว่ (Heartbleed, ROBOT เป็นต้น) |
| 03 | `03-testssl.sh` | testssl.sh | การประเมิน TLS แบบเต็มรูปแบบโดยใช้กรอบ testssl.sh ทดสอบการรองรับโปรโตคอล, การตั้งค่า cipher, การวิเคราะห์ header และช่องโหว่ที่เป็นที่รู้จัก |

**พฤติกรรมที่คาดหวังของ F5 XC:** การสแกน SSL สร้าง TLS handshake จำนวนมากพร้อมข้อเสนอ cipher suite ที่ไม่ปกติ แม้ว่า F5 XC โดยทั่วไปจะไม่บล็อกสิ่งเหล่านี้ แต่รูปแบบทราฟฟิกจะมองเห็นได้ใน access log คุณค่าหลักคือการตรวจสอบว่าการตั้งค่า TLS ของ F5 XC เป็นไปตามแนวปฏิบัติด้านความปลอดภัยที่ดีที่สุด

## traffic-generation

**สคริปต์:** 4 | **ระยะเวลาโดยประมาณ:** 5-10 นาที (ปรับแต่งได้)

ทราฟฟิก HTTP ที่ถูกต้องปริมาณสูงสำหรับการวัดค่า baseline และการทดสอบโหลด สร้างรูปแบบทราฟฟิกปกติในการวิเคราะห์ F5 XC เพื่อเปรียบเทียบกับทราฟฟิกชุดโจมตี มีเป้าหมายคือแอปพลิเคชันทั้งหมด

**ฟีเจอร์ F5 XC:** ทั้งหมด (การเปรียบเทียบ baseline)

| ลำดับ | สคริปต์ | เครื่องมือ | คำอธิบาย |
|---|---|---|---|
| 01 | `01-baseline-http.sh` | curl | คำขอ HTTP GET แบบลำดับไปยังเส้นทางแอปพลิเคชันทั้งหมดพร้อม User-Agent ของเบราว์เซอร์มาตรฐาน สร้าง baseline ของทราฟฟิกปกติในการวิเคราะห์ F5 XC |
| 02 | `02-concurrent-load.sh` | curl | คำขอ HTTP แบบขนานโดยใช้ curl พร้อมการเชื่อมต่อพร้อมกันหลายรายการ ทดสอบการจัดการโหลดและสร้าง baseline ของ throughput |
| 03 | `03-mixed-methods.sh` | curl | เมธอด HTTP แบบผสม (GET, POST, PUT, DELETE) กับ API endpoint พร้อม payload ที่ถูกต้อง สร้างรูปแบบทราฟฟิก API ปกติเพื่อเปรียบเทียบกับทราฟฟิกการโจมตี |
| 04 | `04-user-journey.sh` | playwright | จำลองเส้นทางการใช้งานจริงของผู้ใช้: เรียกดูหน้า, ค้นหาสินค้า, กรอกฟอร์ม และนำทางระหว่างแอปพลิเคชันโดยใช้ Playwright พร้อมการตั้งค่าเบราว์เซอร์มาตรฐาน |

**พฤติกรรมที่คาดหวังของ F5 XC:** คำขอทั้งหมดในชุดนี้ควรผ่านได้โดยไม่มีการแทรกแซงจาก WAF หรือ Bot Defense ใช้ชุดนี้เพื่อสร้างทราฟฟิก "สะอาด" ในแดชบอร์ดการวิเคราะห์ จากนั้นเปรียบเทียบกับทราฟฟิกชุดโจมตีเพื่อสาธิตความแตกต่างระหว่างรูปแบบคำขอที่ถูกต้องและเป็นอันตราย

## waf-encoding-evasion

**สคริปต์:** 7 | **ระยะเวลาโดยประมาณ:** 5-10 นาที

การโจมตีด้วยการเข้ารหัสหลายชั้นเพื่อหลบเลี่ยง ออกแบบมาเพื่อทดสอบว่า WAF ทำการ normalize และถอดรหัส payload อย่างถูกต้องก่อนการตรวจสอบหรือไม่ ครอบคลุม URL encoding แบบชั้นเดียว สองชั้น และสามชั้น; HTML entity encoding (ทศนิยม, เลขฐานสิบหก, ชื่อ, เติมศูนย์); Unicode/UTF-8 insertion (zero-width space, BOM, fullwidth character, soft hyphen, overlong sequence); การเข้ารหัสแบบผสม/ซ้อนหลายชั้น (URL-encoded HTML entity, double-URL ภายใน entity, 3-layer stack); null byte injection; IIS %uXXXX encoding; case manipulation; chunked transfer encoding body splitting; Base64 parameter evasion; และ payload ที่เข้ารหัสใน header/cookie มีเป้าหมายคือแอปพลิเคชันใดก็ได้

**ฟีเจอร์ F5 XC:** WAF (ความลึกของการ normalize การเข้ารหัส)

| ลำดับ | สคริปต์ | เครื่องมือ | คำอธิบาย |
|---|---|---|---|
| 01 | `01-url-encoding.sh` | curl | URL encoding แบบชั้นเดียว สองชั้น และสามชั้นของ SQLi, XSS และ path traversal payload ข้ามเส้นทาง endpoint หลายรายการ ทดสอบว่า WAF ถอดรหัส URL parameter แบบ recursive หรือไม่ |
| 02 | `02-html-entity-encoding.sh` | curl, python3 | HTML entity encoding แบบทศนิยม, เลขฐานสิบหก, ชื่อ และเติมศูนย์ของ attack payload ทั้งใน GET parameter และ POST body รวมถึง template injection ผ่าน `&#123;&#123;7*7&#125;&#125;` |
| 03 | `03-unicode-utf8-evasion.sh` | curl | การแทรก zero-width space (U+200B), BOM (U+FEFF), fullwidth character, soft hyphen และ ZWNJ เข้าไปใน SQL keyword และ XSS tag ทดสอบ overlong UTF-8 sequence (2-byte และ 3-byte) |
| 04 | `04-mixed-nested-encoding.sh` | curl | Payload หลายชั้น: URL-encoded HTML entity, double-URL ภายใน entity, triple-layer stack, JSON body พร้อม Unicode escape (`<`) และรูปแบบ template injection `{{7*7}}` เฉพาะ |
| 05 | `05-null-byte-iis-base64.sh` | curl | Null byte injection (`%00`) สำหรับ extension bypass, IIS `%uXXXX` Unicode encoding และ Base64-encoded payload ใน URL parameter พร้อม decode flag |
| 06 | `06-case-chunked-evasion.sh` | curl | การสุ่ม case รวมกับการเข้ารหัส (`%3CsCrIpT%3E`), chunked transfer encoding เพื่อแบ่ง SQL/XSS keyword ข้าม HTTP chunk, Content-Length/Transfer-Encoding conflict probe และ alternate path separator (`%5c`, `%c0%af`) |
| 07 | `07-header-cookie-evasion.sh` | curl | Attack payload ที่เข้ารหัสถูก inject ผ่าน Cookie, Referer, User-Agent, X-Forwarded-For และ X-Original-URL header ทดสอบว่า WAF ตรวจสอบและถอดรหัส HTTP header field ทั้งหมดหรือไม่ ไม่ใช่แค่ query parameter และ POST body เท่านั้น |

**พฤติกรรมที่คาดหวังของ F5 XC:** WAF ที่กำหนดค่าอย่างถูกต้องควร normalize ชั้นการเข้ารหัสทั้งหมดก่อนการจับคู่รูปแบบ Payload ที่เข้ารหัสชั้นเดียวควรถูกบล็อกทันที การทดสอบการเข้ารหัสสองชั้นและสามชั้นเผยให้เห็นความลึกของ decode pipeline ของ WAF การทดสอบ Unicode insertion เผยให้เห็นว่าอักขระที่มองไม่เห็นถูกลบออกก่อนการจับคู่ keyword หรือไม่ ผลลัพธ์ที่ตัวแปรที่เข้ารหัสผ่านได้ในขณะที่ตัวแปรข้อความธรรมดาถูกบล็อกบ่งชี้ถึงช่องว่างในการ normalize ของนโยบาย WAF

## web-app-attacks

**สคริปต์:** 6 | **ระยะเวลาโดยประมาณ:** 12-20 นาที

การทดสอบช่องโหว่เว็บแอปพลิเคชัน OWASP Top 10 กับ Juice Shop และ DVWA ผ่าน F5 XC รวมถึง SQL injection, XSS, command injection, path traversal, การสแกน Nikto และการสแกนตาม template ของ Nuclei มีเป้าหมายคือแอปพลิเคชัน Juice Shop และ DVWA

**ฟีเจอร์ F5 XC:** WAF (Web Application Firewall)

| ลำดับ | สคริปต์ | เครื่องมือ | คำอธิบาย |
|---|---|---|---|
| 01 | `01-sqli.sh` | sqlmap, curl | SQL injection payload กับ Juice Shop search API และ DVWA SQLi endpoint รัน sqlmap automated scan จากนั้นส่ง injection payload โดยตรง 10 รายการผ่าน curl |
| 02 | `02-xss.sh` | dalfox, curl | Cross-site scripting กับ Juice Shop และ DVWA XSS reflected/stored endpoint รัน dalfox automated scan จากนั้นส่ง XSS payload โดยตรง 12 รายการ รวมถึง script tag, event handler และ cookie exfiltration |
| 03 | `03-command-injection.sh` | curl | OS command injection payload กับ DVWA exec endpoint ส่ง payload 12 รายการรวมถึง pipe chain, เครื่องหมายอัฒภาค, URL-encoded newline และรูปแบบ reverse shell |
| 04 | `04-path-traversal.sh` | curl | Directory traversal กับ endpoint ของแอปพลิเคชันหลายรายการ ทดสอบ `../` traversal มาตรฐาน, ตัวแปร URL-encoded, ตัวแปร double-encoded, ตัวแปร null-byte และตัวแปร Unicode ข้ามเส้นทาง endpoint 6 รายการ |
| 05 | `05-nikto-scan.sh` | nikto | การสแกนช่องโหว่เว็บเซิร์ฟเวอร์แบบเต็มรูปแบบพร้อมจำกัดเวลา 120 วินาที สร้างคำขอ HTTP ที่หลากหลายซึ่งทริกเกอร์การจับคู่ WAF signature |
| 06 | `06-nuclei-scan.sh` | nuclei | การสแกนช่องโหว่ตาม template ที่ระดับความรุนแรงปานกลาง สูง และวิกฤต จำกัดอัตราที่ 50 คำขอ/วินาที |

**พฤติกรรมที่คาดหวังของ F5 XC:** WAF ควรบล็อกหรือแจ้งเตือนคำขอที่มี SQL injection, XSS, command injection และ path traversal payload แดชบอร์ด Security Events จะแสดงหมวดหมู่การละเมิด, signature ID และรายละเอียดคำขอสำหรับแต่ละคำขอที่ถูกบล็อก
