## api-attacks

**สคริปต์:** 4 | **ระยะเวลาโดยประมาณ:** 10-17 นาที

การทดสอบ OWASP API Security Top 10 กับ VAmPI ผ่าน F5 XC ลงทะเบียนผู้ใช้ทดสอบ รับโทเค็นการยืนยันตัวตน จากนั้นทดสอบ BOLA, การโจมตีแบบ brute-force authentication, การเปิดเผยข้อมูลมากเกินไป, broken function-level authorization, รูปแบบ SSRF, SQL injection กับ REST API endpoints, การค้นหาพารามิเตอร์ที่ซ่อนอยู่ และ endpoint fuzzing กำหนดเป้าหมายไปที่แอปพลิเคชัน VAmPI

**ฟีเจอร์ F5 XC:** API Security / API Discovery

| ลำดับ | สคริปต์ | เครื่องมือ | คำอธิบาย |
|---|---|---|---|
| 01 | `01-vampi-owasp-top10.sh` | curl, jq | ชุดทดสอบ OWASP API Top 10 แบบครอบคลุม ลงทะเบียนผู้ใช้ทดสอบ รับโทเค็นการยืนยันตัวตน จากนั้นทดสอบ BOLA (การเข้าถึงข้อมูลของผู้ใช้อื่น), การโจมตีแบบ brute-force authentication, การเปิดเผยข้อมูลมากเกินไป, broken function-level authorization, รูปแบบ SSRF และการตรวจสอบ security misconfiguration |
| 02 | `02-sqlmap-api.sh` | sqlmap | SQL injection กับ VAmPI REST API endpoints: การค้นหาผู้ใช้, ล็อกอิน (POST) และลงทะเบียน (POST) ด้วย JSON payloads |
| 03 | `03-arjun-param-discovery.sh` | arjun | การค้นหาพารามิเตอร์ที่ซ่อนอยู่ใน Juice Shop, DVWA และ VAmPI endpoints ค้นพบพารามิเตอร์ query ที่ไม่มีเอกสารซึ่งอาจยอมรับ injection payloads |
| 04 | `04-ffuf-api-fuzz.sh` | ffuf | Fuzzing สองเฟส: การค้นหา endpoint โดยใช้ wordlist ของ API paths ทั่วไป (api, swagger, graphql, admin, .env, .git) ข้าม application prefixes ทั้งหมด จากนั้น fuzzing HTTP method (GET, POST, PUT, DELETE, PATCH, OPTIONS, TRACE, PROPFIND) กับ endpoints สำคัญ |

**พฤติกรรมที่คาดหวังของ F5 XC:** API Discovery ควรแมป VAmPI API endpoints และระบุรูปแบบคำขอที่ผิดปกติ นโยบาย API Security ควรแจ้งเตือนความพยายามเข้าถึงโดยไม่ได้รับอนุญาต, injection payloads ใน JSON bodies และรูปแบบการสำรวจ API Inventory ใน F5 XC จะแสดง endpoints ที่ค้นพบพร้อมจำนวนคำขอ

## bot-simulation

**สคริปต์:** 4 | **ระยะเวลาโดยประมาณ:** 5-10 นาที

กิจกรรมเบราว์เซอร์แบบอัตโนมัติและรูปแบบการ crawling แบบบอทผ่าน F5 XC รวมถึง headless Chrome, Puppeteer พร้อม stealth plugin, การ crawling แบบรวดเร็ว และการโต้ตอบกับฟอร์มแบบอัตโนมัติ กำหนดเป้าหมายไปที่แอปพลิเคชันทั้งหมด

**ฟีเจอร์ F5 XC:** Bot Defense

| ลำดับ | สคริปต์ | เครื่องมือ | คำอธิบาย |
|---|---|---|---|
| 01 | `01-headless-chrome.sh` | playwright | การเรียกดูด้วย Headless Chromium: โหลดเป้าหมาย นำทางระหว่าง application paths ดึงชื่อหน้าและเนื้อหา ถ่ายภาพหน้าจอ สร้างทราฟฟิกจากเบราว์เซอร์โดยไม่มี stealth plugins (ตรวจจับได้ว่าเป็นระบบอัตโนมัติ) |
| 02 | `02-puppeteer-stealth.sh` | puppeteer, puppeteer-extra-plugin-stealth | การเรียกดูแบบ stealth โดยใช้ puppeteer-extra พร้อม stealth plugin พยายามหลบเลี่ยงการตรวจจับบอทโดยการปรับแต่งคุณสมบัติ navigator, WebGL และ browser fingerprints อื่นๆ |
| 03 | `03-rapid-crawl.sh` | curl, wget | คำขอ HTTP ความถี่สูงโดยใช้ curl และ wget อย่างรวดเร็วต่อเนื่อง จำลองพฤติกรรม scraper โดยไม่มีการหน่วงเวลาระหว่างคำขอและการหมุนเวียน User-Agent อย่างเข้มงวด |
| 04 | `04-form-interaction.sh` | playwright | การกรอกฟอร์มและส่งข้อมูลแบบอัตโนมัติกับ DVWA login, Juice Shop registration และ VAmPI API endpoints โดยใช้ Playwright's form interaction APIs |

**พฤติกรรมที่คาดหวังของ F5 XC:** Bot Defense ควรจำแนกทราฟฟิกว่าเป็นระบบอัตโนมัติโดยอิงจากการวิเคราะห์ browser fingerprint, เวลาของคำขอ และผลลัพธ์ JavaScript challenge Headless Chrome ที่ไม่มี stealth ควรถูกตรวจจับทันที การทดสอบ Stealth Puppeteer จะตรวจสอบว่าเทคนิคการหลบเลี่ยงขั้นสูงสามารถผ่านการตรวจจับได้หรือไม่ แดชบอร์ด Bot Defense จะแสดงหมวดหมู่การจำแนกบอทและการดำเนินการบรรเทา

## cdn-load-testing

**สคริปต์:** 18 | **ระยะเวลาโดยประมาณ:** 20-30 นาที

การทดสอบพฤติกรรม CDN cache และการปกป้อง origin รวมถึง baseline throughput, การแยก query string, การเปลี่ยนแปลง accept-encoding, การจำลอง thundering herd, การ bypass POST/PUT, การชนกันของ cache key, conditional GET, range request, การจำลอง purge, การหมดทรัพยากร connection pool, การเพิ่มประสิทธิภาพ keepalive, ค่าใช้จ่าย TLS handshake, การ caching วัตถุขนาดใหญ่, multi-origin failover, rate limiting, การทดสอบอัตราส่วน gzip, HTTP/2 multiplexing และ combined kraken benchmark กำหนดเป้าหมายไปที่แอปพลิเคชัน CDN Simulator

**ฟีเจอร์ F5 XC:** CDN Integration

## demoapp-attacks

**สคริปต์:** 2 | **ระยะเวลาโดยประมาณ:** 2-3 นาที

payloads การโจมตีที่กำหนดเป้าหมายไปที่ WAF testing endpoints ในตัวของ F5 DemoApp (`/WAF/SQL`, `/WAF/XSS`, `/WAF/DIR`) ส่ง SQL injection, XSS และ path traversal payloads ไปยัง endpoints ที่ออกแบบมาเพื่อสาธิตพฤติกรรมการบล็อกของ WAF โดยตรง ใช้ชุดนี้เมื่อกำหนดเป้าหมายไปที่เซิร์ฟเวอร์เนื้อหา DemoApp แทนแอปพลิเคชัน origin-server มาตรฐาน

**ฟีเจอร์ F5 XC:** WAF (Web Application Firewall)

| ลำดับ | สคริปต์ | เครื่องมือ | คำอธิบาย |
|---|---|---|---|
| 01 | `01-sqli-waf-endpoint.sh` | curl, python3 | SQL injection payloads 15 รายการกับ `/WAF/SQL?age=` รวมถึง UNION SELECT, DROP TABLE, WAITFOR DELAY, xp_cmdshell, comment-based bypass และ boolean logic payloads |
| 02 | `02-xss-waf-endpoint.sh` | curl, python3 | XSS payloads 18 รายการกับ `/WAF/XSS?update=` รวมถึง script tags, event handlers (onerror, onload, ontoggle, onfocus), base64 eval, style expression, polyglot payloads และ cookie exfiltration |

**พฤติกรรมที่คาดหวังของ F5 XC:** WAF ควรบล็อกคำขอที่มี SQL injection และ XSS payloads และส่งกลับหน้าบล็อก แดชบอร์ด Security Events จะแสดงหมวดหมู่การละเมิดและ signature matches สำหรับแต่ละคำขอที่ถูกบล็อก

## javascript-exploits

**สคริปต์:** 3 | **ระยะเวลาโดยประมาณ:** 3-5 นาที

รูปแบบ client-side script injection และ DOM manipulation ที่กระตุ้นการตรวจจับ Client-Side Defense รวมถึง inline script injection ที่เลียนแบบ skimmers แบบ Magecart, DOM manipulation ผ่าน headless Chromium และการจำลอง third-party script กำหนดเป้าหมายไปที่แอปพลิเคชัน CSD Demo

**ฟีเจอร์ F5 XC:** Client-Side Defense (CSD)

| ลำดับ | สคริปต์ | เครื่องมือ | คำอธิบาย |
|---|---|---|---|
| 01 | `01-inline-script-injection.sh` | curl | ส่งคำขอที่มี inline JavaScript payloads ที่ออกแบบมาเพื่อเลียนแบบ credit card skimmers แบบ Magecart และสคริปต์ขโมยข้อมูล |
| 02 | `02-dom-manipulation.sh` | playwright | ใช้ headless Chromium เพื่อโหลดหน้า CSD demo และ inject การแก้ไข DOM: เพิ่ม script tags, แก้ไข form actions และแทรก invisible iframes ที่ขโมยข้อมูลฟอร์ม |
| 03 | `03-third-party-script-sim.sh` | curl, playwright | จำลองสคริปต์ third-party ที่ถูกบุกรุกโดย inject คำขอที่อ้างอิงแหล่ง JavaScript ภายนอกและพยายามโหลดสคริปต์ที่ไม่ได้รับอนุญาตเข้าสู่บริบทของหน้า |

**พฤติกรรมที่คาดหวังของ F5 XC:** Client-Side Defense ควรตรวจจับการแก้ไขสคริปต์ที่ไม่ได้รับอนุญาตในหน้าที่ได้รับการป้องกัน แดชบอร์ด CSD จะแสดงการแจ้งเตือนสำหรับแหล่งสคริปต์ใหม่, องค์ประกอบ DOM ที่ถูกแก้ไข และความพยายามขโมยข้อมูล สิ่งนี้ตรวจสอบฟังก์ชันการทำงาน CSD Phase 2

## mitre-attack

## ssl-scanning

**สคริปต์:** 3 | **ระยะเวลาโดยประมาณ:** 3-5 นาที

การวิเคราะห์การกำหนดค่า TLS/SSL ของ HTTPS endpoint ของ F5 XC load balancer สำรวจ cipher suites, เวอร์ชันโปรโตคอล, รายละเอียดใบรับรอง และตรวจสอบช่องโหว่ TLS ที่เป็นที่รู้จัก กำหนดเป้าหมายไปที่ F5 XC load balancer endpoint

**ฟีเจอร์ F5 XC:** WAF (ข้อมูลเชิงข้อเท็จจริง)

| ลำดับ | สคริปต์ | เครื่องมือ | คำอธิบาย |
|---|---|---|---|
| 01 | `01-sslscan.sh` | sslscan | สำรวจ cipher suites, เวอร์ชันโปรโตคอล และรายละเอียดใบรับรองที่รองรับ |
| 02 | `02-sslyze.sh` | sslyze | การวิเคราะห์ TLS แบบครอบคลุมรวมถึงการตรวจสอบใบรับรอง, การจัดลำดับ cipher suite และการตรวจสอบช่องโหว่ (Heartbleed, ROBOT เป็นต้น) |
| 03 | `03-testssl.sh` | testssl.sh | การประเมิน TLS แบบครบถ้วนโดยใช้กรอบการทำงาน testssl.sh ทดสอบการรองรับโปรโตคอล, การตั้งค่า cipher, การวิเคราะห์ header และช่องโหว่ที่เป็นที่รู้จัก |

**พฤติกรรมที่คาดหวังของ F5 XC:** การสแกน SSL สร้าง TLS handshakes จำนวนมากพร้อมข้อเสนอ cipher suite ที่ผิดปกติ แม้ว่า F5 XC จะไม่บล็อกสิ่งเหล่านี้โดยทั่วไป แต่รูปแบบทราฟฟิกจะมองเห็นได้ใน access logs คุณค่าหลักคือการตรวจสอบว่าการกำหนดค่า TLS ของ F5 XC เป็นไปตามแนวปฏิบัติด้านความปลอดภัยที่ดีที่สุด

## traffic-generation

**สคริปต์:** 4 | **ระยะเวลาโดยประมาณ:** 5-10 นาที (ปรับได้)

ทราฟฟิก HTTP ที่ถูกต้องปริมาณมากสำหรับการวัด baseline และการทดสอบโหลด สร้างรูปแบบทราฟฟิกปกติในการวิเคราะห์ F5 XC เพื่อเปรียบเทียบกับทราฟฟิกชุดโจมตี กำหนดเป้าหมายไปที่แอปพลิเคชันทั้งหมด

**ฟีเจอร์ F5 XC:** ทั้งหมด (การเปรียบเทียบ baseline)

| ลำดับ | สคริปต์ | เครื่องมือ | คำอธิบาย |
|---|---|---|---|
| 01 | `01-baseline-http.sh` | curl | คำขอ HTTP GET แบบต่อเนื่องไปยัง application paths ทั้งหมดด้วย User-Agent ของเบราว์เซอร์มาตรฐาน สร้าง baseline ของทราฟฟิกปกติในการวิเคราะห์ F5 XC |
| 02 | `02-concurrent-load.sh` | curl | คำขอ HTTP แบบขนานโดยใช้ curl พร้อมหลาย concurrent connections ทดสอบการจัดการโหลดและสร้าง throughput baselines |
| 03 | `03-mixed-methods.sh` | curl | HTTP methods แบบผสม (GET, POST, PUT, DELETE) กับ API endpoints ด้วย payloads ที่ถูกต้อง สร้างรูปแบบ API traffic ปกติเพื่อเปรียบเทียบกับทราฟฟิกโจมตี |
| 04 | `04-user-journey.sh` | playwright | จำลองการเดินทางของผู้ใช้จริง: เรียกดูหน้า ค้นหาสินค้า กรอกฟอร์ม และนำทางระหว่างแอปพลิเคชันโดยใช้ Playwright ด้วยการตั้งค่าเบราว์เซอร์มาตรฐาน |

**พฤติกรรมที่คาดหวังของ F5 XC:** คำขอทั้งหมดในชุดนี้ควรผ่านโดยไม่มีการแทรกแซงจาก WAF หรือ Bot Defense ใช้ชุดนี้เพื่อสร้างทราฟฟิก "สะอาด" ในแดชบอร์ดการวิเคราะห์ จากนั้นเปรียบเทียบกับทราฟฟิกชุดโจมตีเพื่อสาธิตความแตกต่างระหว่างรูปแบบคำขอที่ถูกต้องและเป็นอันตราย

## waf-encoding-evasion

**สคริปต์:** 7 | **ระยะเวลาโดยประมาณ:** 5-10 นาที

การโจมตีหลบเลี่ยงด้วยการเข้ารหัสหลายชั้นที่ออกแบบมาเพื่อทดสอบว่า WAF ทำการ normalize และถอดรหัส payloads อย่างถูกต้องก่อนการตรวจสอบหรือไม่ ครอบคลุม URL encoding แบบเดี่ยว สองชั้น และสามชั้น; HTML entity encoding (ทศนิยม, hex, ชื่อ, zero-padded); Unicode/UTF-8 insertion (zero-width space, BOM, fullwidth characters, soft hyphen, overlong sequences); mixed/nested multi-layer encoding (URL-encoded HTML entities, double-URL inside entities, 3-layer stacks); null byte injection; IIS %uXXXX encoding; case manipulation; chunked transfer encoding body splitting; Base64 parameter evasion; และ header/cookie encoded payloads กำหนดเป้าหมายไปที่แอปพลิเคชันใดก็ได้

**ฟีเจอร์ F5 XC:** WAF (ความลึกของการ normalize encoding)

| ลำดับ | สคริปต์ | เครื่องมือ | คำอธิบาย |
|---|---|---|---|
| 01 | `01-url-encoding.sh` | curl | URL encoding แบบเดี่ยว สองชั้น และสามชั้นของ SQLi, XSS และ path traversal payloads ข้าม endpoint paths หลายเส้นทาง ทดสอบว่า WAF ถอดรหัส URL parameters แบบ recursive หรือไม่ |
| 02 | `02-html-entity-encoding.sh` | curl, python3 | HTML entity encoding แบบทศนิยม, hex, ชื่อ และ zero-padded ของ attack payloads ทั้งใน GET parameters และ POST bodies รวมถึง template injection ผ่าน `&#123;&#123;7*7&#125;&#125;` |
| 03 | `03-unicode-utf8-evasion.sh` | curl | การแทรก Zero-width space (U+200B), BOM (U+FEFF), fullwidth character, soft hyphen และ ZWNJ เข้าไปใน SQL keywords และ XSS tags ทดสอบ overlong UTF-8 sequences (2-byte และ 3-byte) |
| 04 | `04-mixed-nested-encoding.sh` | curl | Multi-layer payloads: URL-encoded HTML entities, double-URL inside entities, triple-layer stacks, JSON body ที่มี Unicode escapes (`<`) และรูปแบบ template injection `{{7*7}}` เฉพาะของผู้ใช้ |
| 05 | `05-null-byte-iis-base64.sh` | curl | Null byte injection (`%00`) สำหรับ extension bypass, IIS `%uXXXX` Unicode encoding และ Base64-encoded payloads ใน URL parameters พร้อม decode flags |
| 06 | `06-case-chunked-evasion.sh` | curl | Case randomization ผสมกับ encoding (`%3CsCrIpT%3E`), chunked transfer encoding เพื่อแยก SQL/XSS keywords ข้าม HTTP chunks, Content-Length/Transfer-Encoding conflict probes และ alternate path separators (`%5c`, `%c0%af`) |
| 07 | `07-header-cookie-evasion.sh` | curl | Encoded attack payloads ที่ถูก inject ผ่าน Cookie, Referer, User-Agent, X-Forwarded-For และ X-Original-URL headers ทดสอบว่า WAF ตรวจสอบและถอดรหัส HTTP header fields ทั้งหมดหรือไม่ ไม่ใช่แค่ query parameters และ POST bodies |

**พฤติกรรมที่คาดหวังของ F5 XC:** WAF ที่กำหนดค่าอย่างถูกต้องควร normalize ชั้น encoding ทั้งหมดก่อนการจับคู่รูปแบบ payloads ที่เข้ารหัสชั้นเดียวควรถูกบล็อกทันที การทดสอบ double และ triple encoding เผยให้เห็นความลึกของ decode pipeline ของ WAF การทดสอบ Unicode insertion เผยให้เห็นว่าอักขระที่มองไม่เห็นถูกลบออกก่อนการจับคู่คำสำคัญหรือไม่ ผลลัพธ์ที่ตัวแปร encoded ผ่านในขณะที่ plain-text เทียบเท่าถูกบล็อกบ่งบอกถึงช่องว่างของการ normalize ในนโยบาย WAF

## web-app-attacks

**สคริปต์:** 6 | **ระยะเวลาโดยประมาณ:** 12-20 นาที

การทดสอบช่องโหว่เว็บแอปพลิเคชัน OWASP Top 10 กับ Juice Shop และ DVWA ผ่าน F5 XC รวมถึง SQL injection, XSS, command injection, path traversal, การสแกน Nikto และการสแกนตาม Nuclei template กำหนดเป้าหมายไปที่แอปพลิเคชัน Juice Shop และ DVWA

**ฟีเจอร์ F5 XC:** WAF (Web Application Firewall)

| ลำดับ | สคริปต์ | เครื่องมือ | คำอธิบาย |
|---|---|---|---|
| 01 | `01-sqli.sh` | sqlmap, curl | SQL injection payloads กับ Juice Shop search API และ DVWA SQLi endpoint เรียกใช้ sqlmap automated scans จากนั้นส่ง injection payloads โดยตรงด้วย curl 10 รายการ |
| 02 | `02-xss.sh` | dalfox, curl | Cross-site scripting กับ Juice Shop และ DVWA XSS reflected/stored endpoints เรียกใช้ dalfox automated scans จากนั้นส่ง XSS payloads โดยตรง 12 รายการ รวมถึง script tags, event handlers และ cookie exfiltration |
| 03 | `03-command-injection.sh` | curl | OS command injection payloads กับ DVWA exec endpoint ส่ง 12 payloads รวมถึง pipe chains, semicolons, URL-encoded newlines และ reverse shell patterns |
| 04 | `04-path-traversal.sh` | curl | Directory traversal กับ application endpoints หลายจุด ทดสอบ `../` traversals มาตรฐาน, URL-encoded variants, double-encoded variants, null-byte variants และ Unicode variants ข้าม 6 endpoint paths |
| 05 | `05-nikto-scan.sh` | nikto | การสแกนช่องโหว่เว็บเซิร์ฟเวอร์แบบเต็มรูปแบบด้วยขีดจำกัดเวลา 120 วินาที สร้างคำขอ HTTP ที่หลากหลายซึ่งกระตุ้นการจับคู่ WAF signature |
| 06 | `06-nuclei-scan.sh` | nuclei | การสแกนช่องโหว่ตาม template ที่ระดับความรุนแรงปานกลาง สูง และวิกฤต จำกัดอัตราที่ 50 คำขอ/วินาที |

**พฤติกรรมที่คาดหวังของ F5 XC:** WAF ควรบล็อกหรือแจ้งเตือนคำขอที่มี SQL injection, XSS, command injection และ path traversal payloads แดชบอร์ด Security Events จะแสดงหมวดหมู่การละเมิด, signature IDs และรายละเอียดคำขอสำหรับแต่ละคำขอที่ถูกบล็อก
