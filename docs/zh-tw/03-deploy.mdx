所有 Terraform 檔案都位於 [`terraform/`](https://github.com/f5xc-salesdemos/traffic-generator/tree/main/terraform) 目錄中。克隆儲存庫並直接部署：

```bash
git clone https://github.com/f5xc-salesdemos/traffic-generator.git
cd traffic-generator/terraform
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars with your Azure subscription ID and target FQDN
```

## Terraform 配置

### Terraform 檔案結構

terraform 目錄包含 9 個檔案，遵循 Demo Resource Standard：

- `versions.tf` -- Terraform 與提供者版本限制 (azurerm ~> 4.0, azuread ~> 3.0)
- `providers.tf` -- Azure RM 與 Azure AD 提供者配置
- `data.tf` -- Azure AD 資料來源，用於部署者自動解析
- `locals.tf` -- 部署者解析、Azure 雲端採用框架資源命名、標準標籤
- `main.tf` -- 資源群組（命名為 `rg-traffic-generator-{environment}-{deployer}`）
- `variables.tf` -- 所有輸入變數（2 個必填、9 個選填），透過 Azure AD 進行部署者自動解析
- `network.tf` -- VNet (10.201.0.0/16)、子網路、公用 IP、NSG（連接埠 22 SSH）、NIC
- `vm.tf` -- Ubuntu 24.04 VM，透過 templatefile() 使用 cloud-init
- `outputs.tf` -- 17 個輸出：15 個所有示範資源共享的標準輸出（deployer、public_ip、private_ip、ssh_command、resource_group_name、vm_name、nsg_name、vnet_name、subnet_id、component、environment、resource_group_id、vm_id、nsg_id、location）加上 2 個元件特定輸出（target_fqdn、status_check）

### 提供者與變數

`main.tf` 配置 Azure 提供者：

```hcl file=./_data/main.tf
```

`variables.tf` 定義所有可配置參數。`vm_size` 預設為 `Standard_F16s_v2`（16 vCPU 運算最佳化）-- 請參閱 [VM 大小調整](#vm-sizing) 了解擴展指引：

```hcl file=./_data/variables.tf
```

### 網路基礎架構

`network.tf` 建立 VNet、子網路、公用 IP、NSG（連接埠 22 SSH）和 NIC：

```hcl file=./_data/network.tf
```

### 搭配 Cloud-Init 的虛擬機器

`vm.tf` 建立 Ubuntu 24.04 VM，並傳遞包含目標和工具變數的 cloud-init 範本：

```hcl file=./_data/vm.tf
```

### Cloud-Init 佈建

`cloud-init.yaml` 是經過生產驗證的配置，已在超過 48 小時的持續負載測試中驗證，包括 4 輪源站壓力測試。它使用具有重試邏輯的輔助函式庫、進度記錄到 `/var/log/cloud-init-progress.log`、9 個佈建階段，以及條件式狀態報告（`status.json` 中的 ready/degraded）。它佈建以下內容：

- **核心調校** -- `somaxconn=131072`、`tcp_max_tw_buckets=4000000`、`tcp_fin_timeout=5`、64 MiB 通訊端緩衝區、`file-max=4194304`
- **NIC 最佳化** -- 跨所有核心的 RPS/RFS、透過 ethtool 最大化環形緩衝區、THP=always
- **APT 套件** -- nikto、nmap、masscan、sqlmap、hydra、medusa、ncrack、tshark、hping3、tcpdump、netcat、ngrep、iperf3、mtr、sslscan、socat、dirb、whatweb、wrk、hey、vegeta、ethtool
- **GitHub Releases** -- nuclei、dalfox、ffuf、gobuster、feroxbuster、subfinder、httpx、amass
- **Python 套件** -- mitmproxy、sslyze、dnsrecon、fierce、theHarvester、playwright、scapy、impacket、arjun
- **Node.js 20.x** -- Puppeteer 搭配隱匿外掛，用於機器人模擬
- **瀏覽器自動化** -- Playwright Chromium 用於無頭瀏覽器測試
- **Git 克隆** -- testssl.sh、recon-ng、spiderfoot、SecLists、waf-bypass、PayloadsAllTheThings
- **套件部署** -- 從此儲存庫克隆 19 個流量套件到 `/opt/traffic-generator/suites/`
- **配置檔** -- `config.env` 包含 `TARGET_FQDN`、`TARGET_ORIGIN_IP` 和 `CRAPI_PORT`

cloud-init 使用 Terraform 範本變數：`${target_fqdn}`、`${target_origin_ip}` 和 `${tool_tier}`。Shell 變數如 `$NPROC` 在 Terraform templatefile 中以 `$$NPROC` 進行跳脫。

```yaml file=./_data/cloud-init.yaml
```

### 輸出

`outputs.tf` 公開 17 個輸出，包括公用/私人 IP、SSH 命令、資源識別碼和元件特定值（target_fqdn、status_check）：

```hcl file=./_data/outputs.tf
```

### 變數檔案範例

將 `terraform.tfvars.example` 複製為 `terraform.tfvars` 並填入您的值。`.gitignore` 排除了 `terraform.tfvars` 以防止提交認證資訊：

```hcl file=./_data/terraform.tfvars.example
```

將 `subscription_id` 替換為您的 Azure 訂閱 ID（從 `az account show --query id -o tsv` 取得），並將 `target_fqdn` 替換為您的 F5 XC 負載平衡器網域。

## 部署

```bash
terraform init

terraform plan

terraform apply
```

Terraform 在部署成功後會輸出公用 IP、SSH 命令和目標 FQDN。

## VM 大小調整

建議使用 F 系列運算最佳化 VM 來執行此 CPU 密集型流量產生工作負載。cloud-init 的核心調校和工具並行度會根據 vCPU 數量自動擴展。

| VM SKU | vCPU | RAM | 網路 | 使用案例 |
|---|---|---|---|---|
| Standard_F4s_v2 | 4 | 8 GiB | 10 Gbps | 輕量實驗室/示範 |
| Standard_F8s_v2 | 8 | 16 GiB | 12.5 Gbps | 標準示範 |
| Standard_F16s_v2 | 16 | 32 GiB | 12.5 Gbps | 負載測試（預設） |
| Standard_F32s_v2 | 32 | 64 GiB | 16 Gbps | 重度基準測試 |
| Standard_D16s_v3 | 16 | 64 GiB | 12.5 Gbps | 記憶體密集型工作負載 |

:::tip
F16s_v2 經由針對 CDN F16s_v2 和 F32s_v2 目標的 A/B/C/D 基準測試驗證為最佳性價比選項。完整結果請參閱 [benchmarks/](https://github.com/f5xc-salesdemos/traffic-generator/tree/main/benchmarks)。
:::

## 部署後驗證

Cloud-init 佈建**標準層級需要 15-20 分鐘**，**完整層級約需 25 分鐘**。VM 可立即透過 SSH 存取，但工具仍在安裝中，直到 cloud-init 完成為止。

### 等待 Cloud-Init

SSH 連線到 VM 並等待佈建完成：

```bash
ssh azureuser@$(terraform output -raw public_ip)

cloud-init status --wait
```

完成時的預期輸出：

```
status: done
```

### 檢查狀態檔

cloud-init 腳本在完成時會寫入狀態檔：

```bash
cat /opt/traffic-generator/status.json
```

預期輸出：

```json
{
  "status": "ready",
  "tier": "standard",
  "target_fqdn": "demo.example.com",
  "completed": "2026-04-25T14:30:00Z",
  "tools_installed": true
}
```

### 驗證關鍵工具

執行驗證迴圈以確認每個關鍵工具在 PATH 中可用：

```bash
for tool in nikto sqlmap nuclei dalfox ffuf gobuster feroxbuster nmap masscan \
  hydra medusa ncrack sslscan tshark hping3 mitmproxy sslyze subfinder httpx \
  amass arjun mtr whatweb dirb socat netcat ngrep iperf3 whois dig wrk hey; do
  if command -v "$tool" > /dev/null 2>&1; then
    echo "OK: $tool"
  else
    echo "MISSING: $tool"
  fi
done
```

### 驗證套件執行器

確認執行器和套件已安裝：

```bash
ls /opt/traffic-generator/suites/runner.sh
ls /opt/traffic-generator/suites/*/

/opt/traffic-generator/suites/runner.sh web-app-attacks --dry-run
```

## 從上游元件連接

流量產生器的目標是前端源站伺服器的 F5 XC HTTP 負載平衡器。請先部署源站伺服器，配置 F5 XC，然後將 `target_fqdn` 設定為負載平衡器的 FQDN：

```bash
# After deploying origin-server and creating an F5 XC HTTP load balancer:
cat > terraform.tfvars <<EOF
subscription_id  = "your-subscription-id"
target_fqdn      = "your-xc-load-balancer.example.com"
target_origin_ip = "$(cd ../../origin-server/terraform && terraform output -raw public_ip)"
EOF
```

| 必填變數 | 來源 | 說明 |
|---|---|---|
| `target_fqdn` | F5 XC 主控台 | 保護源站伺服器的 HTTP 負載平衡器 FQDN |
| `target_origin_ip` | 源站伺服器 `public_ip` 輸出 | 用於旁路測試的直接源站 IP（選填） |

請繼續前往[工具目錄](../04-tool-catalog/)查看完整工具參考，或前往[套件](../05-suites/)開始產生流量。
