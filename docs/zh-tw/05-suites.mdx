每個流量測試套件都是 `/opt/traffic-generator/suites/` 下的一個目錄，包含由 `runner.sh` 依序執行的編號 shell 腳本。每個腳本接受目標 FQDN 作為第一個參數，並將輸出寫入 stdout 和結果目錄。

## api-attacks

**腳本數量：** 4 | **預估持續時間：** 10-17 分鐘

透過 F5 XC 針對 VAmPI 進行 OWASP API 安全前十大測試。註冊測試使用者、取得認證令牌，然後測試 BOLA、暴力破解認證、過度資料暴露、功能層級授權失效、SSRF 模式、針對 REST API 端點的 SQL 注入、隱藏參數探索及端點模糊測試。目標為 VAmPI 應用程式。

**F5 XC 功能：** API 安全 / API 探索

| 順序 | 腳本 | 工具 | 說明 |
|---|---|---|---|
| 01 | `01-vampi-owasp-top10.sh` | curl, jq | 完整的 OWASP API 前十大測試套件。註冊測試使用者、取得認證令牌，然後測試 BOLA（存取其他使用者的資料）、暴力破解認證、過度資料暴露、功能層級授權失效、SSRF 模式及安全配置錯誤探測。 |
| 02 | `02-sqlmap-api.sh` | sqlmap | 針對 VAmPI REST API 端點的 SQL 注入：使用者查詢、登入（POST）及註冊（POST），使用 JSON 酬載。 |
| 03 | `03-arjun-param-discovery.sh` | arjun | 跨 Juice Shop、DVWA 及 VAmPI 端點的隱藏參數探索。發現可能接受注入酬載的未文件化查詢參數。 |
| 04 | `04-ffuf-api-fuzz.sh` | ffuf | 兩階段模糊測試：使用常見 API 路徑字典（api、swagger、graphql、admin、.env、.git）進行端點探索，橫跨所有應用程式前綴，然後對關鍵端點進行 HTTP 方法模糊測試（GET、POST、PUT、DELETE、PATCH、OPTIONS、TRACE、PROPFIND）。 |

**預期 F5 XC 行為：** API 探索應能對映 VAmPI API 端點並識別異常請求模式。API 安全策略應標記未授權存取嘗試、JSON 主體中的注入酬載及列舉模式。F5 XC 中的 API 清單將顯示已發現的端點及請求計數。

## bot-simulation

**腳本數量：** 4 | **預估持續時間：** 5-10 分鐘

透過 F5 XC 的自動化瀏覽器活動和類機器人爬取模式。包括無頭 Chrome、帶隱匿外掛的 Puppeteer、快速爬取及自動化表單互動。目標為所有應用程式。

**F5 XC 功能：** Bot 防禦

| 順序 | 腳本 | 工具 | 說明 |
|---|---|---|---|
| 01 | `01-headless-chrome.sh` | playwright | 無頭 Chromium 瀏覽：載入目標、在應用程式路徑間導覽、擷取頁面標題和內容、截取螢幕截圖。產生基於瀏覽器的流量但不使用隱匿外掛（可被偵測為自動化）。 |
| 02 | `02-puppeteer-stealth.sh` | puppeteer, puppeteer-extra-plugin-stealth | 使用 puppeteer-extra 及隱匿外掛進行隱匿瀏覽。嘗試透過修補 navigator 屬性、WebGL 及其他瀏覽器指紋來規避機器人偵測。 |
| 03 | `03-rapid-crawl.sh` | curl, wget | 使用 curl 和 wget 快速連續發送高頻 HTTP 請求。模擬爬蟲行為，請求間無延遲且積極輪換 User-Agent。 |
| 04 | `04-form-interaction.sh` | playwright | 使用 Playwright 的表單互動 API 對 DVWA 登入、Juice Shop 註冊及 VAmPI API 端點進行自動化表單填寫和提交。 |

**預期 F5 XC 行為：** Bot 防禦應根據瀏覽器指紋分析、請求時序和 JavaScript 挑戰結果將流量分類為自動化流量。無隱匿功能的無頭 Chrome 應被立即偵測。隱匿 Puppeteer 測試進階規避技術是否能繞過偵測。Bot 防禦儀表板將顯示機器人分類類別和緩解措施。

## ssl-scanning

**腳本數量：** 3 | **預估持續時間：** 3-5 分鐘

F5 XC 負載平衡器 HTTPS 端點的 TLS/SSL 配置分析。列舉加密套件、協定版本、憑證詳細資訊，並檢查已知的 TLS 漏洞。目標為 F5 XC 負載平衡器端點。

**F5 XC 功能：** WAF（資訊性）

| 順序 | 腳本 | 工具 | 說明 |
|---|---|---|---|
| 01 | `01-sslscan.sh` | sslscan | 列舉支援的加密套件、協定版本及憑證詳細資訊。 |
| 02 | `02-sslyze.sh` | sslyze | 完整的 TLS 分析，包括憑證驗證、加密套件排序及漏洞檢查（Heartbleed、ROBOT 等）。 |
| 03 | `03-testssl.sh` | testssl.sh | 使用 testssl.sh 框架進行完整的 TLS 評估。測試協定支援、加密偏好、標頭分析及已知漏洞。 |

**預期 F5 XC 行為：** SSL 掃描會產生許多帶有異常加密套件提議的 TLS 交握。雖然 F5 XC 通常不會阻擋這些流量，但流量模式在存取日誌中是可見的。主要價值在於驗證 F5 XC 的 TLS 配置是否符合安全最佳實踐。

## traffic-generation

**腳本數量：** 4 | **預估持續時間：** 5-10 分鐘（可配置）

用於基準量測和負載測試的大量合法 HTTP 流量。在 F5 XC 分析中建立正常流量模式，以便與攻擊套件流量進行比較。目標為所有應用程式。

**F5 XC 功能：** 全部（基準比較）

| 順序 | 腳本 | 工具 | 說明 |
|---|---|---|---|
| 01 | `01-baseline-http.sh` | curl | 使用標準瀏覽器 User-Agent 對所有應用程式路徑發送循序 HTTP GET 請求。在 F5 XC 分析中建立正常流量基準。 |
| 02 | `02-concurrent-load.sh` | curl | 使用 curl 多個並發連接發送並行 HTTP 請求。測試負載處理能力並建立吞吐量基準。 |
| 03 | `03-mixed-methods.sh` | curl | 對 API 端點使用混合 HTTP 方法（GET、POST、PUT、DELETE）及有效酬載。產生正常 API 流量模式以與攻擊流量進行比較。 |
| 04 | `04-user-journey.sh` | playwright | 模擬真實的使用者旅程：瀏覽頁面、搜尋產品、填寫表單，並使用 Playwright 以標準瀏覽器設定在應用程式間導覽。 |

**預期 F5 XC 行為：** 此套件中的所有請求都應在不受 WAF 或 Bot 防禦干預的情況下通過。使用此套件在分析儀表板中產生「乾淨」流量，然後與攻擊套件流量進行比較，以展示合法和惡意請求模式之間的差異。

## waf-encoding-evasion

**腳本數量：** 7 | **預估持續時間：** 5-10 分鐘

多層編碼規避攻擊，設計用於測試 WAF 在檢查前是否正確正規化和解碼酬載。涵蓋單重、雙重和三重 URL 編碼；HTML 實體編碼（十進位、十六進位、命名、零填充）；Unicode/UTF-8 插入（零寬空格、BOM、全形字元、軟連字號、超長序列）；混合/巢狀多層編碼（URL 編碼的 HTML 實體、實體內的雙重 URL、三層堆疊）；null byte 注入；IIS %uXXXX 編碼；大小寫操控；分塊傳輸編碼主體拆分；Base64 參數規避；以及標頭/cookie 編碼酬載。目標為任何應用程式。

**F5 XC 功能：** WAF（編碼正規化深度）

| 順序 | 腳本 | 工具 | 說明 |
|---|---|---|---|
| 01 | `01-url-encoding.sh` | curl | 跨多個端點路徑對 SQLi、XSS 和路徑遍歷酬載進行單重、雙重和三重 URL 編碼。測試 WAF 是否遞迴解碼 URL 參數。 |
| 02 | `02-html-entity-encoding.sh` | curl, python3 | 在 GET 參數和 POST 主體中對攻擊酬載進行十進位、十六進位、命名和零填充 HTML 實體編碼。包含透過 `&#123;&#123;7*7&#125;&#125;` 的範本注入。 |
| 03 | `03-unicode-utf8-evasion.sh` | curl | 在 SQL 關鍵字和 XSS 標籤中插入零寬空格（U+200B）、BOM（U+FEFF）、全形字元、軟連字號和 ZWNJ。測試超長 UTF-8 序列（2 位元組和 3 位元組）。 |
| 04 | `04-mixed-nested-encoding.sh` | curl | 多層酬載：URL 編碼的 HTML 實體、實體內的雙重 URL、三層堆疊、帶有 Unicode 跳脫的 JSON 主體（`<`），以及使用者特定的 `{{7*7}}` 範本注入模式。 |

**預期 F5 XC 行為：** 正確配置的 WAF 應在模式匹配前正規化所有編碼層。單重編碼酬載應被立即阻擋。雙重和三重編碼測試揭示 WAF 解碼管線的深度。Unicode 插入測試揭示在關鍵字匹配前是否移除了不可見字元。編碼變體通過而明文等效物被阻擋的結果表明 WAF 策略中存在正規化缺口。

## web-app-attacks

**腳本數量：** 6 | **預估持續時間：** 12-20 分鐘

透過 F5 XC 針對 Juice Shop 和 DVWA 的 OWASP 前十大 Web 應用程式漏洞測試。包括 SQL 注入、XSS、命令注入、路徑遍歷、Nikto 掃描及 Nuclei 範本掃描。目標為 Juice Shop 和 DVWA 應用程式。

**F5 XC 功能：** WAF（Web 應用程式防火牆）

| 順序 | 腳本 | 工具 | 說明 |
|---|---|---|---|
| 01 | `01-sqli.sh` | sqlmap, curl | 針對 Juice Shop 搜尋 API 和 DVWA SQLi 端點的 SQL 注入酬載。執行 sqlmap 自動化掃描，然後發送 10 個直接的 curl 注入酬載。 |
| 02 | `02-xss.sh` | dalfox, curl | 針對 Juice Shop 和 DVWA XSS 反射型/儲存型端點的跨站腳本攻擊。執行 dalfox 自動化掃描，然後發送 12 個直接的 XSS 酬載，包括 script 標籤、事件處理器和 cookie 竊取。 |
| 03 | `03-command-injection.sh` | curl | 針對 DVWA exec 端點的作業系統命令注入酬載。發送 12 個酬載，包括管道鏈、分號、URL 編碼換行符和反向 shell 模式。 |
