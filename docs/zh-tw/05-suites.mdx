每個流量測試套件都是 `/opt/traffic-generator/suites/` 下的一個目錄，包含由 `runner.sh` 依序執行的編號 shell 腳本。每個腳本接受目標 FQDN 作為第一個參數，並將輸出寫入標準輸出和結果目錄。

## api-attacks

**腳本數：** 4 | **預估持續時間：** 10-17 分鐘

透過 F5 XC 對 VAmPI 進行 OWASP API 安全前十大測試。註冊測試使用者、取得驗證令牌，然後測試 BOLA、暴力破解認證、過度資料暴露、損壞的功能層級授權、SSRF 模式、針對 REST API 端點的 SQL 注入、隱藏參數探索及端點模糊測試。目標為 VAmPI 應用程式。

**F5 XC 功能：** API 安全 / API 探索

| 順序 | 腳本 | 工具 | 描述 |
|---|---|---|---|
| 01 | `01-vampi-owasp-top10.sh` | curl, jq | 全面的 OWASP API 前十大測試套件。註冊測試使用者、取得驗證令牌，然後測試 BOLA（存取其他使用者的資料）、暴力破解認證、過度資料暴露、損壞的功能層級授權、SSRF 模式及安全配置錯誤探測。 |
| 02 | `02-sqlmap-api.sh` | sqlmap | 針對 VAmPI REST API 端點的 SQL 注入：使用者查詢、登入（POST）和註冊（POST），使用 JSON 負載。 |
| 03 | `03-arjun-param-discovery.sh` | arjun | 跨 Juice Shop、DVWA 和 VAmPI 端點的隱藏參數探索。發現可能接受注入負載的未記錄查詢參數。 |
| 04 | `04-ffuf-api-fuzz.sh` | ffuf | 兩階段模糊測試：使用常見 API 路徑字典（api、swagger、graphql、admin、.env、.git）跨所有應用程式前綴進行端點探索，然後對關鍵端點進行 HTTP 方法模糊測試（GET、POST、PUT、DELETE、PATCH、OPTIONS、TRACE、PROPFIND）。 |

**預期 F5 XC 行為：** API 探索應映射 VAmPI API 端點並識別異常請求模式。API 安全策略應標記未授權存取嘗試、JSON 主體中的注入負載及列舉模式。F5 XC 中的 API 清單將顯示已發現的端點及請求計數。

## bot-simulation

**腳本數：** 4 | **預估持續時間：** 5-10 分鐘

透過 F5 XC 進行的自動化瀏覽器活動和類機器人爬取模式。包括無頭 Chrome、帶隱匿外掛的 Puppeteer、快速爬取及自動化表單互動。目標為所有應用程式。

**F5 XC 功能：** Bot Defense

| 順序 | 腳本 | 工具 | 描述 |
|---|---|---|---|
| 01 | `01-headless-chrome.sh` | playwright | 無頭 Chromium 瀏覽：載入目標、在應用程式路徑之間導航、擷取頁面標題和內容、截取螢幕截圖。產生基於瀏覽器的流量，不使用隱匿外掛（可被偵測為自動化）。 |
| 02 | `02-puppeteer-stealth.sh` | puppeteer, puppeteer-extra-plugin-stealth | 使用 puppeteer-extra 搭配隱匿外掛進行隱匿瀏覽。嘗試透過修補 navigator 屬性、WebGL 及其他瀏覽器指紋來規避機器人偵測。 |
| 03 | `03-rapid-crawl.sh` | curl, wget | 使用 curl 和 wget 快速連續發出高頻 HTTP 請求。模擬爬蟲行為，請求之間無延遲並積極輪換 User-Agent。 |
| 04 | `04-form-interaction.sh` | playwright | 使用 Playwright 的表單互動 API 對 DVWA 登入、Juice Shop 註冊及 VAmPI API 端點進行自動化表單填寫和提交。 |

**預期 F5 XC 行為：** Bot Defense 應根據瀏覽器指紋分析、請求時序和 JavaScript 挑戰結果將流量分類為自動化流量。無隱匿的無頭 Chrome 應被立即偵測到。隱匿 Puppeteer 測試進階規避技術是否能繞過偵測。Bot Defense 儀表板將顯示機器人分類類別和緩解操作。

## demoapp-attacks

**F5 XC 功能：** WAF（Web 應用程式防火牆）

| 順序 | 腳本 | 工具 | 描述 |
|---|---|---|---|

| 02 | `02-xss-waf-endpoint.sh` | curl, python3 | 針對 `/WAF/XSS?update=` 的 18 個 XSS 負載，包括 script 標籤、事件處理器（onerror、onload、ontoggle、onfocus）、base64 eval、style expression、polyglot 負載及 cookie 竊取。 |

**預期 F5 XC 行為：** WAF 應阻擋包含 SQL 注入和 XSS 負載的請求並返回阻擋頁面。安全事件儀表板將顯示每個被阻擋請求的違規類別和特徵碼匹配。

**F5 XC 功能：** WAF（資訊性）

| 順序 | 腳本 | 工具 | 描述 |
|---|---|---|---|
| 01 | `01-sslscan.sh` | sslscan | 列舉支援的密碼套件、協定版本及憑證詳細資訊。 |
| 02 | `02-sslyze.sh` | sslyze | 全面的 TLS 分析，包括憑證驗證、密碼套件排序及弱點檢查（Heartbleed、ROBOT 等）。 |

## traffic-generation

**腳本數：** 4 | **預估持續時間：** 5-10 分鐘（可配置）

用於基線測量和負載測試的大量合法 HTTP 流量。在 F5 XC 分析中建立正常流量模式，以便與攻擊套件流量進行比較。目標為所有應用程式。

**F5 XC 功能：** 全部（基線比較）

| 順序 | 腳本 | 工具 | 描述 |
|---|---|---|---|
| 01 | `01-baseline-http.sh` | curl | 使用標準瀏覽器 User-Agent 對所有應用程式路徑進行順序 HTTP GET 請求。在 F5 XC 分析中建立正常流量基線。 |
| 02 | `02-concurrent-load.sh` | curl | 使用 curl 搭配多個並行連線進行平行 HTTP 請求。測試負載處理能力並建立吞吐量基線。 |
| 03 | `03-mixed-methods.sh` | curl | 對 API 端點使用混合 HTTP 方法（GET、POST、PUT、DELETE）搭配有效負載。產生正常的 API 流量模式以與攻擊流量進行比較。 |
| 04 | `04-user-journey.sh` | playwright | 模擬真實的使用者旅程：瀏覽頁面、搜尋產品、填寫表單，以及使用 Playwright 搭配標準瀏覽器設定在應用程式之間導航。 |

**預期 F5 XC 行為：** 此套件中的所有請求應在不受 WAF 或 Bot Defense 干預的情況下通過。使用此套件在分析儀表板中產生「乾淨」流量，然後與攻擊套件流量進行比較，以展示合法與惡意請求模式之間的差異。

## waf-encoding-evasion

**腳本數：** 7 | **預估持續時間：** 5-10 分鐘

多層編碼規避攻擊，設計用於測試 WAF 是否在檢查前正確正規化和解碼負載。涵蓋單層、雙層和三層 URL 編碼；HTML 實體編碼（十進制、十六進制、命名、零填充）；Unicode/UTF-8 插入（零寬空格、BOM、全形字元、軟連字號、過長序列）；混合/巢狀多層編碼（URL 編碼的 HTML 實體、實體內的雙重 URL、三層堆疊）；null byte 注入；IIS %uXXXX 編碼；大小寫操作；分塊傳輸編碼主體分割；Base64 參數規避；以及標頭/cookie 編碼負載。目標為任何應用程式。

**F5 XC 功能：** WAF（編碼正規化深度）

| 順序 | 腳本 | 工具 | 描述 |
|---|---|---|---|
| 01 | `01-url-encoding.sh` | curl | 跨多個端點路徑對 SQLi、XSS 和路徑遍歷負載進行單層、雙層和三層 URL 編碼。測試 WAF 是否遞迴解碼 URL 參數。 |
| 02 | `02-html-entity-encoding.sh` | curl, python3 | 在 GET 參數和 POST 主體中對攻擊負載進行十進制、十六進制、命名及零填充 HTML 實體編碼。包括透過 `&#123;&#123;7*7&#125;&#125;` 的範本注入。 |
| 03 | `03-unicode-utf8-evasion.sh` | curl | 在 SQL 關鍵字和 XSS 標籤中插入零寬空格（U+200B）、BOM（U+FEFF）、全形字元、軟連字號和 ZWNJ。測試過長 UTF-8 序列（2 位元組和 3 位元組）。 |
| 04 | `04-mixed-nested-encoding.sh` | curl | 多層負載：URL 編碼的 HTML 實體、實體內的雙重 URL、三層堆疊、帶有 Unicode 跳脫的 JSON 主體（`<`），以及使用者特定的 `{{7*7}}` 範本注入模式。 |
| 05 | `05-null-byte-iis-base64.sh` | curl | Null byte 注入（`%00`）用於副檔名繞過、IIS `%uXXXX` Unicode 編碼，以及 URL 參數中帶解碼旗標的 Base64 編碼負載。 |

**預期 F5 XC 行為：** 正確配置的 WAF 應在模式比對之前正規化所有編碼層。單層編碼的負載應被立即阻擋。雙層和三層編碼測試揭示 WAF 解碼管線的深度。Unicode 插入測試揭示是否在關鍵字比對之前去除不可見字元。當編碼變體通過而純文字等效項被阻擋的結果表明 WAF 策略中存在正規化缺口。

## web-app-attacks

**腳本數：** 6 | **預估持續時間：** 12-20 分鐘

透過 F5 XC 對 Juice Shop 和 DVWA 進行 OWASP 前十大 Web 應用程式弱點測試。包括 SQL 注入、XSS、命令注入、路徑遍歷、Nikto 掃描及 Nuclei 範本掃描。目標為 Juice Shop 和 DVWA 應用程式。

**F5 XC 功能：** WAF（Web 應用程式防火牆）

| 順序 | 腳本 | 工具 | 描述 |
|---|---|---|---|
| 01 | `01-sqli.sh` | sqlmap, curl | 針對 Juice Shop 搜尋 API 和 DVWA SQLi 端點的 SQL 注入負載。執行 sqlmap 自動化掃描，然後發送 10 個直接 curl 注入負載。 |
| 02 | `02-xss.sh` | dalfox, curl | 針對 Juice Shop 和 DVWA XSS 反射型/儲存型端點的跨站腳本攻擊。執行 dalfox 自動化掃描，然後發送 12 個直接 XSS 負載，包括 script 標籤、事件處理器和 cookie 竊取。 |
| 03 | `03-command-injection.sh` | curl | 針對 DVWA exec 端點的作業系統命令注入負載。發送 12 個負載，包括管道鏈、分號、URL 編碼換行符和反向 shell 模式。 |
| 04 | `04-path-traversal.sh` | curl | 針對多個應用程式端點的目錄遍歷。跨 6 個端點路徑測試標準 `../` 遍歷、URL 編碼變體、雙重編碼變體、null-byte 變體和 Unicode 變體。 |
| 05 | `05-nikto-scan.sh` | nikto | 完整的 Web 伺服器弱點掃描，時間限制為 120 秒。產生觸發 WAF 特徵碼比對的多樣化 HTTP 請求。 |
| 06 | `06-nuclei-scan.sh` | nuclei | 基於範本的弱點掃描，涵蓋中等、高及嚴重嚴重性等級。速率限制為每秒 50 個請求。 |

**預期 F5 XC 行為：** WAF 應阻擋或標記包含 SQL 注入、XSS、命令注入和路徑遍歷負載的請求。安全事件儀表板將顯示每個被阻擋請求的違規類別、特徵碼 ID 和請求詳細資訊。
