每個流量套件都是 `/opt/traffic-generator/suites/` 下的一個目錄，其中包含由 `runner.sh` 按順序執行的編號 shell 腳本。每個腳本接受目標 FQDN 作為第一個參數，並將輸出寫入標準輸出和結果目錄。

## api-attacks

**腳本數量：** 4 | **預估持續時間：** 10-17 分鐘

透過 F5 XC 對 VAmPI 進行 OWASP API 安全 Top 10 測試。註冊測試使用者、取得認證 token，然後測試 BOLA、暴力破解認證、過度資料暴露、功能層級授權失效、SSRF 模式、對 REST API 端點的 SQL 注入、隱藏參數探索，以及端點模糊測試。目標為 VAmPI 應用程式。

**F5 XC 功能：** API 安全 / API 探索

| 順序 | 腳本 | 工具 | 描述 |
|---|---|---|---|
| 01 | `01-vampi-owasp-top10.sh` | curl, jq | 全面的 OWASP API Top 10 測試套件。註冊測試使用者、取得認證 token，然後測試 BOLA（存取其他使用者的資料）、暴力破解認證、過度資料暴露、功能層級授權失效、SSRF 模式，以及安全配置錯誤探測。 |
| 02 | `02-sqlmap-api.sh` | sqlmap | 對 VAmPI REST API 端點的 SQL 注入：使用者查詢、登入（POST）和註冊（POST），使用 JSON 酬載。 |
| 03 | `03-arjun-param-discovery.sh` | arjun | 跨 Juice Shop、DVWA 和 VAmPI 端點的隱藏參數探索。發現可能接受注入酬載的未記錄查詢參數。 |
| 04 | `04-ffuf-api-fuzz.sh` | ffuf | 兩階段模糊測試：使用常見 API 路徑字典（api、swagger、graphql、admin、.env、.git）進行端點探索，然後對關鍵端點進行 HTTP 方法模糊測試（GET、POST、PUT、DELETE、PATCH、OPTIONS、TRACE、PROPFIND）。 |

**預期 F5 XC 行為：** API 探索應能對應 VAmPI API 端點並識別異常請求模式。API 安全策略應標記未授權的存取嘗試、JSON 主體中的注入酬載，以及列舉模式。F5 XC 中的 API 清單將顯示已發現的端點及請求計數。

## bot-simulation

**腳本數量：** 4 | **預估持續時間：** 5-10 分鐘

透過 F5 XC 進行自動化瀏覽器活動和類機器人爬取模式。包括無頭 Chrome、帶隱匿外掛的 Puppeteer、快速爬取和自動化表單互動。目標為所有應用程式。

**F5 XC 功能：** Bot Defense

| 順序 | 腳本 | 工具 | 描述 |
|---|---|---|---|
| 01 | `01-headless-chrome.sh` | playwright | 無頭 Chromium 瀏覽：載入目標、在應用程式路徑之間導覽、擷取頁面標題和內容、截取螢幕擷圖。產生基於瀏覽器的流量但不使用隱匿外掛（可被偵測為自動化）。 |
| 02 | `02-puppeteer-stealth.sh` | puppeteer, puppeteer-extra-plugin-stealth | 使用 puppeteer-extra 搭配隱匿外掛進行隱匿瀏覽。透過修補 navigator 屬性、WebGL 和其他瀏覽器指紋來嘗試規避機器人偵測。 |
| 03 | `03-rapid-crawl.sh` | curl, wget | 使用 curl 和 wget 快速連續發送高頻率 HTTP 請求。模擬爬蟲行為，請求之間無延遲並進行積極的 User-Agent 輪換。 |
| 04 | `04-form-interaction.sh` | playwright | 使用 Playwright 的表單互動 API 對 DVWA 登入、Juice Shop 註冊和 VAmPI API 端點進行自動化表單填寫和提交。 |

**預期 F5 XC 行為：** Bot Defense 應根據瀏覽器指紋分析、請求時序和 JavaScript 挑戰結果將流量分類為自動化流量。無隱匿的無頭 Chrome 應立即被偵測到。隱匿 Puppeteer 測試進階規避技術是否能繞過偵測。Bot Defense 儀表板將顯示機器人分類類別和緩解動作。

## demoapp-attacks

**腳本數量：** 2 | **預估持續時間：** 2-3 分鐘

針對 F5 DemoApp 內建 WAF 測試端點（`/WAF/SQL`、`/WAF/XSS`、`/WAF/DIR`）的定向攻擊酬載。直接向設計用於展示 WAF 阻擋行為的端點發送 SQL 注入、XSS 和路徑遍歷酬載。當目標為 DemoApp 內容伺服器而非標準源站應用程式時，使用此套件。

**F5 XC 功能：** WAF（Web 應用程式防火牆）

| 順序 | 腳本 | 工具 | 描述 |
|---|---|---|---|
| 01 | `01-sqli-waf-endpoint.sh` | curl, python3 | 對 `/WAF/SQL?age=` 發送 15 個 SQL 注入酬載，包括 UNION SELECT、DROP TABLE、WAITFOR DELAY、xp_cmdshell、基於註解的繞過和布林邏輯酬載。 |
| 02 | `02-xss-waf-endpoint.sh` | curl, python3 | 對 `/WAF/XSS?update=` 發送 18 個 XSS 酬載，包括 script 標籤、事件處理器（onerror、onload、ontoggle、onfocus）、base64 eval、style expression、多型態酬載和 cookie 竊取。 |

**預期 F5 XC 行為：** WAF 應阻擋包含 SQL 注入和 XSS 酬載的請求，並返回阻擋頁面。安全事件儀表板將顯示每個被阻擋請求的違規類別和特徵碼匹配。

**F5 XC 功能：** Client-Side Defense (CSD)

| 順序 | 腳本 | 工具 | 描述 |

## ssl-scanning

**腳本數量：** 3 | **預估持續時間：** 3-5 分鐘

F5 XC 負載平衡器 HTTPS 端點的 TLS/SSL 配置分析。列舉加密套件、協定版本、憑證詳細資訊，並檢查已知的 TLS 弱點。目標為 F5 XC 負載平衡器端點。

**F5 XC 功能：** WAF（資訊性）

| 順序 | 腳本 | 工具 | 描述 |
|---|---|---|---|
| 01 | `01-sslscan.sh` | sslscan | 列舉支援的加密套件、協定版本和憑證詳細資訊。 |
| 02 | `02-sslyze.sh` | sslyze | 全面的 TLS 分析，包括憑證驗證、加密套件排序和弱點檢查（Heartbleed、ROBOT 等）。 |
| 03 | `03-testssl.sh` | testssl.sh | 使用 testssl.sh 框架進行完整的 TLS 評估。測試協定支援、加密偏好、標頭分析和已知弱點。 |

**預期 F5 XC 行為：** SSL 掃描會產生許多具有異常加密套件提議的 TLS 握手。雖然 F5 XC 通常不會阻擋這些流量，但流量模式在存取日誌中是可見的。其主要價值在於驗證 F5 XC 的 TLS 配置是否符合安全最佳實務。

## traffic-generation

**腳本數量：** 4 | **預估持續時間：** 5-10 分鐘（可配置）

用於基準測量和負載測試的大量合法 HTTP 流量。在 F5 XC 分析中建立正常流量模式，以便與攻擊套件流量進行比較。目標為所有應用程式。

**F5 XC 功能：** 全部（基準比較）

| 順序 | 腳本 | 工具 | 描述 |
|---|---|---|---|
| 01 | `01-baseline-http.sh` | curl | 使用標準瀏覽器 User-Agent 對所有應用程式路徑發送循序 HTTP GET 請求。在 F5 XC 分析中建立正常流量基準。 |
| 02 | `02-concurrent-load.sh` | curl | 使用 curl 搭配多個並行連線發送並行 HTTP 請求。測試負載處理能力並建立吞吐量基準。 |
| 03 | `03-mixed-methods.sh` | curl | 對 API 端點使用混合 HTTP 方法（GET、POST、PUT、DELETE）搭配有效酬載。產生正常的 API 流量模式，以便與攻擊流量進行比較。 |
| 04 | `04-user-journey.sh` | playwright | 模擬真實的使用者旅程：使用具有標準瀏覽器設定的 Playwright 瀏覽頁面、搜尋產品、填寫表單，以及在應用程式之間導覽。 |

**預期 F5 XC 行為：** 此套件中的所有請求應在不觸發 WAF 或 Bot Defense 干預的情況下通過。使用此套件在分析儀表板中產生「乾淨」流量，然後與攻擊套件流量進行比較，以展示合法與惡意請求模式之間的差異。

## waf-encoding-evasion

**腳本數量：** 7 | **預估持續時間：** 5-10 分鐘

多層編碼規避攻擊，設計用於測試 WAF 是否在檢查前正確地正規化和解碼酬載。涵蓋單重、雙重和三重 URL 編碼；HTML 實體編碼（十進位、十六進位、具名、零填充）；Unicode/UTF-8 插入（零寬空格、BOM、全形字元、軟連字號、過長序列）；混合/巢狀多層編碼（URL 編碼的 HTML 實體、實體內的雙重 URL、三層堆疊）；null byte 注入；IIS %uXXXX 編碼；大小寫操縱；分塊傳輸編碼主體分割；Base64 參數規避；以及標頭/cookie 編碼酬載。目標為任何應用程式。

**F5 XC 功能：** WAF（編碼正規化深度）

| 順序 | 腳本 | 工具 | 描述 |
|---|---|---|---|
| 01 | `01-url-encoding.sh` | curl | 跨多個端點路徑對 SQLi、XSS 和路徑遍歷酬載進行單重、雙重和三重 URL 編碼。測試 WAF 是否遞迴解碼 URL 參數。 |
| 02 | `02-html-entity-encoding.sh` | curl, python3 | 在 GET 參數和 POST 主體中對攻擊酬載進行十進位、十六進位、具名和零填充 HTML 實體編碼。包括透過 `&#123;&#123;7*7&#125;&#125;` 的範本注入。 |
| 03 | `03-unicode-utf8-evasion.sh` | curl | 在 SQL 關鍵字和 XSS 標籤中插入零寬空格（U+200B）、BOM（U+FEFF）、全形字元、軟連字號和 ZWNJ。測試過長 UTF-8 序列（2 位元組和 3 位元組）。 |
| 04 | `04-mixed-nested-encoding.sh` | curl | 多層酬載：URL 編碼的 HTML 實體、實體內的雙重 URL、三層堆疊、帶有 Unicode 跳脫的 JSON 主體（`<`），以及使用者特定的 `{{7*7}}` 範本注入模式。 |
| 05 | `05-null-byte-iis-base64.sh` | curl | 用於副檔名繞過的 null byte 注入（`%00`）、IIS `%uXXXX` Unicode 編碼，以及帶有解碼旗標的 URL 參數中的 Base64 編碼酬載。 |
| 06 | `06-case-chunked-evasion.sh` | curl | 大小寫隨機化搭配編碼（`%3CsCrIpT%3E`）、分塊傳輸編碼將 SQL/XSS 關鍵字分割跨 HTTP 區塊、Content-Length/Transfer-Encoding 衝突探測，以及替代路徑分隔符號（`%5c`、`%c0%af`）。 |

**預期 F5 XC 行為：** 正確配置的 WAF 應在模式匹配之前正規化所有編碼層。單重編碼的酬載應立即被阻擋。雙重和三重編碼測試揭示 WAF 解碼管線的深度。Unicode 插入測試揭示在關鍵字匹配之前是否會移除不可見字元。當編碼變體通過但純文字等效物被阻擋的結果，表示 WAF 策略中存在正規化缺口。

## web-app-attacks

**腳本數量：** 6 | **預估持續時間：** 12-20 分鐘

透過 F5 XC 對 Juice Shop 和 DVWA 進行 OWASP Top 10 Web 應用程式弱點測試。包括 SQL 注入、XSS、命令注入、路徑遍歷、Nikto 掃描和基於 Nuclei 範本的掃描。目標為 Juice Shop 和 DVWA 應用程式。

**F5 XC 功能：** WAF（Web 應用程式防火牆）

| 順序 | 腳本 | 工具 | 描述 |
|---|---|---|---|
| 01 | `01-sqli.sh` | sqlmap, curl | 對 Juice Shop 搜尋 API 和 DVWA SQLi 端點的 SQL 注入酬載。執行 sqlmap 自動掃描，然後發送 10 個直接的 curl 注入酬載。 |
| 02 | `02-xss.sh` | dalfox, curl | 對 Juice Shop 和 DVWA XSS 反射型/儲存型端點的跨站腳本攻擊。執行 dalfox 自動掃描，然後發送 12 個直接 XSS 酬載，包括 script 標籤、事件處理器和 cookie 竊取。 |
| 03 | `03-command-injection.sh` | curl | 對 DVWA exec 端點的作業系統命令注入酬載。發送 12 個酬載，包括管道鏈、分號、URL 編碼換行符和反向 shell 模式。 |
| 04 | `04-path-traversal.sh` | curl | 跨多個應用程式端點的目錄遍歷。跨 6 個端點路徑測試標準 `../` 遍歷、URL 編碼變體、雙重編碼變體、null byte 變體和 Unicode 變體。 |
| 05 | `05-nikto-scan.sh` | nikto | 具有 120 秒時間限制的完整 Web 伺服器弱點掃描。產生觸發 WAF 特徵碼匹配的多樣化 HTTP 請求。 |
| 06 | `06-nuclei-scan.sh` | nuclei | 基於範本的弱點掃描，涵蓋中等、高和嚴重嚴重性等級。速率限制為每秒 50 個請求。 |

**預期 F5 XC 行為：** WAF 應阻擋或標記包含 SQL 注入、XSS、命令注入和路徑遍歷酬載的請求。安全事件儀表板將顯示每個被阻擋請求的違規類別、特徵碼 ID 和請求詳細資訊。
