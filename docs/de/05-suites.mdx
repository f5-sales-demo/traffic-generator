Jede Traffic Suite ist ein Verzeichnis unter `/opt/traffic-generator/suites/`, das nummerierte Shell-Skripte enthält, die von `runner.sh` der Reihe nach ausgeführt werden. Jedes Skript akzeptiert den Ziel-FQDN als erstes Argument und schreibt die Ausgabe sowohl nach stdout als auch in das Ergebnisverzeichnis.

## api-attacks

**Skripte:** 4 | **Geschätzte Dauer:** 10-17 Minuten

OWASP API Security Top 10 Tests gegen VAmPI über F5 XC. Registriert einen Testbenutzer, erhält ein Auth-Token und testet dann BOLA, Brute-Force-Authentifizierung, übermäßige Datenexposition, fehlerhafte Autorisierung auf Funktionsebene, SSRF-Muster, SQL-Injection gegen REST-API-Endpunkte, Erkennung versteckter Parameter und Endpunkt-Fuzzing. Zielt auf die VAmPI-Anwendung ab.

**F5 XC Feature:** API Security / API Discovery

| Reihenfolge | Skript | Tools | Beschreibung |
|---|---|---|---|
| 01 | `01-vampi-owasp-top10.sh` | curl, jq | Umfassende OWASP API Top 10 Testsuite. Registriert einen Testbenutzer, erhält ein Auth-Token und testet dann BOLA (Zugriff auf Daten anderer Benutzer), Brute-Force-Authentifizierung, übermäßige Datenexposition, fehlerhafte Autorisierung auf Funktionsebene, SSRF-Muster und Sicherheitsfehlkonfigurationsprüfung. |
| 02 | `02-sqlmap-api.sh` | sqlmap | SQL-Injection gegen VAmPI REST-API-Endpunkte: Benutzer-Lookup, Login (POST) und Registrierung (POST) mit JSON-Payloads. |
| 03 | `03-arjun-param-discovery.sh` | arjun | Erkennung versteckter Parameter über Juice Shop, DVWA und VAmPI-Endpunkte hinweg. Entdeckt undokumentierte Query-Parameter, die möglicherweise Injection-Payloads akzeptieren. |
| 04 | `04-ffuf-api-fuzz.sh` | ffuf | Zweiphasen-Fuzzing: Endpunkt-Erkennung mittels einer Wortliste gängiger API-Pfade (api, swagger, graphql, admin, .env, .git) über alle Anwendungspräfixe hinweg, dann HTTP-Methoden-Fuzzing (GET, POST, PUT, DELETE, PATCH, OPTIONS, TRACE, PROPFIND) gegen Schlüsselendpunkte. |

**Erwartetes F5 XC Verhalten:** API Discovery sollte die VAmPI-API-Endpunkte abbilden und ungewöhnliche Anfragemuster identifizieren. API-Security-Richtlinien sollten unautorisierte Zugriffsversuche, Injection-Payloads in JSON-Bodies und Enumerationsmuster kennzeichnen. Das API-Inventar in F5 XC zeigt entdeckte Endpunkte mit Anfragezählern an.

## waf-encoding-evasion

**Skripte:** 7 | **Geschätzte Dauer:** 5-10 Minuten

Mehrschichtige Encoding-Evasion-Angriffe, die testen sollen, ob eine WAF Payloads vor der Inspektion ordnungsgemäß normalisiert und dekodiert. Deckt einfache, doppelte und dreifache URL-Kodierung ab; HTML-Entity-Kodierung (dezimal, hexadezimal, benannt, null-gepolstert); Unicode/UTF-8-Einfügung (Zero-Width Space, BOM, Fullwidth-Zeichen, Soft Hyphen, überlange Sequenzen); gemischte/verschachtelte mehrschichtige Kodierung (URL-kodierte HTML-Entities, doppelte URL innerhalb von Entities, 3-Schicht-Stapel); Null-Byte-Injection; IIS %uXXXX-Kodierung; Groß-/Kleinschreibungsmanipulation; Chunked Transfer Encoding Body-Splitting; Base64-Parameter-Evasion; und Header/Cookie-kodierte Payloads. Zielt auf jede Anwendung ab.

**F5 XC Feature:** WAF (Encoding-Normalisierungstiefe)

| Reihenfolge | Skript | Tools | Beschreibung |
|---|---|---|---|
| 01 | `01-url-encoding.sh` | curl | Einfache, doppelte und dreifache URL-Kodierung von SQLi-, XSS- und Path-Traversal-Payloads über mehrere Endpunktpfade hinweg. Testet, ob die WAF URL-Parameter rekursiv dekodiert. |
| 02 | `02-html-entity-encoding.sh` | curl, python3 | Dezimale, hexadezimale, benannte und null-gepolsterte HTML-Entity-Kodierung von Angriffs-Payloads sowohl in GET-Parametern als auch in POST-Bodies. Umfasst Template-Injection über `&#123;&#123;7*7&#125;&#125;`. |
| 03 | `03-unicode-utf8-evasion.sh` | curl | Zero-Width Space (U+200B), BOM (U+FEFF), Fullwidth-Zeichen, Soft Hyphen und ZWNJ-Einfügung in SQL-Schlüsselwörter und XSS-Tags. Testet überlange UTF-8-Sequenzen (2-Byte und 3-Byte). |
| 04 | `04-mixed-nested-encoding.sh` | curl | Mehrschichtige Payloads: URL-kodierte HTML-Entities, doppelte URL innerhalb von Entities, dreischichtige Stapel, JSON-Body mit Unicode-Escapes (`<`) und das spezifische `{{7*7}}`-Template-Injection-Muster des Benutzers. |
| 05 | `05-null-byte-iis-base64.sh` | curl | Null-Byte-Injection (`%00`) für Extension-Bypass, IIS `%uXXXX` Unicode-Kodierung und Base64-kodierte Payloads in URL-Parametern mit Decode-Flags. |
| 06 | `06-case-chunked-evasion.sh` | curl | Groß-/Kleinschreibungsrandomisierung kombiniert mit Kodierung (`%3CsCrIpT%3E`), Chunked Transfer Encoding zum Aufteilen von SQL/XSS-Schlüsselwörtern über HTTP-Chunks, Content-Length/Transfer-Encoding-Konfliktprüfungen und alternative Pfadtrennzeichen (`%5c`, `%c0%af`). |
| 07 | `07-header-cookie-evasion.sh` | curl | Kodierte Angriffs-Payloads, die über Cookie-, Referer-, User-Agent-, X-Forwarded-For- und X-Original-URL-Header eingeschleust werden. Testet, ob die WAF alle HTTP-Header-Felder inspiziert und dekodiert, nicht nur Query-Parameter und POST-Bodies. |

**Erwartetes F5 XC Verhalten:** Eine ordnungsgemäß konfigurierte WAF sollte alle Encoding-Schichten vor dem Pattern-Matching normalisieren. Einfach kodierte Payloads sollten sofort blockiert werden. Tests mit doppelter und dreifacher Kodierung offenbaren die Tiefe der WAF-Dekodierungspipeline. Unicode-Einfügungstests zeigen, ob unsichtbare Zeichen vor dem Schlüsselwortabgleich entfernt werden. Ergebnisse, bei denen kodierte Varianten durchgelassen werden, während Klartext-Äquivalente blockiert werden, deuten auf Normalisierungslücken in der WAF-Richtlinie hin.
