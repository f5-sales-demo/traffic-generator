Jede Traffic-Suite ist ein Verzeichnis unter `/opt/traffic-generator/suites/`, das nummerierte Shell-Skripte enthält, die von `runner.sh` der Reihe nach ausgeführt werden. Jedes Skript akzeptiert den Ziel-FQDN als erstes Argument und schreibt die Ausgabe sowohl auf stdout als auch in das Ergebnisverzeichnis.

## api-attacks

**Skripte:** 4 | **Geschätzte Dauer:** 10-17 Minuten

OWASP API Security Top 10-Tests gegen VAmPI über F5 XC. Registriert einen Testbenutzer, erhält ein Auth-Token und testet anschließend BOLA, Brute-Force-Authentifizierung, übermäßige Datenfreigabe, fehlerhafte Autorisierung auf Funktionsebene, SSRF-Muster, SQL-Injection gegen REST-API-Endpunkte, Erkennung versteckter Parameter und Endpunkt-Fuzzing. Zielt auf die VAmPI-Anwendung ab.

**F5 XC Feature:** API Security / API Discovery

| Reihenfolge | Skript | Tools | Beschreibung |
|---|---|---|---|
| 01 | `01-vampi-owasp-top10.sh` | curl, jq | Umfassende OWASP API Top 10-Testsuite. Registriert einen Testbenutzer, erhält ein Auth-Token und testet anschließend BOLA (Zugriff auf Daten anderer Benutzer), Brute-Force-Authentifizierung, übermäßige Datenfreigabe, fehlerhafte Autorisierung auf Funktionsebene, SSRF-Muster und Sicherheitsfehlkonfigurations-Probing. |
| 02 | `02-sqlmap-api.sh` | sqlmap | SQL-Injection gegen VAmPI REST-API-Endpunkte: Benutzersuche, Login (POST) und Registrierung (POST) mit JSON-Payloads. |
| 03 | `03-arjun-param-discovery.sh` | arjun | Erkennung versteckter Parameter über Juice Shop-, DVWA- und VAmPI-Endpunkte. Entdeckt undokumentierte Query-Parameter, die möglicherweise Injection-Payloads akzeptieren. |
| 04 | `04-ffuf-api-fuzz.sh` | ffuf | Zweiphasiges Fuzzing: Endpunkt-Erkennung mittels einer Wortliste gängiger API-Pfade (api, swagger, graphql, admin, .env, .git) über alle Anwendungspräfixe, gefolgt von HTTP-Methoden-Fuzzing (GET, POST, PUT, DELETE, PATCH, OPTIONS, TRACE, PROPFIND) gegen Schlüssel-Endpunkte. |

**Erwartetes F5 XC Verhalten:** API Discovery sollte die VAmPI-API-Endpunkte kartieren und ungewöhnliche Anfragemuster identifizieren. API Security-Richtlinien sollten unautorisierte Zugriffsversuche, Injection-Payloads in JSON-Bodies und Enumerationsmuster kennzeichnen. Das API-Inventar in F5 XC zeigt entdeckte Endpunkte mit Anfragenzählern an.

## waf-encoding-evasion

**Skripte:** 7 | **Geschätzte Dauer:** 5-10 Minuten

Mehrschichtige Encoding-Evasion-Angriffe, die testen, ob eine WAF Payloads vor der Inspektion ordnungsgemäß normalisiert und dekodiert. Deckt einfaches, doppeltes und dreifaches URL-Encoding ab; HTML-Entity-Encoding (dezimal, hexadezimal, benannt, mit Nullen aufgefüllt); Unicode/UTF-8-Einfügung (Zero-Width-Space, BOM, Fullwidth-Zeichen, Soft Hyphen, Overlong-Sequenzen); gemischtes/verschachteltes mehrschichtiges Encoding (URL-kodierte HTML-Entities, doppelt-URL innerhalb von Entities, 3-Schicht-Stacks); Null-Byte-Injection; IIS %uXXXX-Encoding; Groß-/Kleinschreibungsmanipulation; Chunked-Transfer-Encoding-Body-Splitting; Base64-Parameter-Evasion; sowie Header/Cookie-kodierte Payloads. Zielt auf jede Anwendung ab.

**F5 XC Feature:** WAF (Encoding-Normalisierungstiefe)

| Reihenfolge | Skript | Tools | Beschreibung |
|---|---|---|---|
| 01 | `01-url-encoding.sh` | curl | Einfaches, doppeltes und dreifaches URL-Encoding von SQLi-, XSS- und Path-Traversal-Payloads über mehrere Endpunktpfade. Testet, ob die WAF URL-Parameter rekursiv dekodiert. |
| 02 | `02-html-entity-encoding.sh` | curl, python3 | Dezimales, hexadezimales, benanntes und mit Nullen aufgefülltes HTML-Entity-Encoding von Angriffs-Payloads in GET-Parametern und POST-Bodies. Umfasst Template-Injection über `&#123;&#123;7*7&#125;&#125;`. |
| 03 | `03-unicode-utf8-evasion.sh` | curl | Zero-Width-Space (U+200B), BOM (U+FEFF), Fullwidth-Zeichen, Soft Hyphen und ZWNJ-Einfügung in SQL-Schlüsselwörter und XSS-Tags. Testet Overlong-UTF-8-Sequenzen (2-Byte und 3-Byte). |
| 04 | `04-mixed-nested-encoding.sh` | curl | Mehrschichtige Payloads: URL-kodierte HTML-Entities, doppelt-URL innerhalb von Entities, dreischichtige Stacks, JSON-Body mit Unicode-Escapes (`<`) und das benutzerspezifische `{{7*7}}`-Template-Injection-Muster. |
| 05 | `05-null-byte-iis-base64.sh` | curl | Null-Byte-Injection (`%00`) für Extension-Bypass, IIS `%uXXXX` Unicode-Encoding und Base64-kodierte Payloads in URL-Parametern mit Decode-Flags. |
| 06 | `06-case-chunked-evasion.sh` | curl | Groß-/Kleinschreibungsrandomisierung kombiniert mit Encoding (`%3CsCrIpT%3E`), Chunked-Transfer-Encoding zum Aufteilen von SQL/XSS-Schlüsselwörtern über HTTP-Chunks, Content-Length/Transfer-Encoding-Konfliktproben und alternative Pfadtrennzeichen (`%5c`, `%c0%af`). |
| 07 | `07-header-cookie-evasion.sh` | curl | Kodierte Angriffs-Payloads, die über Cookie-, Referer-, User-Agent-, X-Forwarded-For- und X-Original-URL-Header injiziert werden. Testet, ob die WAF alle HTTP-Header-Felder inspiziert und dekodiert, nicht nur Query-Parameter und POST-Bodies. |

**Erwartetes F5 XC Verhalten:** Eine ordnungsgemäß konfigurierte WAF sollte alle Encoding-Schichten vor dem Pattern-Matching normalisieren. Einfach kodierte Payloads sollten sofort blockiert werden. Doppelt und dreifach kodierte Tests offenbaren die Tiefe der WAF-Dekodierungspipeline. Unicode-Einfügungstests zeigen, ob unsichtbare Zeichen vor dem Schlüsselwort-Matching entfernt werden. Ergebnisse, bei denen kodierte Varianten durchkommen, während Klartext-Äquivalente blockiert werden, deuten auf Normalisierungslücken in der WAF-Richtlinie hin.
