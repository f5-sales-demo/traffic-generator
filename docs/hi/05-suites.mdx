## api-attacks

**स्क्रिप्ट्स:** 4 | **अनुमानित अवधि:** 10-17 मिनट

F5 XC के माध्यम से VAmPI के विरुद्ध OWASP API सुरक्षा Top 10 परीक्षण। एक टेस्ट यूज़र पंजीकृत करता है, ऑथ टोकन प्राप्त करता है, फिर BOLA, ब्रूट-फोर्स प्रमाणीकरण, अत्यधिक डेटा एक्सपोज़र, टूटा हुआ फंक्शन-लेवल प्राधिकरण, SSRF पैटर्न, REST API एंडपॉइंट्स के विरुद्ध SQL इंजेक्शन, छिपे हुए पैरामीटर खोज, और एंडपॉइंट फ़ज़िंग का परीक्षण करता है। VAmPI एप्लिकेशन को लक्षित करता है।

**F5 XC फ़ीचर:** API Security / API Discovery

| क्रम | स्क्रिप्ट | टूल्स | विवरण |
|---|---|---|---|
| 01 | `01-vampi-owasp-top10.sh` | curl, jq | व्यापक OWASP API Top 10 टेस्ट सुइट। एक टेस्ट यूज़र पंजीकृत करता है, ऑथ टोकन प्राप्त करता है, फिर BOLA (अन्य यूज़र्स के डेटा तक पहुँच), ब्रूट-फोर्स प्रमाणीकरण, अत्यधिक डेटा एक्सपोज़र, टूटा हुआ फंक्शन-लेवल प्राधिकरण, SSRF पैटर्न, और सुरक्षा मिसकॉन्फ़िगरेशन प्रोबिंग का परीक्षण करता है। |
| 02 | `02-sqlmap-api.sh` | sqlmap | VAmPI REST API एंडपॉइंट्स के विरुद्ध SQL इंजेक्शन: यूज़र लुकअप, लॉगिन (POST), और रजिस्टर (POST) JSON पेलोड्स के साथ। |
| 03 | `03-arjun-param-discovery.sh` | arjun | Juice Shop, DVWA, और VAmPI एंडपॉइंट्स पर छिपे हुए पैरामीटर खोज। अप्रलेखित क्वेरी पैरामीटर्स की खोज करता है जो इंजेक्शन पेलोड्स स्वीकार कर सकते हैं। |
| 04 | `04-ffuf-api-fuzz.sh` | ffuf | दो-चरणीय फ़ज़िंग: सामान्य API पथों (api, swagger, graphql, admin, .env, .git) की वर्डलिस्ट का उपयोग करके सभी एप्लिकेशन प्रीफ़िक्सों पर एंडपॉइंट खोज, फिर प्रमुख एंडपॉइंट्स के विरुद्ध HTTP मेथड फ़ज़िंग (GET, POST, PUT, DELETE, PATCH, OPTIONS, TRACE, PROPFIND)। |

**अपेक्षित F5 XC व्यवहार:** API Discovery को VAmPI API एंडपॉइंट्स को मैप करना चाहिए और असामान्य अनुरोध पैटर्न की पहचान करनी चाहिए। API Security नीतियों को अनधिकृत पहुँच प्रयासों, JSON बॉडी में इंजेक्शन पेलोड्स, और एन्यूमरेशन पैटर्न को फ़्लैग करना चाहिए। F5 XC में API Inventory अनुरोध गणना के साथ खोजे गए एंडपॉइंट्स दिखाएगी।

## bot-simulation

**स्क्रिप्ट्स:** 4 | **अनुमानित अवधि:** 5-10 मिनट

F5 XC के माध्यम से स्वचालित ब्राउज़र गतिविधि और बॉट-जैसे क्रॉलिंग पैटर्न। इसमें हेडलेस Chrome, स्टेल्थ प्लगइन के साथ Puppeteer, तीव्र क्रॉलिंग, और स्वचालित फ़ॉर्म इंटरैक्शन शामिल है। सभी एप्लिकेशनों को लक्षित करता है।

**F5 XC फ़ीचर:** Bot Defense

| क्रम | स्क्रिप्ट | टूल्स | विवरण |
|---|---|---|---|
| 01 | `01-headless-chrome.sh` | playwright | हेडलेस Chromium ब्राउज़िंग: लक्ष्य लोड करता है, एप्लिकेशन पथों के बीच नेविगेट करता है, पेज शीर्षक और सामग्री निकालता है, स्क्रीनशॉट लेता है। स्टेल्थ प्लगइन के बिना ब्राउज़र-आधारित ट्रैफ़िक उत्पन्न करता है (ऑटोमेशन के रूप में पहचान योग्य)। |
| 02 | `02-puppeteer-stealth.sh` | puppeteer, puppeteer-extra-plugin-stealth | स्टेल्थ प्लगइन के साथ puppeteer-extra का उपयोग करके स्टेल्थ ब्राउज़िंग। navigator प्रॉपर्टीज़, WebGL, और अन्य ब्राउज़र फ़िंगरप्रिंट्स को पैच करके बॉट डिटेक्शन से बचने का प्रयास करता है। |
| 03 | `03-rapid-crawl.sh` | curl, wget | curl और wget का उपयोग करके तीव्र उत्तराधिकार में उच्च-आवृत्ति HTTP अनुरोध। अनुरोधों के बीच कोई विलंब नहीं और आक्रामक User-Agent रोटेशन के साथ स्क्रैपर व्यवहार का अनुकरण करता है। |
| 04 | `04-form-interaction.sh` | playwright | Playwright के फ़ॉर्म इंटरैक्शन API का उपयोग करके DVWA लॉगिन, Juice Shop पंजीकरण, और VAmPI API एंडपॉइंट्स के विरुद्ध स्वचालित फ़ॉर्म भरना और सबमिशन। |

**अपेक्षित F5 XC व्यवहार:** Bot Defense को ब्राउज़र फ़िंगरप्रिंट विश्लेषण, अनुरोध समय, और JavaScript चैलेंज परिणामों के आधार पर ट्रैफ़िक को स्वचालित के रूप में वर्गीकृत करना चाहिए। स्टेल्थ के बिना हेडलेस Chrome का तुरंत पता लगाया जाना चाहिए। स्टेल्थ Puppeteer यह परीक्षण करता है कि क्या उन्नत बचाव तकनीकें डिटेक्शन को बायपास करती हैं। Bot Defense डैशबोर्ड बॉट वर्गीकरण श्रेणियाँ और शमन कार्रवाइयाँ दिखाएगा।

## demoapp-attacks

**स्क्रिप्ट्स:** 2 | **अनुमानित अवधि:** 2-3 मिनट

F5 DemoApp के बिल्ट-इन WAF परीक्षण एंडपॉइंट्स (`/WAF/SQL`, `/WAF/XSS`, `/WAF/DIR`) के विरुद्ध लक्षित हमले के पेलोड्स। SQL इंजेक्शन, XSS, और पथ ट्रैवर्सल पेलोड्स को सीधे उन एंडपॉइंट्स पर भेजता है जो WAF ब्लॉकिंग व्यवहार प्रदर्शित करने के लिए डिज़ाइन किए गए हैं। जब आप मानक ऑरिजिन-सर्वर एप्लिकेशनों के बजाय DemoApp कंटेंट सर्वर को लक्षित कर रहे हों तो इस सुइट का उपयोग करें।

**F5 XC फ़ीचर:** WAF (Web Application Firewall)

| क्रम | स्क्रिप्ट | टूल्स | विवरण |
|---|---|---|---|
| 01 | `01-sqli-waf-endpoint.sh` | curl, python3 | `/WAF/SQL?age=` के विरुद्ध 15 SQL इंजेक्शन पेलोड्स जिनमें UNION SELECT, DROP TABLE, WAITFOR DELAY, xp_cmdshell, कमेंट-आधारित बायपास, और बूलियन लॉजिक पेलोड्स शामिल हैं। |
| 02 | `02-xss-waf-endpoint.sh` | curl, python3 | `/WAF/XSS?update=` के विरुद्ध 18 XSS पेलोड्स जिनमें स्क्रिप्ट टैग, इवेंट हैंडलर (onerror, onload, ontoggle, onfocus), base64 eval, स्टाइल एक्सप्रेशन, पॉलीग्लॉट पेलोड्स, और कुकी एक्सफ़िल्ट्रेशन शामिल हैं। |

**अपेक्षित F5 XC व्यवहार:** WAF को SQL इंजेक्शन और XSS पेलोड्स वाले अनुरोधों को अवरुद्ध करना चाहिए और एक ब्लॉक पेज लौटाना चाहिए। Security Events डैशबोर्ड प्रत्येक अवरुद्ध अनुरोध के लिए उल्लंघन श्रेणियाँ और सिग्नेचर मैच दिखाएगा।

## javascript-exploits

**स्क्रिप्ट्स:** 3 | **अनुमानित अवधि:** 3-5 मिनट

क्लाइंट-साइड स्क्रिप्ट इंजेक्शन और DOM मैनिपुलेशन पैटर्न जो Client-Side Defense डिटेक्शन को ट्रिगर करते हैं। इसमें Magecart-शैली स्किमर्स की नकल करने वाले इनलाइन स्क्रिप्ट इंजेक्शन, हेडलेस Chromium के माध्यम से DOM मैनिपुलेशन, और तृतीय-पक्ष स्क्रिप्ट सिमुलेशन शामिल है। CSD Demo एप्लिकेशन को लक्षित करता है।

**F5 XC फ़ीचर:** Client-Side Defense (CSD)

| क्रम | स्क्रिप्ट | टूल्स | विवरण |
|---|---|---|---|
| 01 | `01-inline-script-injection.sh` | curl | इनलाइन JavaScript पेलोड्स वाले अनुरोध भेजता है जो Magecart-शैली क्रेडिट कार्ड स्किमर्स और डेटा एक्सफ़िल्ट्रेशन स्क्रिप्ट्स की नकल करने के लिए डिज़ाइन किए गए हैं। |
| 02 | `02-dom-manipulation.sh` | playwright | CSD डेमो पेज लोड करने और DOM संशोधन इंजेक्ट करने के लिए हेडलेस Chromium का उपयोग करता है: स्क्रिप्ट टैग जोड़ना, फ़ॉर्म एक्शन संशोधित करना, और अदृश्य iframes सम्मिलित करना जो फ़ॉर्म डेटा एक्सफ़िल्ट्रेट करते हैं। |
| 03 | `03-third-party-script-sim.sh` | curl, playwright | बाहरी JavaScript स्रोतों को संदर्भित करने वाले अनुरोधों को इंजेक्ट करके और पेज संदर्भ में अनधिकृत स्क्रिप्ट्स लोड करने का प्रयास करके समझौता किए गए तृतीय-पक्ष स्क्रिप्ट्स का अनुकरण करता है। |

**अपेक्षित F5 XC व्यवहार:** Client-Side Defense को संरक्षित पेज में अनधिकृत स्क्रिप्ट संशोधनों का पता लगाना चाहिए। CSD डैशबोर्ड नए स्क्रिप्ट स्रोतों, संशोधित DOM तत्वों, और डेटा एक्सफ़िल्ट्रेशन प्रयासों के लिए अलर्ट दिखाएगा। यह CSD फ़ेज़ 2 कार्यक्षमता को मान्य करता है।

| 06 | `06-case-chunked-evasion.sh` | curl | एन्कोडिंग के साथ केस रैंडमाइज़ेशन (`%3CsCrIpT%3E`), HTTP चंक्स में SQL/XSS कीवर्ड्स को विभाजित करने के लिए चंक्ड ट्रांसफर एन्कोडिंग, Content-Length/Transfer-Encoding कॉन्फ़्लिक्ट प्रोब्स, और वैकल्पिक पथ सेपरेटर्स (`%5c`, `%c0%af`)। |
| 07 | `07-header-cookie-evasion.sh` | curl | Cookie, Referer, User-Agent, X-Forwarded-For, और X-Original-URL हेडर्स के माध्यम से इंजेक्ट किए गए एन्कोडेड अटैक पेलोड्स। परीक्षण करता है कि WAF केवल क्वेरी पैरामीटर्स और POST बॉडी ही नहीं बल्कि सभी HTTP हेडर फ़ील्ड्स का निरीक्षण और डिकोड करता है या नहीं। |

**अपेक्षित F5 XC व्यवहार:** एक ठीक से कॉन्फ़िगर किया गया WAF पैटर्न मैचिंग से पहले सभी एन्कोडिंग लेयर्स को नॉर्मलाइज़ करना चाहिए। सिंगल-एन्कोडेड पेलोड्स को तुरंत अवरुद्ध किया जाना चाहिए। डबल और ट्रिपल एन्कोडिंग परीक्षण WAF की डिकोड पाइपलाइन की गहराई को प्रकट करते हैं। Unicode इंसर्शन परीक्षण प्रकट करते हैं कि क्या कीवर्ड मैचिंग से पहले अदृश्य कैरेक्टर्स को हटाया जाता है। ऐसे परिणाम जहाँ एन्कोडेड वेरिएंट पास हो जाते हैं जबकि प्लेन-टेक्स्ट समकक्ष अवरुद्ध होते हैं, WAF नीति में नॉर्मलाइज़ेशन अंतराल इंगित करते हैं।

## web-app-attacks

**स्क्रिप्ट्स:** 6 | **अनुमानित अवधि:** 12-20 मिनट

F5 XC के माध्यम से Juice Shop और DVWA के विरुद्ध OWASP Top 10 वेब एप्लिकेशन भेद्यता परीक्षण। इसमें SQL इंजेक्शन, XSS, कमांड इंजेक्शन, पथ ट्रैवर्सल, Nikto स्कैनिंग, और Nuclei टेम्पलेट-आधारित स्कैनिंग शामिल है। Juice Shop और DVWA एप्लिकेशनों को लक्षित करता है।

**F5 XC फ़ीचर:** WAF (Web Application Firewall)

| क्रम | स्क्रिप्ट | टूल्स | विवरण |
|---|---|---|---|
| 01 | `01-sqli.sh` | sqlmap, curl | Juice Shop सर्च API और DVWA SQLi एंडपॉइंट के विरुद्ध SQL इंजेक्शन पेलोड्स। sqlmap स्वचालित स्कैन चलाता है फिर 10 डायरेक्ट curl-आधारित इंजेक्शन पेलोड्स भेजता है। |
| 02 | `02-xss.sh` | dalfox, curl | Juice Shop और DVWA XSS रिफ़्लेक्टेड/स्टोर्ड एंडपॉइंट्स के विरुद्ध क्रॉस-साइट स्क्रिप्टिंग। dalfox स्वचालित स्कैन चलाता है फिर स्क्रिप्ट टैग, इवेंट हैंडलर, और कुकी एक्सफ़िल्ट्रेशन सहित 12 डायरेक्ट XSS पेलोड्स भेजता है। |
| 03 | `03-command-injection.sh` | curl | DVWA exec एंडपॉइंट के विरुद्ध OS कमांड इंजेक्शन पेलोड्स। पाइप चेन, सेमीकोलन, URL-एन्कोडेड न्यूलाइन्स, और रिवर्स शेल पैटर्न सहित 12 पेलोड्स भेजता है। |
| 04 | `04-path-traversal.sh` | curl | एकाधिक एप्लिकेशन एंडपॉइंट्स के विरुद्ध डायरेक्टरी ट्रैवर्सल। 6 एंडपॉइंट पथों पर मानक `../` ट्रैवर्सल, URL-एन्कोडेड वेरिएंट, डबल-एन्कोडेड वेरिएंट, नल-बाइट वेरिएंट, और Unicode वेरिएंट का परीक्षण करता है। |
| 05 | `05-nikto-scan.sh` | nikto | 120-सेकंड समय सीमा के साथ पूर्ण वेब सर्वर भेद्यता स्कैन। विविध HTTP अनुरोध उत्पन्न करता है जो WAF सिग्नेचर मैचिंग को ट्रिगर करते हैं। |
| 06 | `06-nuclei-scan.sh` | nuclei | मध्यम, उच्च, और गंभीर गंभीरता स्तरों पर टेम्पलेट-आधारित भेद्यता स्कैनिंग। 50 अनुरोध/सेकंड तक रेट-लिमिटेड। |

**अपेक्षित F5 XC व्यवहार:** WAF को SQL इंजेक्शन, XSS, कमांड इंजेक्शन, और पथ ट्रैवर्सल पेलोड्स वाले अनुरोधों को अवरुद्ध या फ़्लैग करना चाहिए। Security Events डैशबोर्ड प्रत्येक अवरुद्ध अनुरोध के लिए उल्लंघन श्रेणियाँ, सिग्नेचर IDs, और अनुरोध विवरण दिखाएगा।
