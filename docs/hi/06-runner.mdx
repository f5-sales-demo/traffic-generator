## अवलोकन

`runner.sh` सूट ऑर्केस्ट्रेटर है जो एक सूट डायरेक्टरी में सभी क्रमांकित स्क्रिप्ट्स को क्रम में निष्पादित करता है, आउटपुट कैप्चर करता है, और रिजल्ट्स मेटाडेटा रिकॉर्ड करता है। यह VM पर `/opt/traffic-generator/suites/runner.sh` पर इंस्टॉल होता है।

## उपयोग

```bash
runner.sh <suite-name> [--dry-run]
```

**आर्गुमेंट्स:**

| आर्गुमेंट | आवश्यक | विवरण |
|---|---|---|
| `suite-name` | हाँ | सूट डायरेक्टरी का नाम (जैसे, `web-app-attacks`, `api-attacks`) |
| `--dry-run` | नहीं | बिना कोई स्क्रिप्ट चलाए यह प्रिंट करें कि क्या निष्पादित होगा |

**उदाहरण:**

```bash
# web-app-attacks सूट चलाएँ
/opt/traffic-generator/suites/runner.sh web-app-attacks

# api-attacks स्क्रिप्ट्स का पूर्वावलोकन करने के लिए ड्राई-रन
/opt/traffic-generator/suites/runner.sh api-attacks --dry-run

# बॉट सिमुलेशन चलाएँ
/opt/traffic-generator/suites/runner.sh bot-simulation
```

## कॉन्फ़िगरेशन

### config.env

रनर `/opt/traffic-generator/config.env` से कॉन्फ़िगरेशन पढ़ता है:

```bash
# Required: target FQDN (F5 XC load balancer domain)
TARGET_FQDN=demo.example.com

# Optional: direct origin IP for baseline testing (bypasses F5 XC)
TARGET_ORIGIN_IP=
```

यह फ़ाइल Terraform प्रोविज़निंग के दौरान `target_fqdn` और `target_origin_ip` वेरिएबल्स से स्वचालित रूप से लिखी जाती है। डिप्लॉयमेंट के बाद टारगेट बदलने के लिए, VM पर सीधे इस फ़ाइल को संपादित करें।

### एनवायरनमेंट वेरिएबल ओवरराइड

`config.env` में किसी भी वेरिएबल को सूट चलाने से पहले एक्सपोर्ट करके ओवरराइड किया जा सकता है:

```bash
# एक ही रन के लिए टारगेट FQDN ओवरराइड करें
TARGET_FQDN=staging.example.com /opt/traffic-generator/suites/runner.sh web-app-attacks

# export के माध्यम से ओवरराइड करें
export TARGET_FQDN=staging.example.com
/opt/traffic-generator/suites/runner.sh web-app-attacks
```

कॉन्फ़िग फ़ाइल पथ को भी ओवरराइड किया जा सकता है:

```bash
CONFIG_FILE=/tmp/my-config.env /opt/traffic-generator/suites/runner.sh web-app-attacks
```

## ड्राई-रन मोड

`--dry-run` फ़्लैग प्रत्येक स्क्रिप्ट को प्रिंट करता है जो निष्पादित होगी, बिना उसे चलाए:

```bash
/opt/traffic-generator/suites/runner.sh web-app-attacks --dry-run
```

आउटपुट:

```

[DRY-RUN] Would execute: /opt/traffic-generator/suites/web-app-attacks/01-sqli.sh demo.example.com
=== 02-xss.sh ===
[DRY-RUN] Would execute: /opt/traffic-generator/suites/web-app-attacks/02-xss.sh demo.example.com
=== 03-command-injection.sh ===
[DRY-RUN] Would execute: /opt/traffic-generator/suites/web-app-attacks/03-command-injection.sh demo.example.com
=== 04-path-traversal.sh ===
[DRY-RUN] Would execute: /opt/traffic-generator/suites/web-app-attacks/04-path-traversal.sh demo.example.com
=== 05-nikto-scan.sh ===
[DRY-RUN] Would execute: /opt/traffic-generator/suites/web-app-attacks/05-nikto-scan.sh demo.example.com
```

डिप्लॉयमेंट के बाद सूट संरचना सत्यापित करने या नए टारगेट के खिलाफ़ सूट चलाने से पहले ड्राई-रन का उपयोग करें।

## रिजल्ट्स डायरेक्टरी

प्रत्येक सूट रन एक टाइमस्टैम्प्ड रिजल्ट्स डायरेक्टरी बनाता है:

```
/opt/traffic-generator/results/<YYYYMMDD-HHMMSS>-<suite-name>/
```

**संरचना:**

```
/opt/traffic-generator/results/20260425-143000-web-app-attacks/
  meta.json
  01-sqli.sh.log
  02-xss.sh.log
  03-command-injection.sh.log
  04-path-traversal.sh.log
  05-nikto-scan.sh.log
  06-nuclei-scan.sh.log
```

प्रत्येक स्क्रिप्ट का stdout और stderr एक `.log` फ़ाइल में कैप्चर होता है जिसका नाम स्क्रिप्ट के नाम पर रखा जाता है।

### meta.json फॉर्मेट

`meta.json` फ़ाइल सूट निष्पादन मेटाडेटा रिकॉर्ड करती है:

```json
{
  "suite": "web-app-attacks",
  "target": "demo.example.com",
  "started": "2026-04-25T14:30:00Z",
  "completed": "2026-04-25T14:45:23Z",
  "status": "completed",
  "passed": 5,
  "failed": 1,
  "skipped": 0
}
```

| फ़ील्ड | विवरण |
|---|---|
| `suite` | सूट का नाम |
| `target` | रन के लिए उपयोग किया गया टारगेट FQDN |
| `started` | सूट शुरू होने का UTC टाइमस्टैम्प |
| `completed` | सूट समाप्त होने का UTC टाइमस्टैम्प |
| `status` | निष्पादन के दौरान `running`, पूर्ण होने पर `completed` |
| `passed` | कोड 0 के साथ बाहर निकलने वाली स्क्रिप्ट्स की संख्या |
| `failed` | नॉन-ज़ीरो कोड के साथ बाहर निकलने वाली स्क्रिप्ट्स की संख्या |
| `skipped` | छोड़ी गई स्क्रिप्ट्स की संख्या (निष्पादन योग्य नहीं या ड्राई-रन) |

<Aside type="tip">
किसी स्क्रिप्ट से नॉन-ज़ीरो एग्जिट कोड का मतलब यह नहीं है कि सूट विफल हो गया। कई सुरक्षा उपकरण जब कमजोरियाँ पाते हैं तो नॉन-ज़ीरो एग्जिट करते हैं, जो अटैक ट्रैफ़िक जनरेशन के दौरान अपेक्षित व्यवहार है।
</Aside>

## व्यक्तिगत स्क्रिप्ट्स चलाना

प्रत्येक स्क्रिप्ट को रनर के बिना स्टैंडअलोन रूप से निष्पादित किया जा सकता है:

```bash
# एक स्क्रिप्ट सीधे चलाएँ
/opt/traffic-generator/suites/web-app-attacks/01-sqli.sh demo.example.com

# एक विशिष्ट API अटैक चलाएँ
/opt/traffic-generator/suites/api-attacks/01-vampi-owasp-top10.sh demo.example.com
```

स्क्रिप्ट्स पहले पोज़िशनल आर्गुमेंट के रूप में टारगेट FQDN स्वीकार करती हैं। वे सीधे `config.env` नहीं पढ़ती -- केवल रनर ही ऐसा करता है। स्टैंडअलोन चलाते समय, आपको FQDN स्पष्ट रूप से पास करना होगा।

## सभी सूट्स को क्रम में चलाना

हर सूट को एक के बाद एक चलाने के लिए:

```bash
for suite in web-app-attacks api-attacks bot-simulation reconnaissance ssl-scanning javascript-exploits traffic-generation; do
  echo "========================================="
  echo "Starting suite: ${suite}"
  echo "========================================="
  /opt/traffic-generator/suites/runner.sh "$suite"
  echo ""
done
```

यह सभी F5 XC सुरक्षा सुविधाओं में व्यापक ट्रैफ़िक उत्पन्न करता है। टारगेट की प्रतिक्रिया समय के आधार पर पूरे रन में 45-80 मिनट लगने की उम्मीद करें।

## SSH के माध्यम से रिमोट निष्पादन

SSH सत्र बनाए रखे बिना अपनी स्थानीय मशीन से सूट्स चलाएँ:

```bash
TGEN_IP=$(terraform output -raw public_ip)

# एक सूट चलाएँ
ssh azureuser@${TGEN_IP} '/opt/traffic-generator/suites/runner.sh web-app-attacks'

# टारगेट ओवरराइड के साथ सूट चलाएँ
ssh azureuser@${TGEN_IP} 'TARGET_FQDN=staging.example.com /opt/traffic-generator/suites/runner.sh api-attacks'

# बैकग्राउंड में चलाएँ (डिस्कनेक्शन-सुरक्षित)
ssh azureuser@${TGEN_IP} 'nohup /opt/traffic-generator/suites/runner.sh web-app-attacks > /tmp/web-app-attacks.log 2>&1 &'

# बाद में रिजल्ट्स जाँचें
ssh azureuser@${TGEN_IP} 'ls -la /opt/traffic-generator/results/ | tail -5'
ssh azureuser@${TGEN_IP} 'cat /opt/traffic-generator/results/$(ls -t /opt/traffic-generator/results/ | head -1)/meta.json'
```

लंबे समय तक चलने वाले सूट्स के लिए, SSH डिस्कनेक्शन से प्रक्रिया को बंद होने से रोकने के लिए `nohup` या `tmux` का उपयोग करें:

```bash
ssh azureuser@${TGEN_IP} 'tmux new-session -d -s traffic "/opt/traffic-generator/suites/runner.sh reconnaissance"'

# प्रगति देखने के लिए बाद में फिर से जुड़ें
ssh -t azureuser@${TGEN_IP} 'tmux attach-session -t traffic'
```

## निष्पादन प्रवाह

<Steps>

1. **कॉन्फ़िगरेशन लोड करें** -- रनर `config.env` (या `CONFIG_FILE` ओवरराइड) पढ़ता है, फिर एनवायरनमेंट में `TARGET_FQDN` की जाँच करता है।

2. **सूट मान्य करें** -- पुष्टि करता है कि नामित सूट डायरेक्टरी सूट्स डायरेक्टरी के अंतर्गत मौजूद है। न मिलने पर उपलब्ध सूट्स की सूची दिखाता है।

3. **रिजल्ट्स डायरेक्टरी बनाएँ** -- `/opt/traffic-generator/results/<timestamp>-<suite>/` बनाता है और `status: running` के साथ प्रारंभिक `meta.json` लिखता है।

4. **स्क्रिप्ट्स निष्पादित करें** -- सूट डायरेक्टरी में `[0-9]*` से मेल खाने वाली फ़ाइलों को नाम के अनुसार क्रमबद्ध करके दोहराता है। गैर-निष्पादन योग्य फ़ाइलों को छोड़ देता है। `TARGET_FQDN` को पहले आर्गुमेंट के रूप में पास करता है। आउटपुट `<script-name>.log` में कैप्चर करता है।

5. **रिजल्ट्स रिकॉर्ड करें** -- पूर्णता टाइमस्टैम्प और पास/फेल/स्किप काउंट्स के साथ `meta.json` अपडेट करता है।

</Steps>
