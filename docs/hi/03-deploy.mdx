सभी Terraform फ़ाइलें [`terraform/`](https://github.com/f5xc-salesdemos/traffic-generator/tree/main/terraform) डायरेक्टरी में हैं। रिपॉजिटरी क्लोन करें और सीधे डिप्लॉय करें:

```bash
git clone https://github.com/f5xc-salesdemos/traffic-generator.git
cd traffic-generator/terraform
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars with your Azure subscription ID and target FQDN
```

## Terraform कॉन्फ़िगरेशन

### Terraform फ़ाइल संरचना

terraform डायरेक्टरी में Demo Resource Standard का पालन करने वाली 9 फ़ाइलें हैं:

- `versions.tf` -- Terraform और प्रोवाइडर संस्करण बाधाएँ (azurerm ~> 4.0, azuread ~> 3.0)
- `providers.tf` -- Azure RM और Azure AD प्रोवाइडर कॉन्फ़िगरेशन
- `data.tf` -- डिप्लॉयर ऑटो-रिज़ॉल्यूशन के लिए Azure AD डेटा सोर्स
- `locals.tf` -- डिप्लॉयर रिज़ॉल्यूशन, Azure Cloud Adoption Framework रिसोर्स नामकरण, मानक टैग
- `main.tf` -- रिसोर्स ग्रुप (नाम `rg-traffic-generator-{environment}-{deployer}`)
- `variables.tf` -- सभी इनपुट वेरिएबल (2 आवश्यक, 9 वैकल्पिक) Azure AD के माध्यम से डिप्लॉयर ऑटो-रिज़ॉल्यूशन के साथ
- `network.tf` -- VNet (10.201.0.0/16), सबनेट, पब्लिक IP, NSG (पोर्ट 22 SSH), NIC
- `vm.tf` -- templatefile() के माध्यम से cloud-init के साथ Ubuntu 24.04 VM
- `outputs.tf` -- 17 आउटपुट: सभी डेमो रिसोर्स द्वारा साझा किए गए 15 मानक आउटपुट (deployer, public_ip, private_ip, ssh_command, resource_group_name, vm_name, nsg_name, vnet_name, subnet_id, component, environment, resource_group_id, vm_id, nsg_id, location) और 2 कंपोनेंट-विशिष्ट आउटपुट (target_fqdn, status_check)

### प्रोवाइडर और वेरिएबल

`main.tf` Azure प्रोवाइडर को कॉन्फ़िगर करता है:

```hcl file=./_data/main.tf
```

`variables.tf` सभी कॉन्फ़िगर करने योग्य पैरामीटर परिभाषित करता है। `vm_size` का डिफ़ॉल्ट `Standard_F16s_v2` (16 vCPU कंप्यूट-ऑप्टिमाइज़्ड) है -- स्केलिंग मार्गदर्शन के लिए [VM साइज़िंग](#vm-साइज़िंग) देखें:

```hcl file=./_data/variables.tf
```

### नेटवर्क इन्फ्रास्ट्रक्चर

`network.tf` VNet, सबनेट, पब्लिक IP, NSG (पोर्ट 22 SSH), और NIC बनाता है:

```hcl file=./_data/network.tf
```

### Cloud-Init के साथ वर्चुअल मशीन

`vm.tf` Ubuntu 24.04 VM बनाता है और टार्गेट तथा टूलिंग वेरिएबल के साथ cloud-init टेम्पलेट पास करता है:

```hcl file=./_data/vm.tf
```

### Cloud-Init प्रोविज़निंग

`cloud-init.yaml` प्रोडक्शन-मान्य कॉन्फ़िगरेशन है जो 48+ घंटों के निरंतर लोड टेस्टिंग के तहत सत्यापित है, जिसमें 4 राउंड ऑरिजिन टॉर्चर टेस्टिंग शामिल है। यह रिट्राई लॉजिक, `/var/log/cloud-init-progress.log` में प्रगति लॉगिंग, 9 प्रोविज़निंग चरणों, और सशर्त स्टेटस रिपोर्टिंग (`status.json` में ready/degraded) के साथ एक हेल्पर लाइब्रेरी का उपयोग करता है। यह प्रोविज़न करता है:

- **कर्नेल ट्यूनिंग** -- `somaxconn=131072`, `tcp_max_tw_buckets=4000000`, `tcp_fin_timeout=5`, 64 MiB सॉकेट बफ़र, `file-max=4194304`
- **NIC ऑप्टिमाइज़ेशन** -- सभी कोर में RPS/RFS, ethtool के माध्यम से अधिकतम रिंग बफ़र, THP=always
- **APT पैकेज** -- nikto, nmap, masscan, sqlmap, hydra, medusa, ncrack, tshark, hping3, tcpdump, netcat, ngrep, iperf3, mtr, sslscan, socat, dirb, whatweb, wrk, hey, vegeta, ethtool
- **GitHub रिलीज़** -- nuclei, dalfox, ffuf, gobuster, feroxbuster, subfinder, httpx, amass
- **Python पैकेज** -- mitmproxy, sslyze, dnsrecon, fierce, theHarvester, playwright, scapy, impacket, arjun
- **Node.js 20.x** -- बॉट सिमुलेशन के लिए स्टेल्थ प्लगइन के साथ Puppeteer
- **ब्राउज़र ऑटोमेशन** -- हेडलेस ब्राउज़र टेस्टिंग के लिए Playwright Chromium
- **Git क्लोन** -- testssl.sh, recon-ng, spiderfoot, SecLists, waf-bypass, PayloadsAllTheThings
- **सूट डिप्लॉयमेंट** -- इस रिपॉजिटरी से `/opt/traffic-generator/suites/` में क्लोन किए गए 19 ट्रैफ़िक सूट
- **कॉन्फ़िग फ़ाइल** -- `TARGET_FQDN`, `TARGET_ORIGIN_IP`, और `CRAPI_PORT` के साथ `config.env`

cloud-init Terraform टेम्पलेट वेरिएबल का उपयोग करता है: `${target_fqdn}`, `${target_origin_ip}`, और `${tool_tier}`। `$NPROC` जैसे शेल वेरिएबल Terraform templatefile में `$$NPROC` के रूप में एस्केप किए जाते हैं।

```yaml file=./_data/cloud-init.yaml
```

### आउटपुट

`outputs.tf` 17 आउटपुट एक्सपोज़ करता है जिसमें पब्लिक/प्राइवेट IP, SSH कमांड, रिसोर्स आइडेंटिफ़ायर, और कंपोनेंट-विशिष्ट मान (target_fqdn, status_check) शामिल हैं:

```hcl file=./_data/outputs.tf
```

### उदाहरण वेरिएबल फ़ाइल

`terraform.tfvars.example` को `terraform.tfvars` में कॉपी करें और अपने मान भरें। `.gitignore` क्रेडेंशियल कमिट होने से रोकने के लिए `terraform.tfvars` को बाहर रखता है:

```hcl file=./_data/terraform.tfvars.example
```

`subscription_id` को अपने Azure सब्सक्रिप्शन ID (`az account show --query id -o tsv` से) और `target_fqdn` को अपने F5 XC लोड बैलेंसर डोमेन से बदलें।

## डिप्लॉय करें

```bash
terraform init

terraform plan

terraform apply
```

सफल डिप्लॉयमेंट के बाद Terraform पब्लिक IP, SSH कमांड, और टार्गेट FQDN आउटपुट करता है।

## VM साइज़िंग

इस CPU-बाउंड ट्रैफ़िक जनरेशन वर्कलोड के लिए F-series कंप्यूट-ऑप्टिमाइज़्ड VM की अनुशंसा की जाती है। cloud-init कर्नेल ट्यूनिंग और टूल कंकरेंसी vCPU काउंट के साथ स्वचालित रूप से स्केल होती है।

| VM SKU | vCPU | RAM | नेटवर्क | उपयोग का मामला |
|---|---|---|---|---|
| Standard_F4s_v2 | 4 | 8 GiB | 10 Gbps | हल्का लैब/डेमो |
| Standard_F8s_v2 | 8 | 16 GiB | 12.5 Gbps | मानक डेमो |
| Standard_F16s_v2 | 16 | 32 GiB | 12.5 Gbps | लोड टेस्टिंग (डिफ़ॉल्ट) |
| Standard_F32s_v2 | 32 | 64 GiB | 16 Gbps | भारी बेंचमार्किंग |
| Standard_D16s_v3 | 16 | 64 GiB | 12.5 Gbps | RAM-भारी वर्कलोड |

:::tip
CDN F16s_v2 और F32s_v2 टार्गेट के विरुद्ध A/B/C/D बेंचमार्क टेस्टिंग द्वारा F16s_v2 को इष्टतम मूल्य-प्रदर्शन विकल्प के रूप में मान्य किया गया था। पूर्ण परिणामों के लिए [benchmarks/](https://github.com/f5xc-salesdemos/traffic-generator/tree/main/benchmarks) देखें।
:::

## डिप्लॉय के बाद सत्यापन

Cloud-init प्रोविज़निंग में **स्टैंडर्ड टियर के लिए 15-20 मिनट** और **फुल टियर के लिए लगभग 25 मिनट** लगते हैं। VM तुरंत SSH के माध्यम से एक्सेसिबल है, लेकिन cloud-init पूरा होने तक टूल्स अभी भी इंस्टॉल हो रहे होते हैं।

### Cloud-Init की प्रतीक्षा करें

VM में SSH करें और प्रोविज़निंग पूरा होने की प्रतीक्षा करें:

```bash
ssh azureuser@$(terraform output -raw public_ip)

cloud-init status --wait
```

पूर्ण होने पर अपेक्षित आउटपुट:

```
status: done
```

### स्टेटस फ़ाइल जाँचें

cloud-init स्क्रिप्ट पूर्ण होने पर एक स्टेटस फ़ाइल लिखती है:

```bash
cat /opt/traffic-generator/status.json
```

अपेक्षित आउटपुट:

```json
{
  "status": "ready",
  "tier": "standard",
  "target_fqdn": "demo.example.com",
  "completed": "2026-04-25T14:30:00Z",
  "tools_installed": true
}
```

### महत्वपूर्ण टूल्स सत्यापित करें

प्रत्येक महत्वपूर्ण टूल PATH पर उपलब्ध है यह पुष्टि करने के लिए एक सत्यापन लूप चलाएँ:

```bash
for tool in nikto sqlmap nuclei dalfox ffuf gobuster feroxbuster nmap masscan \
  hydra medusa ncrack sslscan tshark hping3 mitmproxy sslyze subfinder httpx \
  amass arjun mtr whatweb dirb socat netcat ngrep iperf3 whois dig wrk hey; do
  if command -v "$tool" > /dev/null 2>&1; then
    echo "OK: $tool"
  else
    echo "MISSING: $tool"
  fi
done
```

### सूट रनर सत्यापित करें

पुष्टि करें कि रनर और सूट इंस्टॉल हैं:

```bash
ls /opt/traffic-generator/suites/runner.sh
ls /opt/traffic-generator/suites/*/

/opt/traffic-generator/suites/runner.sh web-app-attacks --dry-run
```

## अपस्ट्रीम कंपोनेंट से वायरिंग

ट्रैफ़िक जनरेटर एक F5 XC HTTP लोड बैलेंसर को टार्गेट करता है जो ऑरिजिन सर्वर के सामने होता है। पहले ऑरिजिन सर्वर डिप्लॉय करें, F5 XC कॉन्फ़िगर करें, फिर `target_fqdn` को लोड बैलेंसर के FQDN पर सेट करें:

```bash
# After deploying origin-server and creating an F5 XC HTTP load balancer:
cat > terraform.tfvars <<EOF
subscription_id  = "your-subscription-id"
target_fqdn      = "your-xc-load-balancer.example.com"
target_origin_ip = "$(cd ../../origin-server/terraform && terraform output -raw public_ip)"
EOF
```

| आवश्यक वेरिएबल | स्रोत | विवरण |
|---|---|---|
| `target_fqdn` | F5 XC कंसोल | ऑरिजिन सर्वर की सुरक्षा करने वाले HTTP लोड बैलेंसर का FQDN |
| `target_origin_ip` | ऑरिजिन सर्वर `public_ip` आउटपुट | बायपास टेस्टिंग के लिए डायरेक्ट ऑरिजिन IP (वैकल्पिक) |

संपूर्ण टूल संदर्भ के लिए [Tool Catalog](../04-tool-catalog/) या ट्रैफ़िक जनरेट करना शुरू करने के लिए [Suites](../05-suites/) पर आगे बढ़ें।
