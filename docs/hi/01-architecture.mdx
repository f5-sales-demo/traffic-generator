## उद्देश्य

यह कंपोनेंट एक स्वचालित ट्रैफ़िक जनरेशन प्लेटफ़ॉर्म प्रदान करता है जो F5 Distributed Cloud HTTP लोड बैलेंसर के विरुद्ध हमला ट्रैफ़िक, रिकॉनसैंस स्कैन, बॉट सिमुलेशन, और API दुरुपयोग उत्पन्न करता है। यह एक सामान्य डेमो आर्किटेक्चर में "हमलावर" है -- दुर्भावनापूर्ण और संदिग्ध ट्रैफ़िक का स्रोत जिसे F5 XC सुरक्षा सुविधाएँ पहचानने और अवरुद्ध करने के लिए डिज़ाइन की गई हैं।

डेमो आर्किटेक्चर में:

```
Traffic Generator VM -> F5 XC HTTP LB (WAF/Bot/API/CSD) -> Origin Server VM
```

ट्रैफ़िक जनरेटर F5 XC लोड बैलेंसर के सार्वजनिक FQDN को अनुरोध भेजता है। F5 XC प्लेटफ़ॉर्म ट्रैफ़िक का निरीक्षण और फ़िल्टर करता है और फिर वैध अनुरोधों को origin सर्वर पर अग्रेषित करता है। ऑपरेटर तब पहचान और प्रवर्तन प्रदर्शित करने के लिए F5 XC सुरक्षा इवेंट लॉग की समीक्षा करता है।

## आर्किटेक्चर

```mermaid
graph LR
    TG[Traffic Generator VM<br/>Ubuntu 24.04<br/>Standard_F16s_v2] -->|Attack Traffic<br/>HTTPS| XCHLB[F5 XC HTTP LB<br/>WAF / Bot Defense<br/>API Security / CSD]
    XCHLB -->|Filtered Traffic<br/>HTTP| ORIGIN[Origin Server VM<br/>nginx + Docker Apps]
    TG -->|Direct Baseline<br/>HTTP optional| ORIGIN
    RUNNER[runner.sh] --> SUITES[19 Traffic Suites]
    SUITES --> TG
```

ट्रैफ़िक जनरेटर VM Azure पर इनके साथ चलता है:

- **Ubuntu 24.04 LTS** बेस इमेज के रूप में
- **50+ सुरक्षा उपकरण** प्रोविज़निंग के दौरान cloud-init के माध्यम से स्थापित
- **19 व्यवस्थित ट्रैफ़िक सुइट्स** क्रमांकित स्क्रिप्ट्स के साथ जो क्रम में निष्पादित होती हैं
- **runner.sh** सुइट निष्पादन के लिए ऑर्केस्ट्रेटर जिसमें परिणाम लॉगिंग है
- **config.env** लक्ष्य कॉन्फ़िगरेशन के लिए (FQDN, origin IP)

## टियर्ड इंस्टॉलेशन

ट्रैफ़िक जनरेटर दो इंस्टॉलेशन टियर का समर्थन करता है जो `tool_tier` Terraform वेरिएबल द्वारा नियंत्रित होते हैं:

### स्टैंडर्ड टियर (डिफ़ॉल्ट)

ZAP और Metasploit को छोड़कर उपकरण कैटलॉग में सूचीबद्ध सभी उपकरण स्थापित करता है। प्रोविज़निंग 15-20 मिनट में पूरी होती है। यह टियर सभी 19 ट्रैफ़िक सुइट्स को कवर करता है और अधिकांश डेमो परिदृश्यों के लिए पर्याप्त है।

### फुल टियर

स्टैंडर्ड टियर के ऊपर OWASP ZAP और Metasploit Framework जोड़ता है। प्रोविज़निंग में लगभग 25 मिनट लगते हैं। ये उपकरण बड़े हैं (ZAP ~500 MiB, Metasploit ~1 GiB) और केवल उन्नत भेद्यता स्कैनिंग डेमो के लिए आवश्यक हैं।

वर्तमान VM लागतों के लिए Azure मूल्य कैलकुलेटर देखें। डिफ़ॉल्ट Standard_F16s_v2 एक कंप्यूट-ऑप्टिमाइज़्ड इंस्टेंस है जो निरंतर ट्रैफ़िक जनरेशन के लिए उपयुक्त है।

:::tip
चल रहे शुल्कों से बचने के लिए जब लैब उपयोग में न हो तब `terraform destroy` का उपयोग करें। प्रक्रिया के लिए [टियरडाउन](../08-teardown/) देखें।
:::

## एकीकरण बिंदु

यह कंपोनेंट दो अन्य डेमो कंपोनेंट्स के साथ एकीकृत होता है:

- **Origin सर्वर** -- लक्ष्य बैकएंड जो Juice Shop, DVWA, VAmPI, httpbin, और whoami होस्ट करता है। ट्रैफ़िक जनरेटर इन अनुप्रयोगों तक पहुँचने के लिए F5 XC के माध्यम से हमला ट्रैफ़िक भेजता है। पूर्ण आर्किटेक्चर विवरण के लिए [एकीकरण](../07-integrate/) देखें।

- **CSD डेमो** -- Origin सर्वर पर Client-Side Defense डेमो अनुप्रयोग। `javascript-exploits` ट्रैफ़िक सुइट Magecart-शैली स्क्रिप्ट इंजेक्शन पेलोड उत्पन्न करता है जिन्हें F5 XC Client-Side Defense पहचानता है। यह CSD Phase 2 कार्यक्षमता को मान्य करता है।

## मॉड्यूलर कंपोनेंट डिज़ाइन

प्रत्येक लैब कंपोनेंट स्व-निहित है और स्वतंत्र रूप से तैनात किया जाता है:

- **ट्रैफ़िक जनरेटर** (यह कंपोनेंट) हमले का स्रोत प्रदान करता है
- **Origin सर्वर** कमजोर अनुप्रयोग लक्ष्य प्रदान करता है
- **CDN सिम्युलेटर** CDN एज कैशिंग लेयर प्रदान करता है (वैकल्पिक)
- **F5 XC कॉन्फ़िगरेशन** WAF, Bot Defense, API Security, और CSD नीतियाँ प्रदान करता है

मानव ऑपरेटर या AI सहायक एक-एक करके कंपोनेंट्स जोड़ता है। पहले origin सर्वर तैनात करें, उसके सामने F5 XC कॉन्फ़िगर करें, फिर F5 XC लोड बैलेंसर FQDN को लक्षित करते हुए ट्रैफ़िक जनरेटर तैनात करें।
