ラボセッションが完了したら、課金を停止するためにすべてのAzureリソースを削除します。

## Terraform Destroy

`terraform/` ディレクトリから実行します：

```bash
cd terraform/

terraform plan -destroy

terraform destroy
```

Terraformが確認を求めます。`yes` と入力して続行します。

## クリーンアップの確認

すべてのリソースが削除されたことを確認します：

```bash
RG_NAME=$(terraform output -raw resource_group_name)

az group show --name "${RG_NAME}" 2>&1 | grep -q "ResourceGroupNotFound" \
  && echo "Resource group deleted" \
  || echo "Resource group still exists"

az resource list --resource-group "${RG_NAME}" -o table 2>/dev/null
```

## ローカル状態のクリーンアップ

Terraformの状態ファイルとキャッシュファイルを削除します：

```bash
rm -rf .terraform .terraform.lock.hcl terraform.tfstate terraform.tfstate.backup
```

## 結果の保存（オプション）

VMを削除する前にトラフィックスイートの結果を保持したい場合は、ローカルにコピーします：

```bash
TGEN_IP=$(terraform output -raw public_ip)

# すべての結果をローカルマシンにコピー
scp -r azureuser@${TGEN_IP}:/opt/traffic-generator/results/ ./traffic-results/

# または最新の実行結果のみをコピー
LATEST=$(ssh azureuser@${TGEN_IP} 'ls -t /opt/traffic-generator/results/ | head -1')
scp -r azureuser@${TGEN_IP}:/opt/traffic-generator/results/${LATEST} ./traffic-results/
```

## F5 XC のクリーンアップ

トラフィックジェネレーターがターゲットとしていたF5 XCの構成も削除したい場合：

1. HTTP ロードバランサーを削除する
2. オリジンプールを削除する
3. テスト用に作成したWAFポリシー、Bot Defense構成、APIセキュリティポリシー、またはCSD構成を削除する

これらのF5 XCリソースはAzureのコストは発生しませんが、デモ環境が不要になった場合はクリーンアップする必要があります。

:::tip
繰り返しデモで使用する場合は、フルライフサイクルのスクリプト化を検討してください：デモ前に `cd terraform && terraform apply` を実行し、スイートを実行した後、`terraform destroy` で削除します。スタンダードティアは15〜20分でプロビジョニングされるため、当日中のセットアップとクリーンアップが実用的です。
:::
