## 概要

`runner.sh` は、スイートディレクトリ内のすべての番号付きスクリプトを順番に実行し、出力をキャプチャし、結果のメタデータを記録するスイートオーケストレーターです。VM 上の `/opt/traffic-generator/suites/runner.sh` にインストールされます。

## 使用方法

```bash
runner.sh <suite-name> [--dry-run]
```

**引数:**

| 引数 | 必須 | 説明 |
|---|---|---|
| `suite-name` | はい | スイートディレクトリの名前（例: `web-app-attacks`、`api-attacks`） |
| `--dry-run` | いいえ | スクリプトを実行せずに、実行される内容を表示する |

**例:**

```bash
# web-app-attacks スイートを実行
/opt/traffic-generator/suites/runner.sh web-app-attacks

# api-attacks スクリプトをドライランでプレビュー
/opt/traffic-generator/suites/runner.sh api-attacks --dry-run

# ボットシミュレーションを実行
/opt/traffic-generator/suites/runner.sh bot-simulation
```

## 設定

### config.env

ランナーは `/opt/traffic-generator/config.env` から設定を読み取ります:

```bash
# Required: target FQDN (F5 XC load balancer domain)
TARGET_FQDN=demo.example.com

# Optional: direct origin IP for baseline testing (bypasses F5 XC)
TARGET_ORIGIN_IP=
```

このファイルは、Terraform プロビジョニング中に `target_fqdn` および `target_origin_ip` 変数から自動的に書き込まれます。デプロイ後にターゲットを変更するには、VM 上でこのファイルを直接編集してください。

### 環境変数のオーバーライド

`config.env` 内の任意の変数は、スイート実行前にエクスポートすることでオーバーライドできます:

```bash
# 単一の実行でターゲット FQDN をオーバーライド
TARGET_FQDN=staging.example.com /opt/traffic-generator/suites/runner.sh web-app-attacks

# export によるオーバーライド
export TARGET_FQDN=staging.example.com
/opt/traffic-generator/suites/runner.sh web-app-attacks
```

設定ファイルのパス自体もオーバーライドできます:

```bash
CONFIG_FILE=/tmp/my-config.env /opt/traffic-generator/suites/runner.sh web-app-attacks
```

## 結果ディレクトリ

各スイート実行により、タイムスタンプ付きの結果ディレクトリが作成されます:

```
/opt/traffic-generator/results/<YYYYMMDD-HHMMSS>-<suite-name>/
```

**構造:**

```
/opt/traffic-generator/results/20260425-143000-web-app-attacks/
  meta.json
  01-sqli.sh.log
  02-xss.sh.log
  03-command-injection.sh.log
  04-path-traversal.sh.log
  05-nikto-scan.sh.log
  06-nuclei-scan.sh.log
```

各スクリプトの stdout と stderr は、スクリプト名にちなんだ `.log` ファイルにキャプチャされます。

### meta.json フォーマット

`meta.json` ファイルはスイート実行のメタデータを記録します:

```json
{
  "suite": "web-app-attacks",
  "target": "demo.example.com",
  "started": "2026-04-25T14:30:00Z",
  "completed": "2026-04-25T14:45:23Z",
  "status": "completed",
  "passed": 5,
  "failed": 1,
  "skipped": 0
}
```

| フィールド | 説明 |
|---|---|
| `suite` | スイート名 |
| `target` | 実行に使用されたターゲット FQDN |
| `started` | スイートが開始された UTC タイムスタンプ |
| `completed` | スイートが完了した UTC タイムスタンプ |
| `status` | 実行中は `running`、完了時は `completed` |
| `passed` | 終了コード 0 で終了したスクリプトの数 |
| `failed` | ゼロ以外の終了コードで終了したスクリプトの数 |
| `skipped` | スキップされたスクリプトの数（実行不可またはドライラン） |

<Aside type="tip">
スクリプトのゼロ以外の終了コードは、必ずしもスイートが失敗したことを意味しません。多くのセキュリティツールは脆弱性を発見した際にゼロ以外で終了しますが、これは攻撃トラフィック生成時の期待される動作です。
</Aside>

## 個別スクリプトの実行

各スクリプトはランナーなしでスタンドアロンで実行できます:

```bash
# 単一のスクリプトを直接実行
/opt/traffic-generator/suites/web-app-attacks/01-sqli.sh demo.example.com

# 特定の API 攻撃を実行
/opt/traffic-generator/suites/api-attacks/01-vampi-owasp-top10.sh demo.example.com
```

スクリプトはターゲット FQDN を最初の位置引数として受け取ります。スクリプトは `config.env` を直接読み取りません -- それを行うのはランナーのみです。スタンドアロンで実行する場合は、FQDN を明示的に渡す必要があります。

## すべてのスイートを順次実行

すべてのスイートを連続して実行するには:

```bash
for suite in web-app-attacks api-attacks bot-simulation reconnaissance ssl-scanning javascript-exploits traffic-generation; do
  echo "========================================="
  echo "Starting suite: ${suite}"
  echo "========================================="
  /opt/traffic-generator/suites/runner.sh "$suite"
  echo ""
done
```

これにより、すべての F5 XC セキュリティ機能にわたる包括的なトラフィックが生成されます。ターゲットの応答時間に応じて、フル実行には 45〜80 分かかることが予想されます。

## SSH によるリモート実行

SSH セッションを維持せずにローカルマシンからスイートを実行します:

```bash
TGEN_IP=$(terraform output -raw public_ip)

# 単一のスイートを実行
ssh azureuser@${TGEN_IP} '/opt/traffic-generator/suites/runner.sh web-app-attacks'

# ターゲットオーバーライド付きでスイートを実行
ssh azureuser@${TGEN_IP} 'TARGET_FQDN=staging.example.com /opt/traffic-generator/suites/runner.sh api-attacks'

# バックグラウンドで実行（切断耐性あり）
ssh azureuser@${TGEN_IP} 'nohup /opt/traffic-generator/suites/runner.sh web-app-attacks > /tmp/web-app-attacks.log 2>&1 &'

# 後で結果を確認
ssh azureuser@${TGEN_IP} 'ls -la /opt/traffic-generator/results/ | tail -5'
ssh azureuser@${TGEN_IP} 'cat /opt/traffic-generator/results/$(ls -t /opt/traffic-generator/results/ | head -1)/meta.json'
```

長時間実行されるスイートの場合は、SSH 切断によるプロセス終了を防ぐために `nohup` または `tmux` を使用してください:

```bash
ssh azureuser@${TGEN_IP} 'tmux new-session -d -s traffic "/opt/traffic-generator/suites/runner.sh reconnaissance"'

# 後で再接続して進捗を確認
ssh -t azureuser@${TGEN_IP} 'tmux attach-session -t traffic'
```

## 実行フロー

<Steps>

1. **設定の読み込み** -- ランナーが `config.env`（または `CONFIG_FILE` オーバーライド）を読み取り、環境内の `TARGET_FQDN` を確認します。

2. **スイートの検証** -- 指定されたスイートディレクトリがスイートディレクトリ配下に存在することを確認します。見つからない場合は、利用可能なスイートを一覧表示します。

3. **結果ディレクトリの作成** -- `/opt/traffic-generator/results/<timestamp>-<suite>/` を作成し、`status: running` で初期 `meta.json` を書き込みます。

4. **スクリプトの実行** -- スイートディレクトリ内の `[0-9]*` に一致するファイルを名前順にイテレートします。実行不可能なファイルはスキップします。`TARGET_FQDN` を最初の引数として渡します。出力を `<script-name>.log` にキャプチャします。

5. **結果の記録** -- 完了タイムスタンプおよびパス/失敗/スキップのカウントで `meta.json` を更新します。

</Steps>
