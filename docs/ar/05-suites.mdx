## api-attacks

**السكربتات:** 4 | **المدة المقدرة:** 10-17 دقيقة

اختبار OWASP API Security Top 10 ضد VAmPI عبر F5 XC. يسجل مستخدم اختبار، ويحصل على رمز مصادقة، ثم يختبر BOLA، والمصادقة بالقوة الغاشمة، والتعرض المفرط للبيانات، وتفويض مستوى الوظائف المعطل، وأنماط SSRF، وحقن SQL ضد نقاط نهاية REST API، واكتشاف المعاملات المخفية، واختبار الضبابية على نقاط النهاية. يستهدف تطبيق VAmPI.

**ميزة F5 XC:** أمان API / اكتشاف API

| الترتيب | السكربت | الأدوات | الوصف |
|---|---|---|---|
| 01 | `01-vampi-owasp-top10.sh` | curl, jq | مجموعة اختبار شاملة لـ OWASP API Top 10. يسجل مستخدم اختبار، ويحصل على رمز مصادقة، ثم يختبر BOLA (الوصول إلى بيانات مستخدمين آخرين)، والمصادقة بالقوة الغاشمة، والتعرض المفرط للبيانات، وتفويض مستوى الوظائف المعطل، وأنماط SSRF، واستكشاف الأخطاء في تكوين الأمان. |
| 02 | `02-sqlmap-api.sh` | sqlmap | حقن SQL ضد نقاط نهاية VAmPI REST API: البحث عن المستخدم، وتسجيل الدخول (POST)، والتسجيل (POST) مع حمولات JSON. |
| 03 | `03-arjun-param-discovery.sh` | arjun | اكتشاف المعاملات المخفية عبر نقاط نهاية Juice Shop وDVWA وVAmPI. يكتشف معاملات الاستعلام غير الموثقة التي قد تقبل حمولات الحقن. |
| 04 | `04-ffuf-api-fuzz.sh` | ffuf | اختبار ضبابية على مرحلتين: اكتشاف نقاط النهاية باستخدام قائمة كلمات لمسارات API الشائعة (api, swagger, graphql, admin, .env, .git) عبر جميع بادئات التطبيقات، ثم اختبار ضبابية لطرق HTTP (GET, POST, PUT, DELETE, PATCH, OPTIONS, TRACE, PROPFIND) ضد نقاط النهاية الرئيسية. |

**السلوك المتوقع من F5 XC:** يجب أن يقوم اكتشاف API بتعيين نقاط نهاية VAmPI API وتحديد أنماط الطلبات غير المعتادة. يجب أن تضع سياسات أمان API علامة على محاولات الوصول غير المصرح بها، وحمولات الحقن في أجسام JSON، وأنماط التعداد. سيعرض مخزون API في F5 XC نقاط النهاية المكتشفة مع أعداد الطلبات.

## bot-simulation

**السكربتات:** 4 | **المدة المقدرة:** 5-10 دقائق

نشاط المتصفح الآلي وأنماط الزحف الشبيهة بالبوتات عبر F5 XC. يتضمن Chrome بدون واجهة، وPuppeteer مع إضافة التخفي، والزحف السريع، والتفاعل الآلي مع النماذج. يستهدف جميع التطبيقات.

**ميزة F5 XC:** الدفاع ضد البوتات

| الترتيب | السكربت | الأدوات | الوصف |
|---|---|---|---|
| 01 | `01-headless-chrome.sh` | playwright | تصفح Chromium بدون واجهة: يحمل الهدف، ويتنقل بين مسارات التطبيقات، ويستخرج عناوين الصفحات والمحتوى، ويلتقط لقطات شاشة. يولد حركة مرور قائمة على المتصفح بدون إضافات تخفي (قابلة للكشف كأتمتة). |
| 02 | `02-puppeteer-stealth.sh` | puppeteer, puppeteer-extra-plugin-stealth | تصفح متخفي باستخدام puppeteer-extra مع إضافة التخفي. يحاول تجنب كشف البوتات عن طريق تعديل خصائص navigator وWebGL وبصمات المتصفح الأخرى. |
| 03 | `03-rapid-crawl.sh` | curl, wget | طلبات HTTP عالية التردد باستخدام curl وwget في تتابع سريع. يحاكي سلوك المستخرج بدون تأخير بين الطلبات وتدوير عنيف لـ User-Agent. |
| 04 | `04-form-interaction.sh` | playwright | ملء وإرسال النماذج آلياً ضد تسجيل دخول DVWA، وتسجيل Juice Shop، ونقاط نهاية VAmPI API باستخدام واجهات برمجة التفاعل مع النماذج في Playwright. |

**السلوك المتوقع من F5 XC:** يجب أن يصنف الدفاع ضد البوتات حركة المرور على أنها آلية بناءً على تحليل بصمة المتصفح، وتوقيت الطلبات، ونتائج تحدي JavaScript. يجب اكتشاف Chrome بدون واجهة بدون تخفي فوراً. تختبر اختبارات Puppeteer المتخفية ما إذا كانت تقنيات التهرب المتقدمة تتجاوز الكشف. ستعرض لوحة معلومات الدفاع ضد البوتات فئات تصنيف البوتات وإجراءات التخفيف.

اختبار سلوك ذاكرة التخزين المؤقت لشبكة CDN وحماية الأصل. يتضمن الإنتاجية الأساسية، وعزل سلسلة الاستعلام، وتباين accept-encoding، ومحاكاة القطيع المتدافع، وتجاوز POST/PUT، وتصادم مفتاح التخزين المؤقت، وGET المشروط، وطلب النطاق، ومحاكاة التطهير، واستنفاد مجمع الاتصالات، وتحسين keepalive، وعبء مصافحة TLS، وتخزين الكائنات الكبيرة مؤقتاً، والتبديل بين أصول متعددة، وتحديد المعدل، واختبار نسبة gzip، وتعدد إرسال HTTP/2، واختبار أداء kraken المجمع. يستهدف تطبيق محاكي CDN.

**ميزة F5 XC:** تكامل CDN

## crapi-exploits

**ميزة F5 XC:** أمان API

حمولات هجمات موجهة ضد نقاط نهاية اختبار WAF المدمجة في F5 DemoApp (`/WAF/SQL`، `/WAF/XSS`، `/WAF/DIR`). يرسل حمولات حقن SQL وXSS واجتياز المسار مباشرة إلى نقاط النهاية المصممة لإظهار سلوك حظر WAF. استخدم هذه المجموعة عند استهداف خادم محتوى DemoApp بدلاً من تطبيقات خادم الأصل القياسية.

## dvga-exploits

**السكربتات:** 9 | **المدة المقدرة:** 8-12 دقيقة

استغلال ثغرات خاصة بـ GraphQL ضد تطبيق Damn Vulnerable GraphQL Application. يتضمن هجوم DoS بالاستعلامات المجمعة، والتكرار العميق، وتكرار الحقول، وحقن SQL عبر الفلتر، وXSS عبر createPaste، وDoS القائم على الأسماء المستعارة، وإساءة استخدام الاستبطان، وتجاوز التفويض، وكشف المعلومات. يستهدف تطبيق DVGA.

**ميزة F5 XC:** أمان API (GraphQL)

## dvwa-exploits

**السكربتات:** 14 | **المدة المقدرة:** 15-25 دقيقة

اختبار OWASP Top 10 ضد تطبيق Damn Vulnerable Web Application. يتضمن القوة الغاشمة، وحقن الأوامر، وتغيير كلمة مرور CSRF، وتضمين الملفات، ورفع الملفات، وحقن SQL (الأعمى)، وحقن SQL (union)، وXSS (DOM)، وXSS (المنعكس)، وXSS (المخزن)، وتجاوز CAPTCHA، ومعرفات الجلسات الضعيفة، وCORS غير الآمن، وإعادة التوجيه المفتوح. يستهدف تطبيق DVWA.

**ميزة F5 XC:** WAF

## javascript-exploits

**السكربتات:** 3 | **المدة المقدرة:** 3-5 دقائق

أنماط حقن السكربتات من جانب العميل والتلاعب بـ DOM التي تُفعّل كشف الدفاع من جانب العميل. تتضمن حقن السكربتات المضمنة التي تحاكي أدوات سرقة بأسلوب Magecart، والتلاعب بـ DOM عبر Chromium بدون واجهة، ومحاكاة السكربتات الخارجية. يستهدف تطبيق CSD Demo.

**ميزة F5 XC:** الدفاع من جانب العميل (CSD)

| الترتيب | السكربت | الأدوات | الوصف |
|---|---|---|---|
| 01 | `01-inline-script-injection.sh` | curl | يرسل طلبات تحتوي على حمولات JavaScript مضمنة مصممة لمحاكاة أدوات سرقة بطاقات الائتمان بأسلوب Magecart وسكربتات استخراج البيانات. |
| 02 | `02-dom-manipulation.sh` | playwright | يستخدم Chromium بدون واجهة لتحميل صفحة CSD التجريبية وحقن تعديلات DOM: إضافة وسوم script، وتعديل إجراءات النماذج، وإدراج إطارات iframe غير مرئية تستخرج بيانات النماذج. |
| 03 | `03-third-party-script-sim.sh` | curl, playwright | يحاكي السكربتات الخارجية المخترقة عن طريق حقن طلبات تشير إلى مصادر JavaScript خارجية ومحاولة تحميل سكربتات غير مصرح بها في سياق الصفحة. |

**السلوك المتوقع من F5 XC:** يجب أن يكتشف الدفاع من جانب العميل التعديلات غير المصرح بها على السكربتات في الصفحة المحمية. ستعرض لوحة معلومات CSD تنبيهات لمصادر السكربتات الجديدة، وعناصر DOM المعدلة، ومحاولات استخراج البيانات. يتحقق هذا من وظائف المرحلة الثانية من CSD.

**ميزة F5 XC:** WAF، الدفاع ضد البوتات

## mitre-attack

**السكربتات:** 8 | **المدة المقدرة:** 10-15 دقيقة

محاكاة تكتيكات إطار عمل MITRE ATT&CK. يتضمن الاستطلاع (المسح الخارجي)، والوصول الأولي (حشو بيانات الاعتماد)، والتنفيذ (حقن الأوامر)، والوصول إلى بيانات الاعتماد (كلمات المرور الافتراضية)، والاكتشاف (تعداد الأدلة)، ومحاكاة الحركة الجانبية، والجمع (استخراج البيانات)، ومحاكاة التسريب. يربط كل سكربت بتكتيك محدد من MITRE ATT&CK لتقارير تغطية التهديدات المنظمة. يستهدف جميع التطبيقات.

**ميزة F5 XC:** WAF، الدفاع ضد البوتات، أمان API

**السلوك المتوقع من F5 XC:** يولد مسح SSL العديد من مصافحات TLS مع اقتراحات غير معتادة لمجموعات التشفير. بينما لا يحظر F5 XC هذه عادةً، يكون نمط حركة المرور مرئياً في سجلات الوصول. تتمثل القيمة الأساسية في التحقق من أن تكوين TLS في F5 XC يلبي أفضل ممارسات الأمان.

## traffic-generation

**السكربتات:** 4 | **المدة المقدرة:** 5-10 دقائق (قابلة للتكوين)

حركة مرور HTTP مشروعة عالية الحجم للقياس الأساسي واختبار التحميل. تنشئ أنماط حركة مرور طبيعية في تحليلات F5 XC للمقارنة مع حركة مرور مجموعات الهجمات. يستهدف جميع التطبيقات.

**ميزة F5 XC:** الكل (مقارنة أساسية)

| الترتيب | السكربت | الأدوات | الوصف |
|---|---|---|---|
| 01 | `01-baseline-http.sh` | curl | طلبات HTTP GET متتالية إلى جميع مسارات التطبيقات مع User-Agent متصفح قياسي. تنشئ خط أساس لحركة المرور الطبيعية في تحليلات F5 XC. |
| 02 | `02-concurrent-load.sh` | curl | طلبات HTTP متوازية باستخدام curl مع اتصالات متزامنة متعددة. تختبر التعامل مع التحميل وتنشئ خطوط أساس للإنتاجية. |
| 03 | `03-mixed-methods.sh` | curl | طرق HTTP مختلطة (GET, POST, PUT, DELETE) ضد نقاط نهاية API مع حمولات صالحة. تولد أنماط حركة مرور API طبيعية للمقارنة مع حركة مرور الهجمات. |
| 04 | `04-user-journey.sh` | playwright | يحاكي رحلة مستخدم واقعية: تصفح الصفحات، والبحث عن المنتجات، وملء النماذج، والتنقل بين التطبيقات باستخدام Playwright مع إعدادات متصفح قياسية. |

**السلوك المتوقع من F5 XC:** يجب أن تمر جميع الطلبات في هذه المجموعة بدون تدخل من WAF أو الدفاع ضد البوتات. استخدم هذه المجموعة لتوليد حركة مرور "نظيفة" في لوحة معلومات التحليلات، ثم قارنها بحركة مرور مجموعات الهجمات لإظهار الفرق بين أنماط الطلبات المشروعة والخبيثة.

## waf-encoding-evasion

**السكربتات:** 7 | **المدة المقدرة:** 5-10 دقائق

هجمات تهرب بالترميز متعدد الطبقات مصممة لاختبار ما إذا كان WAF يقوم بتطبيع وفك ترميز الحمولات بشكل صحيح قبل الفحص. تغطي ترميز URL الأحادي والمزدوج والثلاثي؛ وترميز كيانات HTML (عشري، وست عشري، ومسمى، ومبطن بالأصفار)؛ وإدراج Unicode/UTF-8 (مسافة صفرية العرض، وBOM، وأحرف كاملة العرض، وواصلة لينة، وتسلسلات مطولة)؛ والترميز المختلط/المتداخل متعدد الطبقات (كيانات HTML مشفرة بـ URL، وURL مزدوج داخل الكيانات، ومكدسات ثلاثية الطبقات)؛ وحقن البايت الفارغ؛ وترميز IIS %uXXXX؛ والتلاعب بحالة الأحرف؛ وتقسيم نص الترميز المجزأ؛ وتهرب معاملات Base64؛ وحمولات مشفرة في الترويسات وملفات تعريف الارتباط. يستهدف أي تطبيق.

**ميزة F5 XC:** WAF (عمق تطبيع الترميز)

| الترتيب | السكربت | الأدوات | الوصف |
|---|---|---|---|
| 01 | `01-url-encoding.sh` | curl | ترميز URL الأحادي والمزدوج والثلاثي لحمولات SQLi وXSS واجتياز المسار عبر مسارات نقاط نهاية متعددة. يختبر ما إذا كان WAF يفك ترميز معاملات URL بشكل تكراري. |
| 02 | `02-html-entity-encoding.sh` | curl, python3 | ترميز كيانات HTML بالشكل العشري والست عشري والمسمى والمبطن بالأصفار لحمولات الهجوم في كل من معاملات GET وأجسام POST. يتضمن حقن القوالب عبر `&#123;&#123;7*7&#125;&#125;`. |
| 03 | `03-unicode-utf8-evasion.sh` | curl | إدراج مسافة صفرية العرض (U+200B)، وBOM (U+FEFF)، وأحرف كاملة العرض، وواصلة لينة، وZWNJ في كلمات SQL المفتاحية ووسوم XSS. يختبر تسلسلات UTF-8 المطولة (2 بايت و3 بايت). |
| 04 | `04-mixed-nested-encoding.sh` | curl | حمولات متعددة الطبقات: كيانات HTML مشفرة بـ URL، وURL مزدوج داخل الكيانات، ومكدسات ثلاثية الطبقات، وجسم JSON مع تهريب Unicode (`<`)، ونمط حقن القوالب `{{7*7}}` المحدد من المستخدم. |
| 05 | `05-null-byte-iis-base64.sh` | curl | حقن البايت الفارغ (`%00`) لتجاوز الامتداد، وترميز IIS Unicode `%uXXXX`، وحمولات مشفرة بـ Base64 في معاملات URL مع أعلام فك الترميز. |
| 06 | `06-case-chunked-evasion.sh` | curl | عشوائية حالة الأحرف مع الترميز (`%3CsCrIpT%3E`)، والترميز المجزأ لتقسيم كلمات SQL/XSS المفتاحية عبر أجزاء HTTP، واستكشافات تعارض Content-Length/Transfer-Encoding، وفواصل المسارات البديلة (`%5c`، `%c0%af`). |
| 07 | `07-header-cookie-evasion.sh` | curl | حمولات هجوم مشفرة محقونة عبر ترويسات Cookie وReferer وUser-Agent وX-Forwarded-For وX-Original-URL. يختبر ما إذا كان WAF يفحص ويفك ترميز جميع حقول ترويسات HTTP، وليس فقط معاملات الاستعلام وأجسام POST. |

**السلوك المتوقع من F5 XC:** يجب أن يقوم WAF المُكوَّن بشكل صحيح بتطبيع جميع طبقات الترميز قبل مطابقة الأنماط. يجب حظر الحمولات المشفرة بطبقة واحدة فوراً. تكشف اختبارات الترميز المزدوج والثلاثي عن عمق خط أنابيب فك الترميز في WAF. تكشف اختبارات إدراج Unicode ما إذا كانت الأحرف غير المرئية تُزال قبل مطابقة الكلمات المفتاحية. النتائج التي تمر فيها المتغيرات المشفرة بينما تُحظر المكافئات النصية العادية تشير إلى فجوات في التطبيع في سياسة WAF.

## web-app-attacks

**السكربتات:** 6 | **المدة المقدرة:** 12-20 دقيقة

**ميزة F5 XC:** WAF (جدار حماية تطبيقات الويب)
