Ogni traffic suite è una directory sotto `/opt/traffic-generator/suites/` contenente script shell numerati eseguiti in ordine da `runner.sh`. Ogni script accetta il FQDN target come primo argomento e scrive l'output sia su stdout che nella directory dei risultati.

## api-attacks

**Script:** 4 | **Durata stimata:** 10-17 minuti

Test OWASP API Security Top 10 contro VAmPI attraverso F5 XC. Registra un utente di test, ottiene un token di autenticazione, quindi testa BOLA, autenticazione brute-force, esposizione eccessiva dei dati, autorizzazione a livello di funzione non corretta, pattern SSRF, SQL injection contro endpoint REST API, scoperta di parametri nascosti e fuzzing degli endpoint. Ha come target l'applicazione VAmPI.

**Funzionalità F5 XC:** API Security / API Discovery

| Ordine | Script | Strumenti | Descrizione |
|---|---|---|---|
| 01 | `01-vampi-owasp-top10.sh` | curl, jq | Suite completa di test OWASP API Top 10. Registra un utente di test, ottiene un token di autenticazione, quindi testa BOLA (accesso ai dati di altri utenti), autenticazione brute-force, esposizione eccessiva dei dati, autorizzazione a livello di funzione non corretta, pattern SSRF e sondaggio delle configurazioni di sicurezza errate. |
| 02 | `02-sqlmap-api.sh` | sqlmap | SQL injection contro endpoint REST API di VAmPI: ricerca utente, login (POST) e registrazione (POST) con payload JSON. |
| 03 | `03-arjun-param-discovery.sh` | arjun | Scoperta di parametri nascosti su endpoint di Juice Shop, DVWA e VAmPI. Scopre parametri query non documentati che possono accettare payload di injection. |
| 04 | `04-ffuf-api-fuzz.sh` | ffuf | Fuzzing a due fasi: scoperta degli endpoint utilizzando una wordlist di percorsi API comuni (api, swagger, graphql, admin, .env, .git) su tutti i prefissi delle applicazioni, quindi fuzzing dei metodi HTTP (GET, POST, PUT, DELETE, PATCH, OPTIONS, TRACE, PROPFIND) contro gli endpoint chiave. |

**Comportamento atteso di F5 XC:** API Discovery dovrebbe mappare gli endpoint API di VAmPI e identificare pattern di richieste insoliti. Le policy di API Security dovrebbero segnalare tentativi di accesso non autorizzato, payload di injection nei body JSON e pattern di enumerazione. L'API Inventory in F5 XC mostrerà gli endpoint scoperti con il conteggio delle richieste.

## bot-simulation

**Script:** 4 | **Durata stimata:** 5-10 minuti

Attività browser automatizzata e pattern di crawling simili a bot attraverso F5 XC. Include Chrome headless, Puppeteer con plugin stealth, crawling rapido e interazione automatizzata con i form. Ha come target tutte le applicazioni.

**Funzionalità F5 XC:** Bot Defense

| Ordine | Script | Strumenti | Descrizione |
|---|---|---|---|
| 01 | `01-headless-chrome.sh` | playwright | Navigazione con Chromium headless: carica il target, naviga tra i percorsi delle applicazioni, estrae titoli e contenuti delle pagine, cattura screenshot. Genera traffico basato su browser senza plugin stealth (rilevabile come automazione). |
| 02 | `02-puppeteer-stealth.sh` | puppeteer, puppeteer-extra-plugin-stealth | Navigazione stealth utilizzando puppeteer-extra con il plugin stealth. Tenta di eludere il rilevamento dei bot modificando le proprietà del navigator, WebGL e altri fingerprint del browser. |
| 03 | `03-rapid-crawl.sh` | curl, wget | Richieste HTTP ad alta frequenza utilizzando curl e wget in rapida successione. Simula il comportamento di uno scraper senza ritardo tra le richieste e rotazione aggressiva dello User-Agent. |
| 04 | `04-form-interaction.sh` | playwright | Compilazione e invio automatizzato di form contro il login di DVWA, la registrazione di Juice Shop e gli endpoint API di VAmPI utilizzando le API di interazione form di Playwright. |

**Comportamento atteso di F5 XC:** Bot Defense dovrebbe classificare il traffico come automatizzato in base all'analisi del fingerprint del browser, alla temporizzazione delle richieste e ai risultati delle challenge JavaScript. Chrome headless senza stealth dovrebbe essere rilevato immediatamente. I test con Puppeteer stealth verificano se le tecniche avanzate di evasione aggirano il rilevamento. La dashboard di Bot Defense mostrerà le categorie di classificazione dei bot e le azioni di mitigazione.

## demoapp-attacks

**Script:** 2 | **Durata stimata:** 2-3 minuti

Payload di attacco mirati contro gli endpoint di test WAF integrati di F5 DemoApp (`/WAF/SQL`, `/WAF/XSS`, `/WAF/DIR`). Invia payload di SQL injection, XSS e path traversal direttamente agli endpoint progettati per dimostrare il comportamento di blocco del WAF. Utilizzare questa suite quando si ha come target il content server DemoApp anziché le applicazioni standard dell'origin server.

**Funzionalità F5 XC:** WAF (Web Application Firewall)

| Ordine | Script | Strumenti | Descrizione |
|---|---|---|---|
| 01 | `01-sqli-waf-endpoint.sh` | curl, python3 | 15 payload di SQL injection contro `/WAF/SQL?age=` inclusi UNION SELECT, DROP TABLE, WAITFOR DELAY, xp_cmdshell, bypass basato su commenti e payload di logica booleana. |
| 02 | `02-xss-waf-endpoint.sh` | curl, python3 | 18 payload XSS contro `/WAF/XSS?update=` inclusi tag script, event handler (onerror, onload, ontoggle, onfocus), eval base64, espressioni style, payload poliglotti ed esfiltrazione dei cookie. |

**Comportamento atteso di F5 XC:** Il WAF dovrebbe bloccare le richieste contenenti payload di SQL injection e XSS e restituire una pagina di blocco. La dashboard Security Events mostrerà le categorie di violazione e le corrispondenze delle signature per ogni richiesta bloccata.

## waf-encoding-evasion

**Script:** 7 | **Durata stimata:** 5-10 minuti

Attacchi di evasione con codifica multi-livello progettati per testare se un WAF normalizza e decodifica correttamente i payload prima dell'ispezione. Copre codifica URL singola, doppia e tripla; codifica entità HTML (decimale, esadecimale, nominale, con zero-padding); inserimento Unicode/UTF-8 (zero-width space, BOM, caratteri fullwidth, soft hyphen, sequenze overlong); codifica mista/annidata multi-livello (entità HTML codificate URL, doppia-URL all'interno di entità, stack a 3 livelli); injection di null byte; codifica IIS %uXXXX; manipolazione maiuscole/minuscole; splitting del body con chunked transfer encoding; evasione parametri Base64; e payload codificati negli header/cookie. Ha come target qualsiasi applicazione.

**Funzionalità F5 XC:** WAF (profondità di normalizzazione della codifica)

| Ordine | Script | Strumenti | Descrizione |
|---|---|---|---|
| 01 | `01-url-encoding.sh` | curl | Codifica URL singola, doppia e tripla di payload SQLi, XSS e path traversal su percorsi endpoint multipli. Testa se il WAF decodifica i parametri URL in modo ricorsivo. |
| 02 | `02-html-entity-encoding.sh` | curl, python3 | Codifica entità HTML decimale, esadecimale, nominale e con zero-padding dei payload di attacco sia nei parametri GET che nei body POST. Include template injection tramite `&#123;&#123;7*7&#125;&#125;`. |
| 03 | `03-unicode-utf8-evasion.sh` | curl | Inserimento di zero-width space (U+200B), BOM (U+FEFF), caratteri fullwidth, soft hyphen e ZWNJ nelle keyword SQL e nei tag XSS. Testa sequenze UTF-8 overlong (2-byte e 3-byte). |
| 04 | `04-mixed-nested-encoding.sh` | curl | Payload multi-livello: entità HTML codificate URL, doppia-URL all'interno di entità, stack a triplo livello, body JSON con escape Unicode (`<`) e il pattern specifico di template injection `{{7*7}}` dell'utente. |
| 05 | `05-null-byte-iis-base64.sh` | curl | Injection di null byte (`%00`) per bypass delle estensioni, codifica Unicode IIS `%uXXXX` e payload codificati Base64 nei parametri URL con flag di decodifica. |
| 06 | `06-case-chunked-evasion.sh` | curl | Randomizzazione delle maiuscole/minuscole combinata con codifica (`%3CsCrIpT%3E`), chunked transfer encoding per dividere keyword SQL/XSS tra chunk HTTP, probe di conflitto Content-Length/Transfer-Encoding e separatori di percorso alternativi (`%5c`, `%c0%af`). |
| 07 | `07-header-cookie-evasion.sh` | curl | Payload di attacco codificati iniettati tramite header Cookie, Referer, User-Agent, X-Forwarded-For e X-Original-URL. Testa se il WAF ispeziona e decodifica tutti i campi degli header HTTP, non solo i parametri query e i body POST. |

**Comportamento atteso di F5 XC:** Un WAF configurato correttamente dovrebbe normalizzare tutti i livelli di codifica prima del pattern matching. I payload con codifica singola dovrebbero essere bloccati immediatamente. I test con codifica doppia e tripla rivelano la profondità della pipeline di decodifica del WAF. I test di inserimento Unicode rivelano se i caratteri invisibili vengono rimossi prima della corrispondenza delle keyword. Risultati in cui le varianti codificate passano mentre gli equivalenti in testo semplice vengono bloccati indicano lacune di normalizzazione nella policy del WAF.

**Funzionalità F5 XC:** WAF (Web Application Firewall)

| Ordine | Script | Strumenti | Descrizione |
|---|---|---|---|
| 01 | `01-sqli.sh` | sqlmap, curl | Payload di SQL injection contro l'API di ricerca di Juice Shop e l'endpoint SQLi di DVWA. Esegue scansioni automatizzate sqlmap poi invia 10 payload di injection diretti tramite curl. |
| 02 | `02-xss.sh` | dalfox, curl | Cross-site scripting contro gli endpoint XSS reflected/stored di Juice Shop e DVWA. Esegue scansioni automatizzate dalfox poi invia 12 payload XSS diretti inclusi tag script, event handler ed esfiltrazione dei cookie. |
| 03 | `03-command-injection.sh` | curl | Payload di OS command injection contro l'endpoint exec di DVWA. Invia 12 payload incluse catene con pipe, punti e virgola, newline codificate URL e pattern di reverse shell. |
| 04 | `04-path-traversal.sh` | curl | Directory traversal contro endpoint di applicazioni multiple. Testa traversal standard `../`, varianti codificate URL, varianti con doppia codifica, varianti con null-byte e varianti Unicode su 6 percorsi endpoint. |
| 05 | `05-nikto-scan.sh` | nikto | Scansione completa delle vulnerabilità del web server con limite di tempo di 120 secondi. Genera richieste HTTP diverse che attivano il matching delle signature del WAF. |
| 06 | `06-nuclei-scan.sh` | nuclei | Scansione vulnerabilità basata su template a livelli di severità medio, alto e critico. Limitata a 50 richieste/secondo. |

**Comportamento atteso di F5 XC:** Il WAF dovrebbe bloccare o segnalare le richieste contenenti payload di SQL injection, XSS, command injection e path traversal. La dashboard Security Events mostrerà le categorie di violazione, gli ID delle signature e i dettagli delle richieste per ogni richiesta bloccata.
