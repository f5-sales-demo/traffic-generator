Ogni suite di traffico è una directory sotto `/opt/traffic-generator/suites/` contenente script shell numerati eseguiti in ordine da `runner.sh`. Ogni script accetta l'FQDN di destinazione come primo argomento e scrive l'output sia su stdout che nella directory dei risultati.

## api-attacks

**Script:** 4 | **Durata stimata:** 10-17 minuti

Test OWASP API Security Top 10 contro VAmPI attraverso F5 XC. Registra un utente di test, ottiene un token di autenticazione, quindi testa BOLA, autenticazione brute-force, esposizione eccessiva dei dati, autorizzazione a livello di funzione non corretta, pattern SSRF, SQL injection contro endpoint REST API, scoperta di parametri nascosti e fuzzing degli endpoint. Prende di mira l'applicazione VAmPI.

**Funzionalità F5 XC:** API Security / API Discovery

| Ordine | Script | Strumenti | Descrizione |
|---|---|---|---|
| 01 | `01-vampi-owasp-top10.sh` | curl, jq | Suite completa di test OWASP API Top 10. Registra un utente di test, ottiene un token di autenticazione, quindi testa BOLA (accesso ai dati di altri utenti), autenticazione brute-force, esposizione eccessiva dei dati, autorizzazione a livello di funzione non corretta, pattern SSRF e sondaggio delle misconfigurazioni di sicurezza. |
| 02 | `02-sqlmap-api.sh` | sqlmap | SQL injection contro endpoint REST API di VAmPI: ricerca utente, login (POST) e registrazione (POST) con payload JSON. |
| 03 | `03-arjun-param-discovery.sh` | arjun | Scoperta di parametri nascosti su endpoint di Juice Shop, DVWA e VAmPI. Scopre parametri query non documentati che possono accettare payload di injection. |
| 04 | `04-ffuf-api-fuzz.sh` | ffuf | Fuzzing in due fasi: scoperta degli endpoint utilizzando una wordlist di percorsi API comuni (api, swagger, graphql, admin, .env, .git) su tutti i prefissi delle applicazioni, quindi fuzzing dei metodi HTTP (GET, POST, PUT, DELETE, PATCH, OPTIONS, TRACE, PROPFIND) sugli endpoint chiave. |

**Comportamento atteso di F5 XC:** API Discovery dovrebbe mappare gli endpoint dell'API VAmPI e identificare pattern di richieste inusuali. Le policy di API Security dovrebbero segnalare tentativi di accesso non autorizzati, payload di injection nei body JSON e pattern di enumerazione. L'inventario API in F5 XC mostrerà gli endpoint scoperti con i conteggi delle richieste.

## bot-simulation

Attività del browser automatizzata e pattern di crawling simili a bot attraverso F5 XC. Include Chrome headless, Puppeteer con plugin stealth, crawling rapido e interazione automatizzata con form. Prende di mira tutte le applicazioni.

**Funzionalità F5 XC:** Bot Defense

| Ordine | Script | Strumenti | Descrizione |
|---|---|---|---|
| 01 | `01-headless-chrome.sh` | playwright | Navigazione con Chromium headless: carica il target, naviga tra i percorsi delle applicazioni, estrae titoli e contenuti delle pagine, acquisisce screenshot. Genera traffico basato su browser senza plugin stealth (rilevabile come automazione). |
| 02 | `02-puppeteer-stealth.sh` | puppeteer, puppeteer-extra-plugin-stealth | Navigazione stealth utilizzando puppeteer-extra con il plugin stealth. Tenta di eludere il rilevamento bot modificando le proprietà del navigator, WebGL e altre impronte digitali del browser. |
| 03 | `03-rapid-crawl.sh` | curl, wget | Richieste HTTP ad alta frequenza utilizzando curl e wget in rapida successione. Simula il comportamento di uno scraper senza ritardo tra le richieste e rotazione aggressiva degli User-Agent. |
| 04 | `04-form-interaction.sh` | playwright | Compilazione e invio automatizzato di form contro il login di DVWA, la registrazione di Juice Shop e gli endpoint API di VAmPI utilizzando le API di interazione con i form di Playwright. |

**Comportamento atteso di F5 XC:** Bot Defense dovrebbe classificare il traffico come automatizzato basandosi sull'analisi dell'impronta digitale del browser, la temporizzazione delle richieste e i risultati delle sfide JavaScript. Chrome headless senza stealth dovrebbe essere rilevato immediatamente. I test con Puppeteer stealth verificano se le tecniche avanzate di evasione aggirano il rilevamento. La dashboard di Bot Defense mostrerà le categorie di classificazione dei bot e le azioni di mitigazione.

## waf-encoding-evasion

**Script:** 7 | **Durata stimata:** 5-10 minuti

Attacchi di evasione con codifica multi-livello progettati per testare se un WAF normalizza e decodifica correttamente i payload prima dell'ispezione. Copre codifica URL singola, doppia e tripla; codifica entità HTML (decimale, esadecimale, denominata, con riempimento zero); inserimento Unicode/UTF-8 (spazio a larghezza zero, BOM, caratteri fullwidth, trattino morbido, sequenze overlong); codifica multi-livello mista/annidata (entità HTML codificate URL, doppio URL dentro entità, stack a 3 livelli); injection null byte; codifica IIS %uXXXX; manipolazione del case; suddivisione del body con transfer encoding chunked; evasione parametri Base64; e payload codificati in header/cookie. Prende di mira qualsiasi applicazione.

**Funzionalità F5 XC:** WAF (profondità di normalizzazione della codifica)

| Ordine | Script | Strumenti | Descrizione |
|---|---|---|---|
| 01 | `01-url-encoding.sh` | curl | Codifica URL singola, doppia e tripla di payload SQLi, XSS e path traversal su percorsi endpoint multipli. Testa se il WAF decodifica i parametri URL in modo ricorsivo. |
| 02 | `02-html-entity-encoding.sh` | curl, python3 | Codifica entità HTML decimale, esadecimale, denominata e con riempimento zero di payload di attacco sia in parametri GET che in body POST. Include template injection tramite `&#123;&#123;7*7&#125;&#125;`. |
| 03 | `03-unicode-utf8-evasion.sh` | curl | Inserimento di spazio a larghezza zero (U+200B), BOM (U+FEFF), caratteri fullwidth, trattino morbido e ZWNJ in keyword SQL e tag XSS. Testa sequenze UTF-8 overlong (2 byte e 3 byte). |
| 04 | `04-mixed-nested-encoding.sh` | curl | Payload multi-livello: entità HTML codificate URL, doppio URL dentro entità, stack a tre livelli, body JSON con escape Unicode (`<`) e il pattern specifico di template injection `{{7*7}}` dell'utente. |
| 05 | `05-null-byte-iis-base64.sh` | curl | Injection null byte (`%00`) per bypass dell'estensione, codifica Unicode IIS `%uXXXX` e payload codificati Base64 nei parametri URL con flag di decodifica. |
| 06 | `06-case-chunked-evasion.sh` | curl | Randomizzazione del case combinata con codifica (`%3CsCrIpT%3E`), transfer encoding chunked per suddividere keyword SQL/XSS tra chunk HTTP, sonde di conflitto Content-Length/Transfer-Encoding e separatori di percorso alternativi (`%5c`, `%c0%af`). |
| 07 | `07-header-cookie-evasion.sh` | curl | Payload di attacco codificati iniettati tramite header Cookie, Referer, User-Agent, X-Forwarded-For e X-Original-URL. Testa se il WAF ispeziona e decodifica tutti i campi header HTTP, non solo i parametri query e i body POST. |

**Comportamento atteso di F5 XC:** Un WAF configurato correttamente dovrebbe normalizzare tutti i livelli di codifica prima del pattern matching. I payload con codifica singola dovrebbero essere bloccati immediatamente. I test con codifica doppia e tripla rivelano la profondità della pipeline di decodifica del WAF. I test di inserimento Unicode rivelano se i caratteri invisibili vengono rimossi prima della corrispondenza delle keyword. Risultati in cui le varianti codificate passano mentre gli equivalenti in testo semplice vengono bloccati indicano lacune di normalizzazione nella policy del WAF.
