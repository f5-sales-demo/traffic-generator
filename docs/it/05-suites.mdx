Ogni suite di traffico è una directory sotto `/opt/traffic-generator/suites/` contenente script shell numerati eseguiti in ordine da `runner.sh`. Ogni script accetta l'FQDN di destinazione come primo argomento e scrive l'output sia su stdout che nella directory dei risultati.

## api-attacks

**Script:** 4 | **Durata Stimata:** 10-17 minuti

Test OWASP API Security Top 10 contro VAmPI attraverso F5 XC. Registra un utente di test, ottiene un token di autenticazione, poi testa BOLA, autenticazione brute-force, esposizione eccessiva dei dati, autorizzazione a livello di funzione compromessa, pattern SSRF, SQL injection contro endpoint REST API, scoperta di parametri nascosti e fuzzing degli endpoint. Ha come obiettivo l'applicazione VAmPI.

**Funzionalità F5 XC:** API Security / API Discovery

| Ordine | Script | Strumenti | Descrizione |
|---|---|---|---|
| 01 | `01-vampi-owasp-top10.sh` | curl, jq | Suite di test completa OWASP API Top 10. Registra un utente di test, ottiene un token di autenticazione, poi testa BOLA (accesso ai dati di altri utenti), autenticazione brute-force, esposizione eccessiva dei dati, autorizzazione a livello di funzione compromessa, pattern SSRF e sondaggio della configurazione di sicurezza. |
| 02 | `02-sqlmap-api.sh` | sqlmap | SQL injection contro endpoint REST API di VAmPI: ricerca utente, login (POST) e registrazione (POST) con payload JSON. |
| 03 | `03-arjun-param-discovery.sh` | arjun | Scoperta di parametri nascosti attraverso endpoint di Juice Shop, DVWA e VAmPI. Scopre parametri query non documentati che possono accettare payload di injection. |
| 04 | `04-ffuf-api-fuzz.sh` | ffuf | Fuzzing a due fasi: scoperta degli endpoint utilizzando una wordlist di percorsi API comuni (api, swagger, graphql, admin, .env, .git) attraverso tutti i prefissi delle applicazioni, poi fuzzing dei metodi HTTP (GET, POST, PUT, DELETE, PATCH, OPTIONS, TRACE, PROPFIND) contro gli endpoint chiave. |

**Comportamento Atteso di F5 XC:** API Discovery dovrebbe mappare gli endpoint API di VAmPI e identificare pattern di richiesta insoliti. Le policy di API Security dovrebbero segnalare tentativi di accesso non autorizzato, payload di injection nei corpi JSON e pattern di enumerazione. L'Inventario API in F5 XC mostrerà gli endpoint scoperti con il conteggio delle richieste.

## bot-simulation

**Script:** 4 | **Durata Stimata:** 5-10 minuti

Attività browser automatizzata e pattern di crawling simili a bot attraverso F5 XC. Include Chrome headless, Puppeteer con plugin stealth, crawling rapido e interazione automatizzata con i form. Ha come obiettivo tutte le applicazioni.

**Funzionalità F5 XC:** Bot Defense

| Ordine | Script | Strumenti | Descrizione |
|---|---|---|---|
| 01 | `01-headless-chrome.sh` | playwright | Navigazione con Chromium headless: carica il target, naviga tra i percorsi delle applicazioni, estrae titoli delle pagine e contenuti, cattura screenshot. Genera traffico basato su browser senza plugin stealth (rilevabile come automazione). |
| 02 | `02-puppeteer-stealth.sh` | puppeteer, puppeteer-extra-plugin-stealth | Navigazione stealth utilizzando puppeteer-extra con il plugin stealth. Tenta di eludere il rilevamento bot modificando le proprietà del navigator, WebGL e altre impronte digitali del browser. |
| 03 | `03-rapid-crawl.sh` | curl, wget | Richieste HTTP ad alta frequenza utilizzando curl e wget in rapida successione. Simula il comportamento di uno scraper senza ritardo tra le richieste e con rotazione aggressiva dello User-Agent. |
| 04 | `04-form-interaction.sh` | playwright | Compilazione e invio automatizzato di form contro il login DVWA, la registrazione Juice Shop e gli endpoint API VAmPI utilizzando le API di interazione form di Playwright. |

**Comportamento Atteso di F5 XC:** Bot Defense dovrebbe classificare il traffico come automatizzato basandosi sull'analisi dell'impronta digitale del browser, sulla temporizzazione delle richieste e sui risultati delle challenge JavaScript. Chrome headless senza stealth dovrebbe essere rilevato immediatamente. I test con Puppeteer stealth verificano se le tecniche avanzate di evasione aggirano il rilevamento. La dashboard Bot Defense mostrerà le categorie di classificazione bot e le azioni di mitigazione.

|---|---|---|---|

## waf-encoding-evasion

**Script:** 7 | **Durata Stimata:** 5-10 minuti

Attacchi di evasione con codifica multi-livello progettati per testare se un WAF normalizza e decodifica correttamente i payload prima dell'ispezione. Copre codifica URL singola, doppia e tripla; codifica entità HTML (decimale, esadecimale, nominata, con zero-padding); inserimento Unicode/UTF-8 (spazio a larghezza zero, BOM, caratteri fullwidth, trattino morbido, sequenze overlong); codifica mista/annidata multi-livello (entità HTML codificate in URL, doppio URL dentro entità, stack a 3 livelli); injection di byte null; codifica IIS %uXXXX; manipolazione del case; splitting del corpo con chunked transfer encoding; evasione parametri Base64; e payload codificati in header/cookie. Ha come obiettivo qualsiasi applicazione.

**Funzionalità F5 XC:** WAF (profondità di normalizzazione della codifica)

| Ordine | Script | Strumenti | Descrizione |
|---|---|---|---|
| 01 | `01-url-encoding.sh` | curl | Codifica URL singola, doppia e tripla di payload SQLi, XSS e path traversal attraverso percorsi endpoint multipli. Testa se il WAF decodifica i parametri URL in modo ricorsivo. |
| 02 | `02-html-entity-encoding.sh` | curl, python3 | Codifica entità HTML decimale, esadecimale, nominata e con zero-padding di payload di attacco sia in parametri GET che in corpi POST. Include template injection tramite `&#123;&#123;7*7&#125;&#125;`. |
| 03 | `03-unicode-utf8-evasion.sh` | curl | Inserimento di spazio a larghezza zero (U+200B), BOM (U+FEFF), caratteri fullwidth, trattino morbido e ZWNJ in parole chiave SQL e tag XSS. Testa sequenze UTF-8 overlong (2 byte e 3 byte). |
| 04 | `04-mixed-nested-encoding.sh` | curl | Payload multi-livello: entità HTML codificate in URL, doppio URL dentro entità, stack a triplo livello, corpo JSON con escape Unicode (`<`) e il pattern specifico di template injection `{{7*7}}` dell'utente. |
| 05 | `05-null-byte-iis-base64.sh` | curl | Injection di byte null (`%00`) per bypass dell'estensione, codifica Unicode IIS `%uXXXX` e payload codificati in Base64 nei parametri URL con flag di decodifica. |
| 06 | `06-case-chunked-evasion.sh` | curl | Randomizzazione del case combinata con codifica (`%3CsCrIpT%3E`), chunked transfer encoding per dividere parole chiave SQL/XSS attraverso chunk HTTP, sonde di conflitto Content-Length/Transfer-Encoding e separatori di percorso alternativi (`%5c`, `%c0%af`). |
| 07 | `07-header-cookie-evasion.sh` | curl | Payload di attacco codificati iniettati tramite header Cookie, Referer, User-Agent, X-Forwarded-For e X-Original-URL. Testa se il WAF ispeziona e decodifica tutti i campi degli header HTTP, non solo i parametri query e i corpi POST. |

**Comportamento Atteso di F5 XC:** Un WAF correttamente configurato dovrebbe normalizzare tutti i livelli di codifica prima della corrispondenza dei pattern. I payload con codifica singola dovrebbero essere bloccati immediatamente. I test con doppia e tripla codifica rivelano la profondità della pipeline di decodifica del WAF. I test di inserimento Unicode rivelano se i caratteri invisibili vengono rimossi prima della corrispondenza delle parole chiave. Risultati in cui le varianti codificate passano mentre gli equivalenti in testo semplice vengono bloccati indicano lacune di normalizzazione nella policy del WAF.
