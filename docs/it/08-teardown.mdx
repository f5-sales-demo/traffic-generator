Al termine della sessione di laboratorio, distruggere tutte le risorse Azure per interrompere la fatturazione.

## Terraform Destroy

Dalla directory `terraform/`:

```bash
cd terraform/

terraform plan -destroy

terraform destroy
```

Terraform chiederà una conferma. Digitare `yes` per procedere.

## Verifica della pulizia

Confermare che tutte le risorse siano state rimosse:

```bash
RG_NAME=$(terraform output -raw resource_group_name)

az group show --name "${RG_NAME}" 2>&1 | grep -q "ResourceGroupNotFound" \
  && echo "Resource group deleted" \
  || echo "Resource group still exists"

az resource list --resource-group "${RG_NAME}" -o table 2>/dev/null
```

## Pulizia dello stato locale

Rimuovere lo stato di Terraform e i file memorizzati nella cache:

```bash
rm -rf .terraform .terraform.lock.hcl terraform.tfstate terraform.tfstate.backup
```

## Conservazione dei risultati (Opzionale)

Se si desidera conservare i risultati della suite di traffico prima di distruggere la VM, copiarli in locale:

```bash
TGEN_IP=$(terraform output -raw public_ip)

# Copy all results to local machine
scp -r azureuser@${TGEN_IP}:/opt/traffic-generator/results/ ./traffic-results/

# Or copy just the latest run
LATEST=$(ssh azureuser@${TGEN_IP} 'ls -t /opt/traffic-generator/results/ | head -1')
scp -r azureuser@${TGEN_IP}:/opt/traffic-generator/results/${LATEST} ./traffic-results/
```

## Pulizia di F5 XC

Se si desidera rimuovere anche la configurazione di F5 XC verso cui il generatore di traffico era indirizzato:

1. Eliminare l'HTTP load balancer
2. Eliminare l'origin pool
3. Eliminare eventuali policy WAF, configurazioni di Bot Defense, policy di API Security o configurazioni CSD create per i test

Queste risorse F5 XC non comportano costi Azure, ma dovrebbero essere pulite quando l'ambiente demo non è più necessario.

:::tip
Per un utilizzo demo ricorrente, considerare di automatizzare con script l'intero ciclo di vita: `cd terraform && terraform apply` prima di una demo, eseguire le suite, quindi `terraform destroy` al termine. Il tier standard viene provisionato in 15-20 minuti, rendendo pratico il setup e lo smantellamento in giornata.
:::
