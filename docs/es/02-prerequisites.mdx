## Obligatorios

### Suscripción de Azure

Una suscripción activa de Azure con rol Contributor o permisos equivalentes para crear:

- Grupos de recursos
- Redes virtuales y subredes
- Grupos de seguridad de red
- Direcciones IP públicas
- Máquinas virtuales (Standard_F16s_v2 por defecto)

### Azure CLI

Instale y autentíquese en Azure CLI para obtener su ID de suscripción:

```bash
curl -sL https://aka.ms/InstallAzureCLIDeb | sudo bash

az login

az account show --query '{name:name, id:id}' -o table
```

Copie el valor de `id` -- este es el `subscription_id` requerido por Terraform.

### Terraform

Se requiere Terraform >= 1.5:

```bash
terraform version
```

Si no está instalado, utilice el repositorio APT de HashiCorp:

```bash
wget -O- https://apt.releases.hashicorp.com/gpg | sudo gpg --dearmor -o /usr/share/keyrings/hashicorp-archive-keyring.gpg
echo "deb [signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] https://apt.releases.hashicorp.com $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/hashicorp.list
sudo apt update && sudo apt install terraform
```

### Par de claves SSH

Un par de claves SSH para el acceso a la VM. Se recomienda Ed25519:

```bash
ssh-keygen -t ed25519 -f ~/.ssh/traffic-generator-key -N ""
```

### Acceso de red

La VM del Generador de Tráfico requiere acceso HTTPS saliente durante el aprovisionamiento para descargar herramientas desde:

- GitHub Releases (nuclei, dalfox, ffuf, gobuster, feroxbuster, subfinder, httpx, amass)
- PyPI (sqlmap, scapy, impacket, arjun, pwntools, mitmproxy, sslyze, playwright)
- Registro npm (puppeteer, puppeteer-extra-plugin-stealth)
- Repositorios APT de Ubuntu (nikto, nmap, masscan, hydra, tshark y otros paquetes del sistema)
- Repositorios Git (testssl.sh, recon-ng, spiderfoot, SecLists)

Si su suscripción de Azure aplica reglas de firewall saliente o restricciones de NSG, asegúrese de que el tráfico HTTPS (puerto 443) hacia estas fuentes esté permitido durante la ventana de aprovisionamiento inicial (15-25 minutos).

### Componentes previamente desplegados

Antes de desplegar el Generador de Tráfico, debe tener:

1. **Servidor de origen** desplegado y accesible (consulte la [documentación del servidor de origen](https://f5xc-salesdemos.github.io/origin-server/03-deploy/))
2. **Balanceador de carga HTTP de F5 XC** configurado con el servidor de origen como backend
3. Una **política WAF** adjunta al balanceador de carga (para la validación del tráfico WAF)

La variable de Terraform `target_fqdn` debe establecerse con el dominio público del balanceador de carga de F5 XC.

:::tip
Utilice `terraform destroy` cuando el laboratorio no esté en uso para evitar cargos continuos. Consulte [Desmontaje](../08-teardown/) para conocer el procedimiento.
:::

:::note
Standard_F16s_v2 (32 GiB de RAM) es el valor predeterminado porque varias herramientas se ejecutan simultáneamente durante la ejecución de las suites. La automatización de navegadores (Playwright, Puppeteer) con Chromium en modo headless es el principal consumidor de memoria. SKUs F-series más pequeños funcionarán para suites sin navegador, pero pueden resultar ajustados al ejecutar simulación de bots.
:::
