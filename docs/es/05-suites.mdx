Cada suite de tráfico es un directorio bajo `/opt/traffic-generator/suites/` que contiene scripts de shell numerados ejecutados en orden por `runner.sh`. Cada script acepta el FQDN de destino como su primer argumento y escribe la salida tanto en stdout como en el directorio de resultados.

## api-attacks

**Scripts:** 4 | **Duración Estimada:** 10-17 minutos

Pruebas del OWASP API Security Top 10 contra VAmPI a través de F5 XC. Registra un usuario de prueba, obtiene un token de autenticación y luego prueba BOLA, autenticación por fuerza bruta, exposición excesiva de datos, autorización de nivel de función rota, patrones SSRF, inyección SQL contra endpoints de API REST, descubrimiento de parámetros ocultos y fuzzing de endpoints. Apunta a la aplicación VAmPI.

**Característica de F5 XC:** API Security / API Discovery

| Orden | Script | Herramientas | Descripción |
|---|---|---|---|
| 01 | `01-vampi-owasp-top10.sh` | curl, jq | Suite de pruebas integral del OWASP API Top 10. Registra un usuario de prueba, obtiene un token de autenticación y luego prueba BOLA (acceso a datos de otros usuarios), autenticación por fuerza bruta, exposición excesiva de datos, autorización de nivel de función rota, patrones SSRF y sondeo de configuración de seguridad incorrecta. |
| 02 | `02-sqlmap-api.sh` | sqlmap | Inyección SQL contra endpoints de API REST de VAmPI: búsqueda de usuario, login (POST) y registro (POST) con payloads JSON. |
| 03 | `03-arjun-param-discovery.sh` | arjun | Descubrimiento de parámetros ocultos en endpoints de Juice Shop, DVWA y VAmPI. Descubre parámetros de consulta no documentados que pueden aceptar payloads de inyección. |
| 04 | `04-ffuf-api-fuzz.sh` | ffuf | Fuzzing en dos fases: descubrimiento de endpoints usando una lista de palabras de rutas API comunes (api, swagger, graphql, admin, .env, .git) en todos los prefijos de aplicación, luego fuzzing de métodos HTTP (GET, POST, PUT, DELETE, PATCH, OPTIONS, TRACE, PROPFIND) contra endpoints clave. |

**Comportamiento Esperado de F5 XC:** API Discovery debería mapear los endpoints de la API VAmPI e identificar patrones de solicitud inusuales. Las políticas de API Security deberían marcar intentos de acceso no autorizado, payloads de inyección en cuerpos JSON y patrones de enumeración. El Inventario de API en F5 XC mostrará los endpoints descubiertos con conteos de solicitudes.

**Característica de F5 XC:** Bot Defense

| Orden | Script | Herramientas | Descripción |
|---|---|---|---|
| 01 | `01-headless-chrome.sh` | playwright | Navegación con Chromium headless: carga el objetivo, navega entre rutas de aplicación, extrae títulos y contenido de páginas, toma capturas de pantalla. Genera tráfico basado en navegador sin plugins stealth (detectable como automatización). |
| 02 | `02-puppeteer-stealth.sh` | puppeteer, puppeteer-extra-plugin-stealth | Navegación stealth usando puppeteer-extra con el plugin stealth. Intenta evadir la detección de bots parcheando propiedades del navigator, WebGL y otras huellas digitales del navegador. |
| 03 | `03-rapid-crawl.sh` | curl, wget | Solicitudes HTTP de alta frecuencia usando curl y wget en sucesión rápida. Simula comportamiento de scraper sin retraso entre solicitudes y rotación agresiva de User-Agent. |
| 04 | `04-form-interaction.sh` | playwright | Llenado y envío automatizado de formularios contra el login de DVWA, registro de Juice Shop y endpoints de API de VAmPI usando las APIs de interacción con formularios de Playwright. |

**Comportamiento Esperado de F5 XC:** Bot Defense debería clasificar el tráfico como automatizado basándose en el análisis de huella digital del navegador, temporización de solicitudes y resultados de desafío JavaScript. Chrome headless sin stealth debería ser detectado inmediatamente. Las pruebas con Puppeteer Stealth verifican si las técnicas avanzadas de evasión evaden la detección. El panel de Bot Defense mostrará categorías de clasificación de bots y acciones de mitigación.

## waf-encoding-evasion

**Scripts:** 7 | **Duración Estimada:** 5-10 minutos

Ataques de evasión por codificación multicapa diseñados para probar si un WAF normaliza y decodifica correctamente los payloads antes de la inspección. Cubre codificación URL simple, doble y triple; codificación de entidades HTML (decimal, hexadecimal, con nombre, con relleno de ceros); inserción Unicode/UTF-8 (espacio de ancho cero, BOM, caracteres de ancho completo, guión suave, secuencias overlong); codificación mixta/anidada multicapa (entidades HTML codificadas en URL, URL doble dentro de entidades, pilas de 3 capas); inyección de byte nulo; codificación IIS %uXXXX; manipulación de mayúsculas/minúsculas; división del cuerpo por codificación de transferencia chunked; evasión de parámetros en Base64; y payloads codificados en cabeceras/cookies. Apunta a cualquier aplicación.

**Característica de F5 XC:** WAF (profundidad de normalización de codificación)

| Orden | Script | Herramientas | Descripción |
|---|---|---|---|
| 01 | `01-url-encoding.sh` | curl | Codificación URL simple, doble y triple de payloads SQLi, XSS y recorrido de directorios a través de múltiples rutas de endpoint. Prueba si el WAF decodifica parámetros URL recursivamente. |
| 02 | `02-html-entity-encoding.sh` | curl, python3 | Codificación de entidades HTML decimal, hexadecimal, con nombre y con relleno de ceros de payloads de ataque tanto en parámetros GET como en cuerpos POST. Incluye inyección de plantillas vía `&#123;&#123;7*7&#125;&#125;`. |
| 03 | `03-unicode-utf8-evasion.sh` | curl | Inserción de espacio de ancho cero (U+200B), BOM (U+FEFF), caracteres de ancho completo, guión suave y ZWNJ en palabras clave SQL y etiquetas XSS. Prueba secuencias UTF-8 overlong (2 bytes y 3 bytes). |
| 04 | `04-mixed-nested-encoding.sh` | curl | Payloads multicapa: entidades HTML codificadas en URL, URL doble dentro de entidades, pilas de triple capa, cuerpo JSON con escapes Unicode (`<`) y el patrón específico de inyección de plantillas `{{7*7}}` del usuario. |
| 05 | `05-null-byte-iis-base64.sh` | curl | Inyección de byte nulo (`%00`) para bypass de extensión, codificación Unicode IIS `%uXXXX` y payloads codificados en Base64 en parámetros URL con indicadores de decodificación. |
| 06 | `06-case-chunked-evasion.sh` | curl | Aleatorización de mayúsculas/minúsculas combinada con codificación (`%3CsCrIpT%3E`), codificación de transferencia chunked para dividir palabras clave SQL/XSS entre fragmentos HTTP, sondas de conflicto Content-Length/Transfer-Encoding y separadores de ruta alternativos (`%5c`, `%c0%af`). |
| 07 | `07-header-cookie-evasion.sh` | curl | Payloads de ataque codificados inyectados vía cabeceras Cookie, Referer, User-Agent, X-Forwarded-For y X-Original-URL. Prueba si el WAF inspecciona y decodifica todos los campos de cabecera HTTP, no solo parámetros de consulta y cuerpos POST. |

**Comportamiento Esperado de F5 XC:** Un WAF correctamente configurado debería normalizar todas las capas de codificación antes de la coincidencia de patrones. Los payloads con codificación simple deberían ser bloqueados inmediatamente. Las pruebas de codificación doble y triple revelan la profundidad del pipeline de decodificación del WAF. Las pruebas de inserción Unicode revelan si los caracteres invisibles se eliminan antes de la coincidencia de palabras clave. Los resultados donde variantes codificadas pasan mientras los equivalentes en texto plano son bloqueados indican brechas de normalización en la política del WAF.
