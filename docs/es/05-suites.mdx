Cada suite de tráfico es un directorio bajo `/opt/traffic-generator/suites/` que contiene scripts de shell numerados ejecutados en orden por `runner.sh`. Cada script acepta el FQDN objetivo como primer argumento y escribe la salida tanto en stdout como en el directorio de resultados.

## api-attacks

**Scripts:** 4 | **Duración estimada:** 10-17 minutos

Pruebas del OWASP API Security Top 10 contra VAmPI a través de F5 XC. Registra un usuario de prueba, obtiene un token de autenticación y luego prueba BOLA, autenticación por fuerza bruta, exposición excesiva de datos, autorización rota a nivel de función, patrones SSRF, inyección SQL contra endpoints de API REST, descubrimiento de parámetros ocultos y fuzzing de endpoints. Dirigido a la aplicación VAmPI.

**Característica de F5 XC:** API Security / API Discovery

| Orden | Script | Herramientas | Descripción |
|---|---|---|---|
| 01 | `01-vampi-owasp-top10.sh` | curl, jq | Suite completa de pruebas OWASP API Top 10. Registra un usuario de prueba, obtiene un token de autenticación y luego prueba BOLA (acceso a datos de otros usuarios), autenticación por fuerza bruta, exposición excesiva de datos, autorización rota a nivel de función, patrones SSRF y sondeo de configuración de seguridad incorrecta. |
| 02 | `02-sqlmap-api.sh` | sqlmap | Inyección SQL contra endpoints de API REST de VAmPI: búsqueda de usuarios, login (POST) y registro (POST) con payloads JSON. |
| 03 | `03-arjun-param-discovery.sh` | arjun | Descubrimiento de parámetros ocultos en endpoints de Juice Shop, DVWA y VAmPI. Descubre parámetros de consulta no documentados que pueden aceptar payloads de inyección. |
| 04 | `04-ffuf-api-fuzz.sh` | ffuf | Fuzzing en dos fases: descubrimiento de endpoints usando una lista de palabras de rutas API comunes (api, swagger, graphql, admin, .env, .git) en todos los prefijos de aplicación, luego fuzzing de métodos HTTP (GET, POST, PUT, DELETE, PATCH, OPTIONS, TRACE, PROPFIND) contra endpoints clave. |

**Comportamiento esperado de F5 XC:** API Discovery debería mapear los endpoints de la API de VAmPI e identificar patrones de solicitud inusuales. Las políticas de API Security deberían señalar intentos de acceso no autorizado, payloads de inyección en cuerpos JSON y patrones de enumeración. El inventario de APIs en F5 XC mostrará los endpoints descubiertos con conteos de solicitudes.

## bot-simulation

**Scripts:** 4 | **Duración estimada:** 5-10 minutos

Actividad automatizada de navegador y patrones de rastreo tipo bot a través de F5 XC. Incluye Chrome headless, Puppeteer con plugin de sigilo, rastreo rápido e interacción automatizada con formularios. Dirigido a todas las aplicaciones.

**Característica de F5 XC:** Bot Defense

| Orden | Script | Herramientas | Descripción |
|---|---|---|---|
| 01 | `01-headless-chrome.sh` | playwright | Navegación con Chromium headless: carga el objetivo, navega entre rutas de aplicación, extrae títulos y contenido de páginas, toma capturas de pantalla. Genera tráfico basado en navegador sin plugins de sigilo (detectable como automatización). |
| 02 | `02-puppeteer-stealth.sh` | puppeteer, puppeteer-extra-plugin-stealth | Navegación sigilosa usando puppeteer-extra con el plugin de sigilo. Intenta evadir la detección de bots parcheando propiedades del navegador, WebGL y otras huellas digitales del navegador. |
| 03 | `03-rapid-crawl.sh` | curl, wget | Solicitudes HTTP de alta frecuencia usando curl y wget en rápida sucesión. Simula comportamiento de scraper sin demora entre solicitudes y rotación agresiva de User-Agent. |
| 04 | `04-form-interaction.sh` | playwright | Llenado y envío automatizado de formularios contra el login de DVWA, registro de Juice Shop y endpoints de API de VAmPI usando las APIs de interacción con formularios de Playwright. |

**Comportamiento esperado de F5 XC:** Bot Defense debería clasificar el tráfico como automatizado basándose en el análisis de huella digital del navegador, temporización de solicitudes y resultados del desafío JavaScript. Chrome headless sin sigilo debería ser detectado inmediatamente. Las pruebas con Puppeteer sigiloso verifican si las técnicas avanzadas de evasión eluden la detección. El panel de Bot Defense mostrará categorías de clasificación de bots y acciones de mitigación.

## demoapp-attacks

**Scripts:** 2 | **Duración estimada:** 2-3 minutos

Payloads de ataque dirigidos contra los endpoints de prueba WAF integrados de F5 DemoApp (`/WAF/SQL`, `/WAF/XSS`, `/WAF/DIR`). Envía payloads de inyección SQL, XSS y recorrido de directorios directamente a los endpoints diseñados para demostrar el comportamiento de bloqueo del WAF. Use esta suite cuando dirija el tráfico al servidor de contenido DemoApp en lugar de las aplicaciones del servidor de origen estándar.

**Característica de F5 XC:** WAF (Web Application Firewall)

| Orden | Script | Herramientas | Descripción |
|---|---|---|---|
| 01 | `01-sqli-waf-endpoint.sh` | curl, python3 | 15 payloads de inyección SQL contra `/WAF/SQL?age=` incluyendo UNION SELECT, DROP TABLE, WAITFOR DELAY, xp_cmdshell, bypass basado en comentarios y payloads de lógica booleana. |
| 02 | `02-xss-waf-endpoint.sh` | curl, python3 | 18 payloads XSS contra `/WAF/XSS?update=` incluyendo etiquetas script, manejadores de eventos (onerror, onload, ontoggle, onfocus), eval en base64, expresión de estilo, payloads políglotas y exfiltración de cookies. |

**Comportamiento esperado de F5 XC:** El WAF debería bloquear las solicitudes que contengan payloads de inyección SQL y XSS y devolver una página de bloqueo. El panel de Eventos de Seguridad mostrará categorías de violación y coincidencias de firmas para cada solicitud bloqueada.

| Orden | Script | Herramientas | Descripción |

## waf-encoding-evasion

**Scripts:** 7 | **Duración estimada:** 5-10 minutos

Ataques de evasión con codificación multicapa diseñados para probar si un WAF normaliza y decodifica correctamente los payloads antes de la inspección. Cubre codificación URL simple, doble y triple; codificación de entidades HTML (decimal, hexadecimal, nombrada, con relleno de ceros); inserción Unicode/UTF-8 (espacio de ancho cero, BOM, caracteres de ancho completo, guión suave, secuencias overlong); codificación mixta/anidada multicapa (entidades HTML codificadas en URL, URL doble dentro de entidades, pilas de 3 capas); inyección de byte nulo; codificación IIS %uXXXX; manipulación de mayúsculas/minúsculas; división de cuerpo con codificación de transferencia fragmentada; evasión de parámetros Base64; y payloads codificados en encabezados/cookies. Dirigido a cualquier aplicación.

**Característica de F5 XC:** WAF (profundidad de normalización de codificación)

| Orden | Script | Herramientas | Descripción |
|---|---|---|---|
| 01 | `01-url-encoding.sh` | curl | Codificación URL simple, doble y triple de payloads SQLi, XSS y recorrido de directorios en múltiples rutas de endpoint. Prueba si el WAF decodifica parámetros URL de forma recursiva. |
| 02 | `02-html-entity-encoding.sh` | curl, python3 | Codificación de entidades HTML decimal, hexadecimal, nombrada y con relleno de ceros de payloads de ataque tanto en parámetros GET como en cuerpos POST. Incluye inyección de plantillas vía `&#123;&#123;7*7&#125;&#125;`. |
| 03 | `03-unicode-utf8-evasion.sh` | curl | Inserción de espacio de ancho cero (U+200B), BOM (U+FEFF), caracteres de ancho completo, guión suave y ZWNJ en palabras clave SQL y etiquetas XSS. Prueba secuencias UTF-8 overlong (2 bytes y 3 bytes). |
| 04 | `04-mixed-nested-encoding.sh` | curl | Payloads multicapa: entidades HTML codificadas en URL, URL doble dentro de entidades, pilas de triple capa, cuerpo JSON con escapes Unicode (`<`) y el patrón específico de inyección de plantillas `{{7*7}}` del usuario. |
| 05 | `05-null-byte-iis-base64.sh` | curl | Inyección de byte nulo (`%00`) para bypass de extensión, codificación Unicode IIS `%uXXXX` y payloads codificados en Base64 en parámetros URL con indicadores de decodificación. |
| 06 | `06-case-chunked-evasion.sh` | curl | Aleatorización de mayúsculas/minúsculas combinada con codificación (`%3CsCrIpT%3E`), codificación de transferencia fragmentada para dividir palabras clave SQL/XSS entre fragmentos HTTP, sondas de conflicto Content-Length/Transfer-Encoding y separadores de ruta alternativos (`%5c`, `%c0%af`). |
| 07 | `07-header-cookie-evasion.sh` | curl | Payloads de ataque codificados inyectados vía encabezados Cookie, Referer, User-Agent, X-Forwarded-For y X-Original-URL. Prueba si el WAF inspecciona y decodifica todos los campos de encabezados HTTP, no solo parámetros de consulta y cuerpos POST. |

**Comportamiento esperado de F5 XC:** Un WAF correctamente configurado debería normalizar todas las capas de codificación antes de la coincidencia de patrones. Los payloads con codificación simple deberían ser bloqueados inmediatamente. Las pruebas de codificación doble y triple revelan la profundidad del pipeline de decodificación del WAF. Las pruebas de inserción Unicode revelan si los caracteres invisibles se eliminan antes de la coincidencia de palabras clave. Los resultados donde las variantes codificadas pasan mientras los equivalentes en texto plano son bloqueados indican brechas de normalización en la política del WAF.
