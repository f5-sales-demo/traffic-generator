## Propósito

Este componente proporciona una plataforma automatizada de generación de tráfico que produce tráfico de ataque, escaneos de reconocimiento, simulación de bots y abuso de API contra un balanceador de carga HTTP de F5 Distributed Cloud. Es el "atacante" en una arquitectura de demostración típica: la fuente de tráfico malicioso y sospechoso que las funciones de seguridad de F5 XC están diseñadas para detectar y bloquear.

En la arquitectura de demostración:

```
Traffic Generator VM -> F5 XC HTTP LB (WAF/Bot/API/CSD) -> Origin Server VM
```

El Generador de Tráfico envía solicitudes al FQDN público del balanceador de carga de F5 XC. La plataforma F5 XC inspecciona y filtra el tráfico antes de reenviar las solicitudes legítimas al servidor de origen. El operador luego revisa los registros de eventos de seguridad de F5 XC para demostrar la detección y aplicación de políticas.

```mermaid

    TG[Traffic Generator VM<br/>Ubuntu 24.04<br/>Standard_F16s_v2] -->|Attack Traffic<br/>HTTPS| XCHLB[F5 XC HTTP LB<br/>WAF / Bot Defense<br/>API Security / CSD]
    XCHLB -->|Filtered Traffic<br/>HTTP| ORIGIN[Origin Server VM<br/>nginx + Docker Apps]
    TG -->|Direct Baseline<br/>HTTP optional| ORIGIN
    RUNNER[runner.sh] --> SUITES[19 Traffic Suites]
    SUITES --> TG
```

La VM del Generador de Tráfico se ejecuta en Azure con:

- **Ubuntu 24.04 LTS** como imagen base
- **Más de 50 herramientas de seguridad** instaladas mediante cloud-init durante el aprovisionamiento
- **19 suites de tráfico organizadas** con scripts numerados ejecutados en orden
- **runner.sh** como orquestador para la ejecución de suites con registro de resultados
- **config.env** para la configuración del objetivo (FQDN, IP de origen)

## Instalación por niveles

El Generador de Tráfico admite dos niveles de instalación controlados por la variable de Terraform `tool_tier`:

### Nivel estándar (predeterminado)

Instala todas las herramientas listadas en el catálogo de herramientas excepto ZAP y Metasploit. El aprovisionamiento se completa en 15-20 minutos. Este nivel cubre las 19 suites de tráfico y es suficiente para la mayoría de los escenarios de demostración.

### Nivel completo

Agrega OWASP ZAP y Metasploit Framework sobre el nivel estándar. El aprovisionamiento tarda aproximadamente 25 minutos. Estas herramientas son grandes (ZAP ~500 MiB, Metasploit ~1 GiB) y solo se necesitan para demostraciones avanzadas de escaneo de vulnerabilidades.

Consulte la calculadora de precios de Azure para conocer los costos actuales de VM. El Standard_F16s_v2 predeterminado es una instancia optimizada para cómputo adecuada para la generación sostenida de tráfico.

:::tip
Use `terraform destroy` cuando el laboratorio no esté en uso para evitar cargos continuos. Consulte [Desmontaje](../08-teardown/) para conocer el procedimiento.
:::

## Puntos de integración

Este componente se integra con otros dos componentes de demostración:

- **Servidor de origen** -- El backend de destino que aloja Juice Shop, DVWA, VAmPI, httpbin y whoami. El Generador de Tráfico envía tráfico de ataque a través de F5 XC para alcanzar estas aplicaciones. Consulte [Integración](../07-integrate/) para los detalles completos de la arquitectura.

- **Demostración CSD** -- La aplicación de demostración de Client-Side Defense en el servidor de origen. La suite de tráfico `javascript-exploits` genera payloads de inyección de scripts estilo Magecart que F5 XC Client-Side Defense detecta. Esto valida la funcionalidad de CSD Fase 2.

## Diseño modular de componentes

Cada componente del laboratorio es autónomo y se despliega de forma independiente:

- **Generador de Tráfico** (este componente) proporciona la fuente de ataques
- **Servidor de Origen** proporciona los objetivos de aplicaciones vulnerables
- **Simulador CDN** proporciona la capa de caché en el borde CDN (opcional)
- **Configuración de F5 XC** proporciona las políticas de WAF, Bot Defense, API Security y CSD

El operador humano o el asistente de IA agrega componentes uno a la vez. Despliegue primero el servidor de origen, configure F5 XC delante de él y luego despliegue el generador de tráfico apuntando al FQDN del balanceador de carga de F5 XC.
