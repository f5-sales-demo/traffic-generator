Cada suíte de tráfego é um diretório em `/opt/traffic-generator/suites/` contendo scripts shell numerados executados em ordem pelo `runner.sh`. Cada script aceita o FQDN alvo como primeiro argumento e grava a saída tanto no stdout quanto no diretório de resultados.

## api-attacks

**Scripts:** 4 | **Duração Estimada:** 10-17 minutos

Testes do OWASP API Security Top 10 contra o VAmPI através do F5 XC. Registra um usuário de teste, obtém um token de autenticação e então testa BOLA, autenticação por força bruta, exposição excessiva de dados, autorização de nível de função quebrada, padrões SSRF, injeção SQL contra endpoints de API REST, descoberta de parâmetros ocultos e fuzzing de endpoints. Tem como alvo a aplicação VAmPI.

**Funcionalidade F5 XC:** API Security / API Discovery

| Ordem | Script | Ferramentas | Descrição |
|---|---|---|---|
| 01 | `01-vampi-owasp-top10.sh` | curl, jq | Suíte abrangente de testes OWASP API Top 10. Registra um usuário de teste, obtém um token de autenticação e então testa BOLA (acesso a dados de outros usuários), autenticação por força bruta, exposição excessiva de dados, autorização de nível de função quebrada, padrões SSRF e sondagem de configuração de segurança incorreta. |
| 02 | `02-sqlmap-api.sh` | sqlmap | Injeção SQL contra endpoints da API REST do VAmPI: pesquisa de usuários, login (POST) e registro (POST) com payloads JSON. |
| 03 | `03-arjun-param-discovery.sh` | arjun | Descoberta de parâmetros ocultos em endpoints do Juice Shop, DVWA e VAmPI. Descobre parâmetros de query não documentados que podem aceitar payloads de injeção. |
| 04 | `04-ffuf-api-fuzz.sh` | ffuf | Fuzzing em duas fases: descoberta de endpoints usando uma wordlist de caminhos comuns de API (api, swagger, graphql, admin, .env, .git) em todos os prefixos de aplicação, seguido de fuzzing de métodos HTTP (GET, POST, PUT, DELETE, PATCH, OPTIONS, TRACE, PROPFIND) contra endpoints-chave. |

**Comportamento Esperado do F5 XC:** O API Discovery deve mapear os endpoints da API do VAmPI e identificar padrões de requisição incomuns. As políticas de API Security devem sinalizar tentativas de acesso não autorizado, payloads de injeção em corpos JSON e padrões de enumeração. O Inventário de API no F5 XC mostrará os endpoints descobertos com contagens de requisições.

## bot-simulation

**Scripts:** 4 | **Duração Estimada:** 5-10 minutos

Atividade automatizada de navegador e padrões de crawling semelhantes a bots através do F5 XC. Inclui Chrome headless, Puppeteer com plugin stealth, crawling rápido e interação automatizada com formulários. Tem como alvo todas as aplicações.

**Funcionalidade F5 XC:** Bot Defense

| Ordem | Script | Ferramentas | Descrição |
|---|---|---|---|
| 01 | `01-headless-chrome.sh` | playwright | Navegação com Chromium headless: carrega o alvo, navega entre caminhos de aplicações, extrai títulos e conteúdo de páginas, captura screenshots. Gera tráfego baseado em navegador sem plugins stealth (detectável como automação). |
| 02 | `02-puppeteer-stealth.sh` | puppeteer, puppeteer-extra-plugin-stealth | Navegação stealth usando puppeteer-extra com o plugin stealth. Tenta evadir a detecção de bots modificando propriedades do navigator, WebGL e outras impressões digitais do navegador. |
| 03 | `03-rapid-crawl.sh` | curl, wget | Requisições HTTP de alta frequência usando curl e wget em sucessão rápida. Simula comportamento de scraper sem atraso entre requisições e rotação agressiva de User-Agent. |
| 04 | `04-form-interaction.sh` | playwright | Preenchimento e envio automatizado de formulários contra login do DVWA, registro do Juice Shop e endpoints da API do VAmPI usando as APIs de interação com formulários do Playwright. |

**Comportamento Esperado do F5 XC:** O Bot Defense deve classificar o tráfego como automatizado com base na análise de impressão digital do navegador, temporização de requisições e resultados de desafios JavaScript. O Chrome headless sem stealth deve ser detectado imediatamente. Os testes com Puppeteer stealth verificam se técnicas avançadas de evasão contornam a detecção. O painel do Bot Defense mostrará categorias de classificação de bots e ações de mitigação.

## demoapp-attacks

Envia payloads de injeção SQL, XSS e travessia de diretório diretamente nos endpoints projetados para demonstrar o comportamento de bloqueio do WAF. Use esta suíte ao direcionar o servidor de conteúdo DemoApp em vez das aplicações padrão do servidor de origem.

**Funcionalidade F5 XC:** WAF (Web Application Firewall)

| Ordem | Script | Ferramentas | Descrição |
|---|---|---|---|
| 01 | `01-sqli-waf-endpoint.sh` | curl, python3 | 15 payloads de injeção SQL contra `/WAF/SQL?age=` incluindo UNION SELECT, DROP TABLE, WAITFOR DELAY, xp_cmdshell, bypass baseado em comentários e payloads de lógica booleana. |
| 02 | `02-xss-waf-endpoint.sh` | curl, python3 | 18 payloads XSS contra `/WAF/XSS?update=` incluindo tags script, manipuladores de eventos (onerror, onload, ontoggle, onfocus), eval em base64, expressão de estilo, payloads poliglotas e exfiltração de cookies. |

**Comportamento Esperado do F5 XC:** O WAF deve bloquear requisições contendo payloads de injeção SQL e XSS e retornar uma página de bloqueio. O painel de Eventos de Segurança mostrará categorias de violação e correspondências de assinatura para cada requisição bloqueada.

## waf-encoding-evasion

**Scripts:** 7 | **Duração Estimada:** 5-10 minutos

Ataques de evasão por codificação multicamada projetados para testar se um WAF normaliza e decodifica adequadamente os payloads antes da inspeção. Cobre codificação URL simples, dupla e tripla; codificação de entidade HTML (decimal, hexadecimal, nomeada, com zeros); inserção Unicode/UTF-8 (espaço de largura zero, BOM, caracteres fullwidth, hífen suave, sequências overlength); codificação multicamada mista/aninhada (entidades HTML codificadas em URL, URL dupla dentro de entidades, pilhas de 3 camadas); injeção de byte nulo; codificação IIS %uXXXX; manipulação de maiúsculas/minúsculas; divisão de corpo por codificação de transferência chunked; evasão de parâmetros Base64; e payloads codificados em cabeçalhos/cookies. Tem como alvo qualquer aplicação.

**Funcionalidade F5 XC:** WAF (profundidade de normalização de codificação)

| Ordem | Script | Ferramentas | Descrição |
|---|---|---|---|
| 01 | `01-url-encoding.sh` | curl | Codificação URL simples, dupla e tripla de payloads de SQLi, XSS e travessia de diretório em múltiplos caminhos de endpoint. Testa se o WAF decodifica parâmetros URL recursivamente. |
| 02 | `02-html-entity-encoding.sh` | curl, python3 | Codificação de entidade HTML decimal, hexadecimal, nomeada e com zeros de payloads de ataque em parâmetros GET e corpos POST. Inclui injeção de template via `&#123;&#123;7*7&#125;&#125;`. |
| 03 | `03-unicode-utf8-evasion.sh` | curl | Inserção de espaço de largura zero (U+200B), BOM (U+FEFF), caractere fullwidth, hífen suave e ZWNJ em palavras-chave SQL e tags XSS. Testa sequências UTF-8 overlength (2 bytes e 3 bytes). |
| 04 | `04-mixed-nested-encoding.sh` | curl | Payloads multicamada: entidades HTML codificadas em URL, URL dupla dentro de entidades, pilhas de tripla camada, corpo JSON com escapes Unicode (`<`) e o padrão específico de injeção de template `{{7*7}}` do usuário. |
| 05 | `05-null-byte-iis-base64.sh` | curl | Injeção de byte nulo (`%00`) para bypass de extensão, codificação Unicode IIS `%uXXXX` e payloads codificados em Base64 em parâmetros URL com flags de decodificação. |
| 06 | `06-case-chunked-evasion.sh` | curl | Randomização de maiúsculas/minúsculas combinada com codificação (`%3CsCrIpT%3E`), codificação de transferência chunked para dividir palavras-chave SQL/XSS entre chunks HTTP, sondas de conflito Content-Length/Transfer-Encoding e separadores de caminho alternativos (`%5c`, `%c0%af`). |
| 07 | `07-header-cookie-evasion.sh` | curl | Payloads de ataque codificados injetados via cabeçalhos Cookie, Referer, User-Agent, X-Forwarded-For e X-Original-URL. Testa se o WAF inspeciona e decodifica todos os campos de cabeçalho HTTP, não apenas parâmetros de query e corpos POST. |

**Comportamento Esperado do F5 XC:** Um WAF configurado adequadamente deve normalizar todas as camadas de codificação antes da correspondência de padrões. Payloads com codificação simples devem ser bloqueados imediatamente. Testes de codificação dupla e tripla revelam a profundidade do pipeline de decodificação do WAF. Testes de inserção Unicode revelam se caracteres invisíveis são removidos antes da correspondência de palavras-chave. Resultados onde variantes codificadas passam enquanto equivalentes em texto simples são bloqueados indicam lacunas de normalização na política do WAF.
