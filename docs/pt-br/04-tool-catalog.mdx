## Teste de Evasão de WAF

| Ferramenta | Método de Instalação | Comando de Exemplo | Usada pela Suíte |
|---|---|---|---|
| gotestwaf | go install | `cd /opt/gotestwaf && gotestwaf --url https://TARGET --noEmailReport --includePayloads=false` | waf-encoding-evasion |
| waf-bypass | git clone + pip | `waf-bypass --host TARGET:443 --timeout 10 --threads 10` | waf-encoding-evasion |

GoTestWAF (da Wallarm) é um framework padrão da indústria para avaliação de WAF que multiplica payloads através de múltiplos codificadores (URL, Base64, JSUnicode) e zonas de posicionamento (body, header, parâmetro de URL, cookie). waf-bypass (da Nemesida) envia payloads com codificação Base64, HTML-ENTITY e UTF-16 através das zonas ARGS, BODY, COOKIE e HEADER.

## Apenas no Tier Completo

Estas ferramentas são instaladas apenas quando `tool_tier = "full"`:

| Ferramenta | Método de Instalação | Comando de Exemplo | Usada pela Suíte |
|---|---|---|---|
| ZAP (OWASP Zed Attack Proxy) | ghlatest | `zap-cli quick-scan -s all -r https://TARGET` | web-app-attacks |
| Metasploit Framework | installer script | `msfconsole -q -x "use auxiliary/scanner/http/http_version; set RHOSTS TARGET; run; exit"` | reconnaissance |

:::note
ZAP e Metasploit adicionam aproximadamente 1,5 GiB ao espaço em disco e de 5 a 10 minutos ao tempo de provisionamento. Utilize o tier completo apenas se você precisar especificamente dessas ferramentas para uma demonstração.
:::

## Referência de Métodos de Instalação

| Método | Descrição |
|---|---|
| apt | Instalado a partir dos repositórios APT do Ubuntu 24.04 via `apt-get install` |
| ghlatest | Binário mais recente baixado do GitHub Releases usando `ghlatest.sh` e `install-release.sh` |
| uv | Instalado como ferramenta Python usando o gerenciador de pacotes `uv` |
| pip | Instalado como pacote Python usando `pip install` no Python do sistema |
| npm | Instalado como pacote global do Node.js usando `npm install -g` |
| go install | Compilado a partir do código-fonte usando `go install` e instalado em `/usr/local/bin` |
| git clone | Repositório clonado no diretório `/opt/` |
| git clone + pip | Repositório clonado e dependências Python instaladas |
| installer script | Script de instalação fornecido pelo fabricante executado durante o cloud-init |
