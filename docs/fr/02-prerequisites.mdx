## Requis

### Abonnement Azure

Un abonnement Azure actif avec le rôle Contributor ou des permissions équivalentes pour créer :

- Des groupes de ressources
- Des réseaux virtuels et sous-réseaux
- Des groupes de sécurité réseau
- Des adresses IP publiques
- Des machines virtuelles (Standard_F16s_v2 par défaut)

### Azure CLI

Installez et authentifiez l'Azure CLI pour récupérer votre ID d'abonnement :

```bash
curl -sL https://aka.ms/InstallAzureCLIDeb | sudo bash

az login

az account show --query '{name:name, id:id}' -o table
```

Copiez la valeur `id` — il s'agit du `subscription_id` requis par Terraform.

### Terraform

Terraform >= 1.5 est requis :

```bash
terraform version
```

S'il n'est pas installé, utilisez le dépôt APT de HashiCorp :

```bash
wget -O- https://apt.releases.hashicorp.com/gpg | sudo gpg --dearmor -o /usr/share/keyrings/hashicorp-archive-keyring.gpg
echo "deb [signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] https://apt.releases.hashicorp.com $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/hashicorp.list
sudo apt update && sudo apt install terraform
```

### Paire de clés SSH

Une paire de clés SSH pour l'accès à la VM. Ed25519 est recommandé :

```bash
ssh-keygen -t ed25519 -f ~/.ssh/traffic-generator-key -N ""
```

### Accès réseau

La VM du générateur de trafic nécessite un accès HTTPS sortant pendant le provisionnement pour télécharger les outils depuis :

- GitHub Releases (nuclei, dalfox, ffuf, gobuster, feroxbuster, subfinder, httpx, amass)
- PyPI (sqlmap, scapy, impacket, arjun, pwntools, mitmproxy, sslyze, playwright)
- Le registre npm (puppeteer, puppeteer-extra-plugin-stealth)
- Les dépôts APT Ubuntu (nikto, nmap, masscan, hydra, tshark et autres paquets système)
- Les dépôts Git (testssl.sh, recon-ng, spiderfoot, SecLists)

Si votre abonnement Azure applique des règles de pare-feu sortant ou des restrictions NSG, assurez-vous que le HTTPS (port 443) vers ces sources est autorisé pendant la fenêtre de provisionnement initial (15-25 minutes).

### Composants pré-déployés

Avant de déployer le générateur de trafic, vous devez disposer de :

1. Un **serveur d'origine** déployé et accessible (voir la [documentation origin-server](https://f5xc-salesdemos.github.io/origin-server/03-deploy/))
2. Un **répartiteur de charge HTTP F5 XC** configuré avec le serveur d'origine comme backend
3. Une **politique WAF** attachée au répartiteur de charge (pour la validation du trafic WAF)

La variable Terraform `target_fqdn` doit être définie sur le domaine public du répartiteur de charge F5 XC.

:::tip
Utilisez `terraform destroy` lorsque le laboratoire n'est pas utilisé pour éviter les frais continus. Consultez la section [Démontage](../08-teardown/) pour la procédure.
:::

:::note
Standard_F16s_v2 (32 Gio de RAM) est la valeur par défaut car plusieurs outils s'exécutent simultanément pendant l'exécution des suites. L'automatisation de navigateur (Playwright, Puppeteer) avec Chromium en mode headless est le principal consommateur de mémoire. Les SKU F-series plus petits fonctionneront pour les suites sans navigateur, mais peuvent être limités lors de l'exécution de la simulation de bots.
:::
