Chaque suite de trafic est un répertoire sous `/opt/traffic-generator/suites/` contenant des scripts shell numérotés exécutés dans l'ordre par `runner.sh`. Chaque script accepte le FQDN cible comme premier argument et écrit la sortie à la fois vers stdout et le répertoire de résultats.

## api-attacks

**Scripts :** 4 | **Durée estimée :** 10-17 minutes

Tests OWASP API Security Top 10 contre VAmPI via F5 XC. Enregistre un utilisateur de test, obtient un jeton d'authentification, puis teste BOLA, l'authentification par force brute, l'exposition excessive de données, l'autorisation au niveau fonctionnel défaillante, les modèles SSRF, l'injection SQL contre les points de terminaison de l'API REST, la découverte de paramètres cachés et le fuzzing de points de terminaison. Cible l'application VAmPI.

**Fonctionnalité F5 XC :** Sécurité API / Découverte d'API

| Ordre | Script | Outils | Description |
|---|---|---|---|
| 01 | `01-vampi-owasp-top10.sh` | curl, jq | Suite complète de tests OWASP API Top 10. Enregistre un utilisateur de test, obtient un jeton d'authentification, puis teste BOLA (accès aux données d'autres utilisateurs), l'authentification par force brute, l'exposition excessive de données, l'autorisation au niveau fonctionnel défaillante, les modèles SSRF et le sondage de mauvaise configuration de sécurité. |
| 02 | `02-sqlmap-api.sh` | sqlmap | Injection SQL contre les points de terminaison de l'API REST VAmPI : recherche d'utilisateur, connexion (POST) et inscription (POST) avec des charges JSON. |
| 03 | `03-arjun-param-discovery.sh` | arjun | Découverte de paramètres cachés sur les points de terminaison Juice Shop, DVWA et VAmPI. Découvre des paramètres de requête non documentés susceptibles d'accepter des charges d'injection. |
| 04 | `04-ffuf-api-fuzz.sh` | ffuf | Fuzzing en deux phases : découverte de points de terminaison à l'aide d'une liste de mots de chemins API courants (api, swagger, graphql, admin, .env, .git) sur tous les préfixes d'application, puis fuzzing de méthodes HTTP (GET, POST, PUT, DELETE, PATCH, OPTIONS, TRACE, PROPFIND) contre les points de terminaison clés. |

**Comportement attendu de F5 XC :** La Découverte d'API devrait cartographier les points de terminaison de l'API VAmPI et identifier les modèles de requêtes inhabituels. Les politiques de Sécurité API devraient signaler les tentatives d'accès non autorisé, les charges d'injection dans les corps JSON et les modèles d'énumération. L'Inventaire d'API dans F5 XC affichera les points de terminaison découverts avec le nombre de requêtes.

## waf-encoding-evasion

**Scripts :** 7 | **Durée estimée :** 5-10 minutes

Attaques d'évasion par encodage multi-couches conçues pour tester si un WAF normalise et décode correctement les charges avant inspection. Couvre l'encodage URL simple, double et triple ; l'encodage par entités HTML (décimal, hexadécimal, nommé, complété par des zéros) ; l'insertion Unicode/UTF-8 (espace de largeur nulle, BOM, caractères pleine largeur, trait d'union conditionnel, séquences trop longues) ; l'encodage multi-couches mixte/imbriqué (entités HTML encodées en URL, double URL dans les entités, empilements à 3 couches) ; l'injection d'octet nul ; l'encodage IIS %uXXXX ; la manipulation de casse ; le fractionnement du corps par encodage de transfert par morceaux ; l'évasion par paramètres Base64 ; et les charges encodées dans les en-têtes/cookies. Cible n'importe quelle application.

**Fonctionnalité F5 XC :** WAF (profondeur de normalisation de l'encodage)

| Ordre | Script | Outils | Description |
|---|---|---|---|
| 01 | `01-url-encoding.sh` | curl | Encodage URL simple, double et triple des charges SQLi, XSS et de traversée de chemin sur plusieurs chemins de points de terminaison. Teste si le WAF décode les paramètres URL de manière récursive. |
| 02 | `02-html-entity-encoding.sh` | curl, python3 | Encodage par entités HTML décimal, hexadécimal, nommé et complété par des zéros des charges d'attaque dans les paramètres GET et les corps POST. Inclut l'injection de template via `&#123;&#123;7*7&#125;&#125;`. |
| 03 | `03-unicode-utf8-evasion.sh` | curl | Insertion d'espace de largeur nulle (U+200B), BOM (U+FEFF), caractères pleine largeur, trait d'union conditionnel et ZWNJ dans les mots-clés SQL et les balises XSS. Teste les séquences UTF-8 trop longues (2 octets et 3 octets). |
| 04 | `04-mixed-nested-encoding.sh` | curl | Charges multi-couches : entités HTML encodées en URL, double URL dans les entités, empilements triple couche, corps JSON avec échappements Unicode (`<`) et le modèle spécifique d'injection de template `{{7*7}}` de l'utilisateur. |
| 05 | `05-null-byte-iis-base64.sh` | curl | Injection d'octet nul (`%00`) pour le contournement d'extension, encodage Unicode IIS `%uXXXX` et charges encodées en Base64 dans les paramètres URL avec indicateurs de décodage. |
| 06 | `06-case-chunked-evasion.sh` | curl | Randomisation de casse combinée avec l'encodage (`%3CsCrIpT%3E`), encodage de transfert par morceaux pour fractionner les mots-clés SQL/XSS entre les morceaux HTTP, sondes de conflit Content-Length/Transfer-Encoding et séparateurs de chemin alternatifs (`%5c`, `%c0%af`). |
| 07 | `07-header-cookie-evasion.sh` | curl | Charges d'attaque encodées injectées via les en-têtes Cookie, Referer, User-Agent, X-Forwarded-For et X-Original-URL. Teste si le WAF inspecte et décode tous les champs d'en-tête HTTP, pas seulement les paramètres de requête et les corps POST. |

**Comportement attendu de F5 XC :** Un WAF correctement configuré devrait normaliser toutes les couches d'encodage avant la correspondance de motifs. Les charges encodées simplement devraient être bloquées immédiatement. Les tests d'encodage double et triple révèlent la profondeur du pipeline de décodage du WAF. Les tests d'insertion Unicode révèlent si les caractères invisibles sont supprimés avant la correspondance de mots-clés. Les résultats où les variantes encodées passent tandis que les équivalents en texte brut sont bloqués indiquent des lacunes de normalisation dans la politique WAF.
