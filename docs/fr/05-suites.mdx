Chaque suite de trafic est un répertoire sous `/opt/traffic-generator/suites/` contenant des scripts shell numérotés exécutés dans l'ordre par `runner.sh`. Chaque script accepte le FQDN cible comme premier argument et écrit la sortie à la fois vers stdout et le répertoire de résultats.

## api-attacks

**Scripts :** 4 | **Durée estimée :** 10-17 minutes

Tests OWASP API Security Top 10 contre VAmPI via F5 XC. Enregistre un utilisateur de test, obtient un jeton d'authentification, puis teste BOLA, l'authentification par force brute, l'exposition excessive de données, l'autorisation au niveau fonctionnel défaillante, les patterns SSRF, l'injection SQL contre les endpoints REST de l'API, la découverte de paramètres cachés et le fuzzing d'endpoints. Cible l'application VAmPI.

**Fonctionnalité F5 XC :** API Security / API Discovery

| Ordre | Script | Outils | Description |
|---|---|---|---|
| 01 | `01-vampi-owasp-top10.sh` | curl, jq | Suite de tests complète OWASP API Top 10. Enregistre un utilisateur de test, obtient un jeton d'authentification, puis teste BOLA (accès aux données d'autres utilisateurs), l'authentification par force brute, l'exposition excessive de données, l'autorisation au niveau fonctionnel défaillante, les patterns SSRF et le sondage de mauvaise configuration de sécurité. |
| 02 | `02-sqlmap-api.sh` | sqlmap | Injection SQL contre les endpoints REST de l'API VAmPI : recherche d'utilisateur, connexion (POST) et inscription (POST) avec des payloads JSON. |
| 03 | `03-arjun-param-discovery.sh` | arjun | Découverte de paramètres cachés sur les endpoints Juice Shop, DVWA et VAmPI. Découvre les paramètres de requête non documentés pouvant accepter des payloads d'injection. |
| 04 | `04-ffuf-api-fuzz.sh` | ffuf | Fuzzing en deux phases : découverte d'endpoints utilisant une liste de mots de chemins API courants (api, swagger, graphql, admin, .env, .git) sur tous les préfixes d'application, puis fuzzing de méthodes HTTP (GET, POST, PUT, DELETE, PATCH, OPTIONS, TRACE, PROPFIND) contre les endpoints clés. |

**Comportement attendu de F5 XC :** API Discovery devrait cartographier les endpoints de l'API VAmPI et identifier les patterns de requêtes inhabituels. Les politiques API Security devraient signaler les tentatives d'accès non autorisé, les payloads d'injection dans les corps JSON et les patterns d'énumération. L'inventaire API dans F5 XC affichera les endpoints découverts avec le nombre de requêtes.

## bot-simulation

**Scripts :** 4 | **Durée estimée :** 5-10 minutes

Activité de navigation automatisée et patterns de crawling de type bot via F5 XC. Inclut Chrome headless, Puppeteer avec plugin stealth, crawling rapide et interaction automatisée avec les formulaires. Cible toutes les applications.

**Fonctionnalité F5 XC :** Bot Defense

| Ordre | Script | Outils | Description |
|---|---|---|---|
| 01 | `01-headless-chrome.sh` | playwright | Navigation Chromium headless : charge la cible, navigue entre les chemins d'application, extrait les titres et le contenu des pages, prend des captures d'écran. Génère du trafic basé sur le navigateur sans plugins stealth (détectable comme automatisation). |
| 02 | `02-puppeteer-stealth.sh` | puppeteer, puppeteer-extra-plugin-stealth | Navigation stealth utilisant puppeteer-extra avec le plugin stealth. Tente d'éluder la détection de bots en modifiant les propriétés du navigator, WebGL et d'autres empreintes du navigateur. |
| 03 | `03-rapid-crawl.sh` | curl, wget | Requêtes HTTP à haute fréquence utilisant curl et wget en succession rapide. Simule un comportement de scraper sans délai entre les requêtes et avec rotation agressive du User-Agent. |
| 04 | `04-form-interaction.sh` | playwright | Remplissage et soumission automatisés de formulaires contre la connexion DVWA, l'inscription Juice Shop et les endpoints de l'API VAmPI en utilisant les API d'interaction de formulaires de Playwright. |

**Comportement attendu de F5 XC :** Bot Defense devrait classifier le trafic comme automatisé en se basant sur l'analyse de l'empreinte du navigateur, le timing des requêtes et les résultats des défis JavaScript. Chrome headless sans stealth devrait être détecté immédiatement. Les tests Puppeteer stealth vérifient si les techniques d'évasion avancées contournent la détection. Le tableau de bord Bot Defense affichera les catégories de classification des bots et les actions d'atténuation.

## waf-encoding-evasion

**Scripts :** 7 | **Durée estimée :** 5-10 minutes

Attaques d'évasion par encodage multi-couche conçues pour tester si un WAF normalise et décode correctement les payloads avant inspection. Couvre l'encodage URL simple, double et triple ; l'encodage par entités HTML (décimal, hexadécimal, nommé, avec remplissage de zéros) ; l'insertion Unicode/UTF-8 (espace de largeur nulle, BOM, caractères pleine largeur, trait d'union conditionnel, séquences overlong) ; l'encodage mixte/imbriqué multi-couche (entités HTML encodées en URL, double URL dans les entités, piles à 3 couches) ; l'injection d'octet nul ; l'encodage IIS %uXXXX ; la manipulation de casse ; le fractionnement du corps par encodage de transfert en morceaux ; l'évasion par paramètre Base64 ; et les payloads encodés dans les en-têtes/cookies. Cible toute application.

**Fonctionnalité F5 XC :** WAF (profondeur de normalisation d'encodage)

| Ordre | Script | Outils | Description |
|---|---|---|---|
| 01 | `01-url-encoding.sh` | curl | Encodage URL simple, double et triple de payloads SQLi, XSS et de traversée de chemin sur plusieurs chemins d'endpoint. Teste si le WAF décode les paramètres URL de manière récursive. |
| 02 | `02-html-entity-encoding.sh` | curl, python3 | Encodage par entités HTML décimal, hexadécimal, nommé et avec remplissage de zéros des payloads d'attaque dans les paramètres GET et les corps POST. Inclut l'injection de template via `&#123;&#123;7*7&#125;&#125;`. |
| 03 | `03-unicode-utf8-evasion.sh` | curl | Insertion d'espace de largeur nulle (U+200B), BOM (U+FEFF), caractère pleine largeur, trait d'union conditionnel et ZWNJ dans les mots-clés SQL et les balises XSS. Teste les séquences UTF-8 overlong (2 octets et 3 octets). |
| 04 | `04-mixed-nested-encoding.sh` | curl | Payloads multi-couche : entités HTML encodées en URL, double URL dans les entités, piles à triple couche, corps JSON avec échappements Unicode (`<`) et le pattern spécifique d'injection de template `{{7*7}}` de l'utilisateur. |
| 05 | `05-null-byte-iis-base64.sh` | curl | Injection d'octet nul (`%00`) pour le contournement d'extension, encodage Unicode IIS `%uXXXX` et payloads encodés en Base64 dans les paramètres URL avec indicateurs de décodage. |
| 06 | `06-case-chunked-evasion.sh` | curl | Randomisation de casse combinée avec l'encodage (`%3CsCrIpT%3E`), encodage de transfert en morceaux pour fractionner les mots-clés SQL/XSS entre les chunks HTTP, sondes de conflit Content-Length/Transfer-Encoding et séparateurs de chemin alternatifs (`%5c`, `%c0%af`). |
| 07 | `07-header-cookie-evasion.sh` | curl | Payloads d'attaque encodés injectés via les en-têtes Cookie, Referer, User-Agent, X-Forwarded-For et X-Original-URL. Teste si le WAF inspecte et décode tous les champs d'en-tête HTTP, pas seulement les paramètres de requête et les corps POST. |

**Comportement attendu de F5 XC :** Un WAF correctement configuré devrait normaliser toutes les couches d'encodage avant la correspondance de patterns. Les payloads à encodage simple devraient être bloqués immédiatement. Les tests d'encodage double et triple révèlent la profondeur du pipeline de décodage du WAF. Les tests d'insertion Unicode révèlent si les caractères invisibles sont supprimés avant la correspondance de mots-clés. Les résultats où les variantes encodées passent tandis que les équivalents en texte brut sont bloqués indiquent des lacunes de normalisation dans la politique WAF.
